## 注意力在算什么

一句话：**每个查询（query）向所有键（key）打分，用分数加权地把对应的值（value）混合起来**。给定三个矩阵 Q、K、V，形状都是 `seq_len × d`（`seq_len` 个 token，每个是 `d` 维向量），前向公式是：

$$\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{Q K^\top}{\sqrt{d}}\right) V$$

拆成四步就一目了然：

1. **打分**：$S = Q K^\top$，得到一个 `seq_len × seq_len` 的**分数矩阵**——第 $i$ 行第 $j$ 列是"查询 $i$ 和键 $j$ 有多相关"。
2. **缩放**：$S \mathrel{/}= \sqrt{d}$，防止点积随维度变大而爆炸，把值压回 softmax 的敏感区间。
3. **softmax**：对 $S$ 的**每一行**做 softmax——正是上一章那道"每行两次归约"的题。
4. **加权求和**：$O = P V$，用每行的概率把 V 的各行混合成输出。

<div class="note">
<strong>为什么 √d 这个缩放不能省</strong>
两个 $d$ 维向量的点积，方差随 $d$ 线性增长。$d=128$ 时打分动辄上百，softmax 会被推到一个几乎"赢者通吃"的极端区，梯度消失。除以 $\sqrt{d}$ 把方差拉回 1 附近，让 softmax 保持在有区分度的区间。这也顺带解释了上一章为什么执着于数值稳定——注意力打分正是那种"数字能上百、不减 max 就溢出"的典型场景。
</div>

## 朴素做法为什么慢：那个 seq×seq 矩阵是灾难

按上面四步直译，就是四个（或更多）分开的 kernel：矩阵乘算出 S → 缩放 → 逐行 softmax → 再矩阵乘。**能跑，但很浪费。** 问题出在那个中间产物 $S$：它有 `seq_len × seq_len` 这么大。序列长度 $8192$ 时，一个 `float` 的 S 就是 $8192^2 \times 4 \approx 256\,\text{MB}$——而且要在全局内存（HBM）里**来回读写三趟**（softmax 要先读 S 求 max、再读 S 求 sum、再读 S 做归一化）。

回忆第 3 章那个刺眼的事实：**GPU 卡在带宽上，不是算力上**。注意力恰恰是重灾区——真正的乘加运算并不多，时间几乎全花在把这个巨大的 S 矩阵搬进搬出显存。第 14 章讲算子融合时说过：**每多一趟全局内存读写，就多一份带宽税**。朴素注意力要交好几趟。

## FlashAttention 的洞见：不物化 S，就得让 softmax "在线"

融合的思路很自然：**别把整个 S 算出来存着，而是一小块 K/V 算出来、立刻用掉、扔掉，只在片上内存里过一遍。** 但这里有个拦路虎——**softmax 要看到一整行才能算**：它要先求这一行的 max、再求这一行的 sum，才能归一化。可我们现在一次只看到 S 的一小段（一个 K tile 对应的几列），怎么在没看全的情况下做 softmax？

答案就是 **online softmax（在线 softmax）**：一边流式扫过 K/V 的各个 tile，一边维护三个"running（滚动）"状态——

- $m$：到目前为止见过的**最大分数**（running max）；
- $\ell$：到目前为止的 **exp 之和**（running denominator，且已按当前 $m$ 缩放）；
- $\mathbf{acc}$：到目前为止的**输出累加器**（同样已按当前 $m$ 缩放）。

关键在于**修正（rescale）**：每来一个新 tile，可能刷新出更大的 max $m_{\text{new}}$。之前所有基于旧 $m$ 算的 $\ell$ 和 $\mathbf{acc}$ 都"减错了基准"，需要乘上一个校正因子 $e^{m_{\text{old}} - m_{\text{new}}}$ 追平。数学上，因为

$$e^{x - m_{\text{new}}} = e^{x - m_{\text{old}}} \cdot e^{m_{\text{old}} - m_{\text{new}}}$$

所以旧的部分和只要整体乘这个因子，就等价于"从一开始就用 $m_{\text{new}}$ 减"。扫完所有 tile，$\mathbf{acc} / \ell$ 就精确等于对完整一行做 softmax 再乘 V 的结果——**分毫不差**，却从没物化过整行。

## 动手：证明在线 softmax 真的等于朴素注意力

在写任何 GPU 代码之前，先把算法本身钉死。下面这段程序**能在浏览器里直接跑**——点 **▶ 运行**。它用纯标准库同时实现两种注意力：**朴素法**（物化整个 S、逐行完整 softmax、再乘 V）和 **flash 法**（分块流式处理 K/V，维护 running max/分母/累加器）。然后用 `assert` 断言两者逐位相同。这段代码就是 flash 算法的"正确性证书"，也是下面 Triton kernel 的可执行蓝本。特别地，它对 `tile=1、2、4` 都验证——**无论怎么分块，结果都必须一致**，这正是在线修正数学正确的标志。

```python
# 证明 online(flash) softmax attention == 朴素 full-softmax attention
import math

def matmul(A, B):
    n, k, m = len(A), len(A[0]), len(B[0])
    C = [[0.0] * m for _ in range(n)]
    for i in range(n):
        for j in range(m):
            s = 0.0
            for p in range(k):
                s += A[i][p] * B[p][j]
            C[i][j] = s
    return C

def transpose(A):
    return [[A[i][j] for i in range(len(A))] for j in range(len(A[0]))]

# ---------- 朴素法：物化整个 score 矩阵，逐行完整 softmax，再乘 V ----------
def naive_attention(Q, K, V):
    d = len(Q[0])
    scale = 1.0 / math.sqrt(d)
    S = matmul(Q, transpose(K))                 # seq×seq，全部留在“显存”里
    for i in range(len(S)):
        for j in range(len(S[0])):
            S[i][j] *= scale
    P = []
    for row in S:
        m = max(row)
        exps = [math.exp(x - m) for x in row]
        ssum = sum(exps)
        P.append([e / ssum for e in exps])
    return matmul(P, V)

# ---------- 在线(flash)法：分块流式处理 K/V，维护 running max/分母/累加器 ----------
def flash_attention(Q, K, V, tile):
    d, dv, N = len(Q[0]), len(V[0]), len(K)
    scale = 1.0 / math.sqrt(d)
    out = []
    for qi in range(len(Q)):
        q = Q[qi]
        m = -math.inf          # running max
        l = 0.0                # running 分母（已按当前 m 缩放）
        acc = [0.0] * dv       # running 输出累加器（已按当前 m 缩放）
        for start in range(0, N, tile):        # 流式扫过 K/V 的每个 tile
            end = min(start + tile, N)
            s = []                              # 本 tile 的 score
            for j in range(start, end):
                dot = 0.0
                for p in range(d):
                    dot += q[p] * K[j][p]
                s.append(dot * scale)
            m_new = max(m, max(s))              # 可能刷新出更大的 max
            corr = math.exp(m - m_new) if m != -math.inf else 0.0  # 旧统计量的校正因子
            p_tile = [math.exp(x - m_new) for x in s]              # 本 tile 相对新 max 的 exp
            l = l * corr + sum(p_tile)          # 分母：旧的追平 + 新的加入
            for t in range(dv):
                acc[t] *= corr                  # 累加器：旧的先追平
            for idx, j in enumerate(range(start, end)):
                for t in range(dv):
                    acc[t] += p_tile[idx] * V[j][t]   # 再加入本 tile 的贡献
            m = m_new
        out.append([acc[t] / l for t in range(dv)])   # 收尾：归一化
    return out

def max_abs_diff(A, B):
    d = 0.0
    for i in range(len(A)):
        for j in range(len(A[0])):
            d = max(d, abs(A[i][j] - B[i][j]))
    return d

Q = [[0.1, 0.2, 0.3, 0.4], [1.0, 0.0, -1.0, 0.5],
     [0.3, 0.3, 0.3, 0.3], [2.0, 1.0, 0.0, -1.0]]
K = [[0.5, 0.1, 0.2, 0.0], [0.0, 1.0, 0.0, 0.5],
     [1.0, 1.0, 1.0, 1.0], [-0.5, 0.2, 0.3, 0.1]]
V = [[1.0, 0.0], [0.0, 1.0], [0.5, 0.5], [2.0, -1.0]]

ref = naive_attention(Q, K, V)
print("朴素法（物化整个 score 矩阵，逐行完整 softmax，再乘 V）：")
for row in ref:
    print("  [" + ", ".join(f"{v:.6f}" for v in row) + "]")

for tile in [1, 2, 4]:
    fl = flash_attention(Q, K, V, tile)
    err = max_abs_diff(ref, fl)
    print(f"\n在线 softmax（flash），tile={tile}：逐块更新 running max/分母并缩放累加器")
    for row in fl:
        print("  [" + ", ".join(f"{v:.6f}" for v in row) + "]")
    assert err < 1e-9, f"tile={tile} 不一致，误差 {err}"
    print(f"  与朴素法比对：最大误差 {err:.1e} < 1e-9 → 完全一致 ✓")

print("\n结论：无论 K/V 分几块流式处理，在线 softmax 的结果都和朴素法逐位相同，")
print("但它从不物化整个 seq×seq 的 score 矩阵——这就是 FlashAttention 省显存的根据。")
```

## Triton flash 前向：把模拟器搬上 GPU

现在把上面验证过的算法翻译成 Triton（第 13、14 章）。这是 flash attention 最好上手的写法——一个 **program 负责一个查询块（几行 Q）**，把 Q tile 载进片上，然后用一个 `for` 循环流式扫过 K/V 的各个 tile，在寄存器里维护 `m`、`l`、`acc`。逻辑和上面的 `flash_attention` 函数**一一对应**，只是把标量换成了 Triton 的块级张量运算。causal（因果掩码，只让每个 query 看到它之前的 key）用一个开关控制。

```triton
import torch
import triton
import triton.language as tl

@triton.jit
def flash_fwd_kernel(
    Q, K, V, O,
    stride_qm, stride_qk,          # Q: (seq, d) 的两个步长
    stride_kn, stride_kk,          # K: (seq, d)
    stride_vn, stride_vk,          # V: (seq, d)
    stride_om, stride_ok,          # O: (seq, d)
    seq_len, scale,
    BLOCK_M: tl.constexpr,         # 每个 program 处理的 query 行数
    BLOCK_N: tl.constexpr,         # 每个 K/V tile 的行数
    D: tl.constexpr,               # head 维度（编译期常量）
    CAUSAL: tl.constexpr,
):
    pid = tl.program_id(0)                        # 第几个 query 块
    offs_m = pid * BLOCK_M + tl.arange(0, BLOCK_M)   # 本块负责的 query 行号
    offs_d = tl.arange(0, D)

    # 载入 Q tile，固定在寄存器 (BLOCK_M, D)
    q_ptrs = Q + offs_m[:, None] * stride_qm + offs_d[None, :] * stride_qk
    q_mask = offs_m[:, None] < seq_len
    q = tl.load(q_ptrs, mask=q_mask, other=0.0)

    # 滚动状态：running max、分母、累加器
    m_i = tl.full((BLOCK_M,), -float('inf'), dtype=tl.float32)   # running max
    l_i = tl.zeros((BLOCK_M,), dtype=tl.float32)                 # running 分母
    acc = tl.zeros((BLOCK_M, D), dtype=tl.float32)               # running 累加器

    # 因果时，本 query 块只需扫到它的最后一行为止
    n_end = seq_len
    if CAUSAL:
        n_end = (pid + 1) * BLOCK_M

    # 流式扫过 K/V 的每个 tile
    for start_n in range(0, n_end, BLOCK_N):
        offs_n = start_n + tl.arange(0, BLOCK_N)
        # 载入 K tile (BLOCK_N, D)、V tile (BLOCK_N, D)
        k_ptrs = K + offs_n[:, None] * stride_kn + offs_d[None, :] * stride_kk
        v_ptrs = V + offs_n[:, None] * stride_vn + offs_d[None, :] * stride_vk
        n_mask = offs_n[:, None] < seq_len
        k = tl.load(k_ptrs, mask=n_mask, other=0.0)
        v = tl.load(v_ptrs, mask=n_mask, other=0.0)

        # 本 tile 的分数 s = q · kᵀ · scale，形状 (BLOCK_M, BLOCK_N)
        s = tl.dot(q, tl.trans(k)) * scale

        # 无效列（越界 / 因果未来位置）压成 -inf，exp 后为 0，不参与
        col_valid = offs_n[None, :] < seq_len
        if CAUSAL:
            col_valid = col_valid & (offs_m[:, None] >= offs_n[None, :])
        s = tl.where(col_valid, s, -float('inf'))

        # ---- 在线 softmax 更新（对照模拟器逐行） ----
        m_tile = tl.max(s, axis=1)               # 本 tile 每行的最大分数
        m_new = tl.maximum(m_i, m_tile)          # 刷新 running max
        corr = tl.exp(m_i - m_new)               # 旧统计量的校正因子
        p = tl.exp(s - m_new[:, None])           # 本 tile 相对新 max 的 exp
        l_i = l_i * corr + tl.sum(p, axis=1)     # 分母：追平旧的 + 加入新的
        acc = acc * corr[:, None]                # 累加器：先把旧的追平
        acc += tl.dot(p.to(v.dtype), v)          # 再加入本 tile 的 p·V
        m_i = m_new

    # 收尾：归一化并写回
    o = acc / l_i[:, None]
    o_ptrs = O + offs_m[:, None] * stride_om + offs_d[None, :] * stride_ok
    tl.store(o_ptrs, o, mask=offs_m[:, None] < seq_len)


def flash_attention(q, k, v, causal=False):
    seq_len, d = q.shape
    scale = 1.0 / (d ** 0.5)
    o = torch.empty_like(q)
    BLOCK_M, BLOCK_N = 32, 32
    grid = (triton.cdiv(seq_len, BLOCK_M),)       # 每个 program 一个 query 块
    flash_fwd_kernel[grid](
        q, k, v, o,
        q.stride(0), q.stride(1),
        k.stride(0), k.stride(1),
        v.stride(0), v.stride(1),
        o.stride(0), o.stride(1),
        seq_len, scale,
        BLOCK_M=BLOCK_M, BLOCK_N=BLOCK_N, D=d, CAUSAL=causal,
    )
    return o


if __name__ == "__main__":
    torch.manual_seed(0)
    seq_len, d = 128, 64
    q = torch.randn(seq_len, d, device="cuda")
    k = torch.randn(seq_len, d, device="cuda")
    v = torch.randn(seq_len, d, device="cuda")

    def ref_attention(q, k, v, causal):
        scale = 1.0 / (d ** 0.5)
        s = (q @ k.T) * scale
        if causal:
            mask = torch.tril(torch.ones(seq_len, seq_len, device="cuda"))
            s = s.masked_fill(mask == 0, float('-inf'))
        p = torch.softmax(s, dim=1)
        return p @ v

    for causal in (False, True):
        out = flash_attention(q, k, v, causal=causal)
        ref = ref_attention(q, k, v, causal)
        print(f"causal={causal}  最大误差：{(out - ref).abs().max().item():.2e}")
```

Triton kernel 里的 <code>m_i / l_i / acc</code>，就是模拟器里的 <code>m / l / acc</code>；<code>corr = tl.exp(m_i - m_new)</code> 就是那句 <code>corr = exp(m_old - m_new)</code>；<code>acc * corr + tl.dot(p, v)</code> 就是"先追平旧的、再加入新的"。区别只有一个：模拟器一次处理一个 query 一个标量分数，Triton 一次处理 <strong>BLOCK_M 行 query × BLOCK_N 列 key 的一整块</strong>——<code>tl.dot</code> 走 Tensor Core（第 19 章），<code>tl.max/tl.sum</code> 走块级归约（第 18 章、第 20 章）。<strong>同一个算法，从标量循环长成了块级张量流水线。</strong>

## CUDA tiled 注意力：一份可编译校验的对照件

Triton 版最贴近生产级 flash，但为了给你一个**能在浏览器点开、能本地编译验证**的 C++ artifact，下面给一份**作用域清晰的 tiled 注意力** CUDA 程序：**一个 block 负责一个 query 行**，把整行分数放进 shared memory（因此要求 `seq_len` 不太大，这里取 256），做标准的两趟归约 softmax（第 20 章的 CUDA 版），再加权求和 V。它**不是** flash（会在 shared memory 里存下一整行分数），但把注意力的骨架完整跑通，并对着 CPU 参考校验、打印 OK——用它确认你对四步流程的理解，Triton 版负责展示 flash 的精髓。

```cuda
#include <cstdio>
#include <cmath>
#include <cuda_runtime.h>

#define SEQ  256   // 序列长度（一行分数要放进 shared memory，故不宜过大）
#define DIM  64    // head 维度
#define THREADS 256

// 一个 block 负责一个 query 行 i：算 S[i,:]→softmax→乘 V，得到 O[i,:]
__global__ void attention(const float* Q, const float* K, const float* V,
                          float* O, int seq, int d, float scale) {
    int row = blockIdx.x;                  // 我负责第几个 query
    int tid = threadIdx.x;
    __shared__ float s[SEQ];               // 本行的分数 S[row, 0..seq-1]
    __shared__ float buf[THREADS];         // 归约暂存

    // ---- 1. 打分：s[j] = (Q[row] · K[j]) * scale ----
    for (int j = tid; j < seq; j += blockDim.x) {
        float dot = 0.0f;
        for (int p = 0; p < d; ++p)
            dot += Q[row * d + p] * K[j * d + p];
        s[j] = dot * scale;
    }
    __syncthreads();

    // ---- 2. 第一次归约：本行最大值 ----
    float lmax = -INFINITY;
    for (int j = tid; j < seq; j += blockDim.x) lmax = fmaxf(lmax, s[j]);
    buf[tid] = lmax; __syncthreads();
    for (int k = blockDim.x / 2; k > 0; k >>= 1) {
        if (tid < k) buf[tid] = fmaxf(buf[tid], buf[tid + k]);
        __syncthreads();
    }
    float row_max = buf[0]; __syncthreads();

    // ---- 3. 第二次归约：sum(exp(s - max))，顺手把 s 就地换成 exp ----
    float lsum = 0.0f;
    for (int j = tid; j < seq; j += blockDim.x) {
        float e = expf(s[j] - row_max);
        s[j] = e;                          // s 现在存的是未归一化的概率
        lsum += e;
    }
    buf[tid] = lsum; __syncthreads();
    for (int k = blockDim.x / 2; k > 0; k >>= 1) {
        if (tid < k) buf[tid] += buf[tid + k];
        __syncthreads();
    }
    float inv = 1.0f / buf[0]; __syncthreads();

    // ---- 4. 加权求和：O[row, t] = Σ_j (s[j]*inv) * V[j, t] ----
    for (int t = tid; t < d; t += blockDim.x) {
        float acc = 0.0f;
        for (int j = 0; j < seq; ++j)
            acc += s[j] * inv * V[j * d + t];
        O[row * d + t] = acc;
    }
}

int main() {
    const int seq = SEQ, d = DIM;
    const int nQ = seq * d, bytes = nQ * sizeof(float);
    float *hQ = (float*)malloc(bytes), *hK = (float*)malloc(bytes);
    float *hV = (float*)malloc(bytes), *hO = (float*)malloc(bytes);
    for (int i = 0; i < nQ; ++i) {         // 造点可复现的数据
        hQ[i] = 0.01f * ((i * 7)  % 13 - 6);
        hK[i] = 0.01f * ((i * 5)  % 11 - 5);
        hV[i] = 0.01f * ((i * 3)  % 17 - 8);
    }
    float *dQ, *dK, *dV, *dO;
    cudaMalloc(&dQ, bytes); cudaMalloc(&dK, bytes);
    cudaMalloc(&dV, bytes); cudaMalloc(&dO, bytes);
    cudaMemcpy(dQ, hQ, bytes, cudaMemcpyHostToDevice);
    cudaMemcpy(dK, hK, bytes, cudaMemcpyHostToDevice);
    cudaMemcpy(dV, hV, bytes, cudaMemcpyHostToDevice);

    float scale = 1.0f / sqrtf((float)d);
    attention<<<seq, THREADS>>>(dQ, dK, dV, dO, seq, d, scale);  // 一行一个 block
    cudaMemcpy(hO, dO, bytes, cudaMemcpyDeviceToHost);

    // ---- CPU 参考（double 精度）：完整 softmax attention ----
    double max_err = 0.0;
    for (int i = 0; i < seq; ++i) {
        double sc[SEQ], m = -1e30, sum = 0.0;
        for (int j = 0; j < seq; ++j) {
            double dot = 0.0;
            for (int p = 0; p < d; ++p) dot += (double)hQ[i*d+p] * hK[j*d+p];
            sc[j] = dot * scale;
            if (sc[j] > m) m = sc[j];
        }
        for (int j = 0; j < seq; ++j) { sc[j] = exp(sc[j] - m); sum += sc[j]; }
        for (int t = 0; t < d; ++t) {
            double acc = 0.0;
            for (int j = 0; j < seq; ++j) acc += sc[j] / sum * hV[j*d+t];
            max_err = fmax(max_err, fabs(acc - hO[i*d+t]));
        }
    }
    printf("seq=%d d=%d，与 CPU 参考的最大误差 = %.2e\n", seq, d, max_err);
    printf("O[0] 前 3 个：%.6f %.6f %.6f\n", hO[0], hO[1], hO[2]);
    printf(max_err < 1e-4 ? "结果正确：OK\n" : "结果不对！\n");

    cudaFree(dQ); cudaFree(dK); cudaFree(dV); cudaFree(dO);
    free(hQ); free(hK); free(hV); free(hO);
    return 0;
}
```

<div class="warn">
<strong>这份 CUDA 版为什么不是 flash</strong>
它把整行分数 <code>s[SEQ]</code> 存进了 shared memory——一旦 <code>seq_len</code> 上到几千，shared memory 根本放不下，这正是朴素法的软肋在 block 尺度上的重演。要把它改造成真 flash，就得像 Triton 版那样：<strong>不存整行</strong>，改成流式扫 K/V 的 tile、在寄存器里维护 <code>m/l/acc</code>。留给你的进阶练习：照着模拟器和 Triton 版，把这个 CUDA kernel 改写成分块在线 softmax，让它不再依赖 <code>s[SEQ]</code> 这块随 seq 膨胀的 shared memory。真能写出来，你就完整掌握了 FlashAttention。
</div>

**这一章的要点**

- **注意力前向 = 四步**：打分 $S=QK^\top$ → 缩放 $/\sqrt{d}$ → 逐行 softmax → 加权求和 $PV$。中间的 softmax 正是第 20 章"每行两次归约"的题。
- **朴素多 kernel 慢在带宽**：它物化 `seq×seq` 的 S 矩阵并在 HBM 里来回读写好几趟，显存 $O(\text{seq}^2)$——GPU 的瓶颈从来是访存（第 3 章）。
- **FlashAttention 的核心是 online softmax**：流式扫 K/V tile，维护 running max $m$、分母 $\ell$、累加器 $\mathbf{acc}$，用校正因子 $e^{m_{\text{old}}-m_{\text{new}}}$ 追平旧统计量。从不物化 S，显存降到 $O(\text{seq})$。
- 在线 softmax **数学上等于**朴素法——模拟器对 `tile=1/2/4` 逐位验证，去掉校正因子就立刻算错，证明了修正项的不可或缺。
- **Triton flash kernel** 与模拟器逐行对应，只是把标量升成块级张量：`tl.dot` 走 Tensor Core、`tl.max/tl.sum` 走块级归约，融 tiling（第 11 章）+ 归约（第 18 章）+ shared memory（第 7 章）+ 流水线（第 19 章）于一体。
- 附带的 **CUDA tiled 注意力**把四步骨架跑通并对 CPU 校验，但把整行分数存进 shared memory 而非 flash——进阶练习是把它改写成分块在线 softmax。
- 全书主线：**GPU 性能几乎总关乎内存而非算力**；FlashAttention 是这句话最漂亮的兑现。下一站是附录——搭环境、查对照表、用 profiler、继续前进。
