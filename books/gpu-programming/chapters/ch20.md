## 先说清楚：softmax 是什么，以及它藏着的两次归约

softmax 把一行实数变成一个概率分布——每个数变成正数、且整行加起来等于 1。对一行 $x_0, x_1, \dots, x_{n-1}$，第 $i$ 个输出是：

$$\text{softmax}(x)_i = \frac{e^{x_i}}{\sum_j e^{x_j}}$$

它在深度学习里无处不在（注意力机制的核心一步就是它，第 21 章会看到）。我们要做的是**按行 softmax**：给一个矩阵，对**每一行**独立做一次 softmax。这正好是第 18 章"归约"的直接应用——只不过每一行需要的不是一次归约，而是**两次**：

1. 先求这一行的**最大值**（一次 max 归约）；
2. 再求 $\sum_j e^{x_j - \max}$（一次 sum 归约）。

为什么需要那个"减最大值"？因为 $e^{x}$ 增长极快：`float` 到 $x \approx 88$ 就 `exp` 溢出成 `inf` 了。而注意力打分动辄上百。技巧是利用一个恒等式——分子分母同乘 $e^{-\max}$ 结果不变：

$$\frac{e^{x_i}}{\sum_j e^{x_j}} = \frac{e^{x_i - \max}}{\sum_j e^{x_j - \max}}$$

减掉最大值后，最大的那个指数变成 $e^0 = 1$，其余都 $\le 1$，**永不溢出，结果却分毫不差**。这就是**数值稳定 softmax（numerically-stable softmax）**，是所有正经实现的标配。

<div class="keypoint">
<strong>把 softmax 看成一条数据流水线</strong>
每一行都要走五步：<strong>读入这一行 → 求 max（归约①）→ 减 max 后取 exp → 求 sum（归约②）→ 每个元素除以 sum</strong>。四种 GPU 技术做的是同一条流水线，区别只在于<strong>"谁来做那两次归约"</strong>：CUDA/SYCL 要你亲手用 shared memory 把 block/work-group 里的线程折叠成一个值；Triton 给你一个 <code>tl.max</code>／<code>tl.sum</code> 直接对整块求；CuTe 则让你在更细的 layout 层面掌控数据怎么摆。
</div>

真正拉开四者写法差距的，是那两次归约。CUDA/SYCL 是**线程级**思维：你要亲手指挥一群线程，先各扫一段、写进 shared memory，再用一棵折叠树把它们收成一个值。Triton 是**块级**思维：你把一整行当成一个"块"，`tl.max(x, axis=0)` 一句话就把归约交给编译器，它自己去决定用什么 warp/shared 策略。

## 动手：数值稳定 softmax 的可执行规范

下面这段程序**能在浏览器里直接跑**——点 **▶ 运行**。它只用标准库，逐行做数值稳定 softmax，把每一行的中间量（max、sum）打印出来，并断言每一行加起来等于 1。**四个 GPU 版本，跑出来都必须和它一致**——它就是本章的"黄金参考"。特别注意第 2 行那个数字上百的例子：它演示了不减最大值就会溢出的真实风险。

```python
# 数值稳定的 row-wise softmax：这是四种 GPU 实现共同的“可执行规范”
import math

def softmax_row(row):
    # 第一遍归约：求这一行的最大值（数值稳定的关键）
    m = row[0]
    for x in row[1:]:
        if x > m:
            m = x
    # 减最大值再取 exp，避免 exp 溢出
    exps = [math.exp(x - m) for x in row]
    # 第二遍归约：求 exp 之和
    s = 0.0
    for e in exps:
        s += e
    # 归一化
    out = [e / s for e in exps]
    return out, m, s

matrix = [
    [1.0, 2.0, 3.0, 4.0],
    [0.0, 0.0, 0.0, 0.0],
    [100.0, 101.0, 102.0, 103.0],   # 大数：不减最大值就会 exp 溢出
]

print("逐行做数值稳定 softmax（先求 max，再求 sum，两次归约）：")
for r, row in enumerate(matrix):
    out, m, s = softmax_row(row)
    pretty = "[" + ", ".join(f"{v:.4f}" for v in out) + "]"
    print(f"  行 {r}: max={m:.1f}  sum(exp)={s:.4f}  softmax={pretty}")
    # 验证：每行必须和为 1
    total = 0.0
    for v in out:
        total += v
    assert abs(total - 1.0) < 1e-9, total
    print(f"       校验：这一行加起来 = {total:.6f}  → 约等于 1.0 ✓")

print("\n关键观察：行 2 的数字高达 100+，若不先减去 max，exp(103) 会溢出成 inf；")
print("减去每行的 max 后，最大的指数变成 exp(0)=1，结果完全不变，却永不溢出。")
```

## CUDA：一个 block 负责一行，shared memory 做两次归约

最贴近硬件的写法（第 10、11 章）。分工策略是 GPU 上做按行归约的经典套路：**一个 block 负责一整行**，block 里的线程先各自扫过这一行的若干列，再用第 18 章那棵折叠树在 shared memory 里归约成一个值。max 和 sum 是同一套归约代码，只是折叠时一个用 `fmaxf`、一个用 `+`。下面是**完整可编译**的 `.cu`，跑完对着 CPU 参考逐行校验。

```cuda
#include <cstdio>
#include <cmath>
#include <cuda_runtime.h>

#define THREADS 256   // 每个 block 的线程数，取 2 的幂便于折叠归约

// 一个 block 负责一行；block 内线程协作做两次归约（先 max，再 sum）
__global__ void softmax_rows(const float* in, float* out, int rows, int cols) {
    int row = blockIdx.x;                 // 我这个 block 负责第几行
    if (row >= rows) return;
    int tid = threadIdx.x;
    const float* x = in  + row * cols;
    float*       y = out + row * cols;

    __shared__ float buf[THREADS];

    // ---- 第一次归约：求本行最大值 ----
    float local_max = -INFINITY;
    for (int c = tid; c < cols; c += blockDim.x)   // 每个线程跨步扫自己那几列
        local_max = fmaxf(local_max, x[c]);
    buf[tid] = local_max;
    __syncthreads();
    for (int s = blockDim.x / 2; s > 0; s >>= 1) { // 折叠树：256→128→…→1
        if (tid < s) buf[tid] = fmaxf(buf[tid], buf[tid + s]);
        __syncthreads();
    }
    float row_max = buf[0];
    __syncthreads();

    // ---- 第二次归约：求 sum(exp(x - max)) ----
    float local_sum = 0.0f;
    for (int c = tid; c < cols; c += blockDim.x)
        local_sum += expf(x[c] - row_max);
    buf[tid] = local_sum;
    __syncthreads();
    for (int s = blockDim.x / 2; s > 0; s >>= 1) {
        if (tid < s) buf[tid] += buf[tid + s];
        __syncthreads();
    }
    float row_sum = buf[0];
    __syncthreads();

    // ---- 归一化写回 ----
    float inv = 1.0f / row_sum;
    for (int c = tid; c < cols; c += blockDim.x)
        y[c] = expf(x[c] - row_max) * inv;
}

int main() {
    const int rows = 4, cols = 2000;                // cols 故意不是线程数的整数倍
    const int N = rows * cols, bytes = N * sizeof(float);

    float* h_in  = (float*)malloc(bytes);
    float* h_out = (float*)malloc(bytes);
    for (int r = 0; r < rows; ++r)
        for (int c = 0; c < cols; ++c)
            h_in[r * cols + c] = 0.001f * c + r;    // 造点有区分度的数据

    float *d_in, *d_out;
    cudaMalloc(&d_in,  bytes);
    cudaMalloc(&d_out, bytes);
    cudaMemcpy(d_in, h_in, bytes, cudaMemcpyHostToDevice);

    softmax_rows<<<rows, THREADS>>>(d_in, d_out, rows, cols);  // 一行一个 block
    cudaMemcpy(h_out, d_out, bytes, cudaMemcpyDeviceToHost);

    // ---- CPU 参考：逐行数值稳定 softmax，比对最大误差 ----
    double max_err = 0.0;
    for (int r = 0; r < rows; ++r) {
        const float* x = h_in + r * cols;
        float m = -INFINITY;
        for (int c = 0; c < cols; ++c) m = fmaxf(m, x[c]);
        double s = 0.0;
        for (int c = 0; c < cols; ++c) s += exp((double)x[c] - m);
        for (int c = 0; c < cols; ++c) {
            double ref = exp((double)x[c] - m) / s;
            double err = fabs(ref - h_out[r * cols + c]);
            if (err > max_err) max_err = err;
        }
    }
    printf("行数=%d 列数=%d，与 CPU 参考的最大误差 = %.2e\n", rows, cols, max_err);
    printf("行 0 前 3 个：%.6f %.6f %.6f\n", h_out[0], h_out[1], h_out[2]);
    printf(max_err < 1e-5 ? "结果正确：OK\n" : "结果不对！\n");

    cudaFree(d_in); cudaFree(d_out);
    free(h_in); free(h_out);
    return 0;
}
```

注意 `cols = 2000` 故意不是 256 的整数倍——`for (int c = tid; c < cols; c += blockDim.x)` 这个**跨步循环（grid-stride 式）**让每个线程处理多列，把任意宽度的行摊到固定 256 个线程上。这是第 18 章"数据比线程多时怎么归约"的标准手法。改一行试试：把 `THREADS` 从 256 改成 128，结果和误差都不会变——**归约的线程数与算法正确性无关**，只影响速度。

## SYCL：一个 work-group 负责一行，local_accessor 做归约

同一套分工，换成第 12 章的可移植写法。CUDA 的 `blockIdx.x` → SYCL 的 `it.get_group(0)`，`threadIdx.x` → `it.get_local_id(0)`，`__shared__` → `local_accessor`，`__syncthreads()` → `it.barrier()`（对照表见附录 B）。逻辑与 CUDA 版**逐行对应**，这正是本章要你看到的——换外衣，不换算法。

```sycl
#include <sycl/sycl.hpp>
#include <cmath>
#include <cstdio>

constexpr int WG = 128;   // 每个 work-group 的 work-item 数，取 2 的幂

int main() {
    const int rows = 4, cols = 2000;
    const int N = rows * cols;
    sycl::queue q;

    // 用 USM shared memory，host/device 都能直接访问
    float* in  = sycl::malloc_shared<float>(N, q);
    float* out = sycl::malloc_shared<float>(N, q);
    for (int r = 0; r < rows; ++r)
        for (int c = 0; c < cols; ++c)
            in[r * cols + c] = 0.001f * c + r;

    q.submit([&](sycl::handler& h) {
        // 每个 work-group 一块局部内存，等价于 CUDA 的 __shared__
        sycl::local_accessor<float, 1> buf(sycl::range<1>(WG), h);
        // 总 work-item = rows 个 work-group × 每组 WG 个
        sycl::nd_range<1> ndr(sycl::range<1>(rows * WG), sycl::range<1>(WG));
        h.parallel_for(ndr, [=](sycl::nd_item<1> it) {
            int row = it.get_group(0);            // 我这个 work-group 负责第几行
            int tid = it.get_local_id(0);
            const float* x = in  + row * cols;
            float*       y = out + row * cols;
            auto fence = sycl::access::fence_space::local_space;

            // ---- 第一次归约：max ----
            float local_max = -INFINITY;
            for (int c = tid; c < cols; c += WG)
                local_max = sycl::fmax(local_max, x[c]);
            buf[tid] = local_max;
            it.barrier(fence);
            for (int s = WG / 2; s > 0; s >>= 1) {
                if (tid < s) buf[tid] = sycl::fmax(buf[tid], buf[tid + s]);
                it.barrier(fence);
            }
            float row_max = buf[0];
            it.barrier(fence);

            // ---- 第二次归约：sum(exp(x - max)) ----
            float local_sum = 0.0f;
            for (int c = tid; c < cols; c += WG)
                local_sum += sycl::exp(x[c] - row_max);
            buf[tid] = local_sum;
            it.barrier(fence);
            for (int s = WG / 2; s > 0; s >>= 1) {
                if (tid < s) buf[tid] += buf[tid + s];
                it.barrier(fence);
            }
            float row_sum = buf[0];
            it.barrier(fence);

            // ---- 归一化写回 ----
            float inv = 1.0f / row_sum;
            for (int c = tid; c < cols; c += WG)
                y[c] = sycl::exp(x[c] - row_max) * inv;
        });
    }).wait();

    // ---- CPU 参考校验 ----
    double max_err = 0.0;
    for (int r = 0; r < rows; ++r) {
        const float* x = in + r * cols;
        float m = -INFINITY;
        for (int c = 0; c < cols; ++c) m = std::fmax(m, x[c]);
        double s = 0.0;
        for (int c = 0; c < cols; ++c) s += std::exp((double)x[c] - m);
        for (int c = 0; c < cols; ++c) {
            double ref = std::exp((double)x[c] - m) / s;
            max_err = std::fmax(max_err, std::fabs(ref - out[r * cols + c]));
        }
    }
    printf("行数=%d 列数=%d，与 CPU 参考的最大误差 = %.2e\n", rows, cols, max_err);
    printf(max_err < 1e-5 ? "结果正确：OK\n" : "结果不对！\n");

    sycl::free(in, q);
    sycl::free(out, q);
    return 0;
}
```

## Triton：一个 program 负责一行，归约是一句话

到了 Triton（第 13、14 章），抽象层次陡然抬高。你不再管"线程"，而是让**一个 program 处理一整行**——把整行 `tl.load` 成一个块，然后 `tl.max`、`tl.sum` 直接对块归约，折叠树、shared memory、warp shuffle 全由编译器生成。上面 CUDA 里那二十行归约，这里各缩成一句话。

```triton
import torch
import triton
import triton.language as tl

@triton.jit
def softmax_kernel(in_ptr, out_ptr, in_row_stride, out_row_stride,
                   n_cols, BLOCK_SIZE: tl.constexpr):
    # 一个 program 处理一行——program_id 就是行号
    row = tl.program_id(0)
    cols = tl.arange(0, BLOCK_SIZE)              # 0,1,...,BLOCK_SIZE-1
    mask = cols < n_cols                         # 尾部超出真实列数的要屏蔽
    in_ptrs = in_ptr + row * in_row_stride + cols

    # 越界位置填 -inf：它们既不会赢得 max，exp 后也是 0，不影响 sum
    x = tl.load(in_ptrs, mask=mask, other=-float('inf'))

    row_max = tl.max(x, axis=0)                  # 第一次归约：一句话
    z = x - row_max
    num = tl.exp(z)                              # exp(-inf)=0，尾部自动清零
    denom = tl.sum(num, axis=0)                  # 第二次归约：一句话
    y = num / denom

    out_ptrs = out_ptr + row * out_row_stride + cols
    tl.store(out_ptrs, y, mask=mask)             # 只写回真实的列


def softmax(x: torch.Tensor) -> torch.Tensor:
    rows, cols = x.shape
    # 块必须能装下一整行；取不小于 cols 的 2 的幂
    BLOCK_SIZE = triton.next_power_of_2(cols)
    out = torch.empty_like(x)
    # 启动网格：一维，rows 个 program，每个处理一行
    softmax_kernel[(rows,)](
        x, out,
        x.stride(0), out.stride(0),
        cols, BLOCK_SIZE=BLOCK_SIZE,
    )
    return out


if __name__ == "__main__":
    torch.manual_seed(0)
    x = torch.randn(4, 2000, device="cuda")
    y = softmax(x)
    ref = torch.softmax(x, dim=1)           # PyTorch 参考实现
    print("最大误差：", (y - ref).abs().max().item())
    print("每行和：", y.sum(dim=1))          # 应当全是 1.0
```

<div class="warn">
<strong>尾部（tail）不能不管</strong>
真实列数 <code>n_cols</code> 很少正好是 2 的幂，所以块里总有一截"多出来的通道"。<code>mask = cols &lt; n_cols</code> 就是第 13 章讲的尾部掩码。诀窍在 <code>other=-float('inf')</code>：把越界位置读成 -∞，它既当不上 max，<code>exp(-inf)=0</code> 又不给 sum 添分，于是尾部对结果<strong>零污染</strong>。这和 CUDA 版里 <code>if (i &lt; n)</code>、SYCL 版里 <code>c &lt; cols</code> 是同一件事的不同表达——处理"数据不是线程整数倍"的那个永恒问题。
</div>

## CuTe / CUTLASS：什么时候你会需要它

前三种技术已经能把 softmax 写得又对又快，为什么还有第四件外衣？答案不在 softmax 本身，而在**它之后**。看第 15 章的 CuTe DSL 和 CUTLASS，你不会为了单跑一个 softmax 去动用它们——那是杀鸡用牛刀。你会在下面这些时候伸手去够它：

- **要把 softmax 融进一个更大的算子**。第 21 章的注意力就是 `softmax(Q·Kᵀ)·V`——softmax 夹在两个矩阵乘之间。用 CuTe，你能让 softmax 的结果**直接留在寄存器/shared memory 里**喂给下一个矩阵乘，而不是写回全局内存再读一遍（第 14 章讲的算子融合的极致形态）。
- **要精确掌控数据的摆放（layout）**。CuTe 的核心是 `Layout` 代数——用一套形状与步长的表达式，精确描述数据在内存里怎么排、怎么切给 Tensor Core（第 19 章）。当归约要沿着一个奇怪的维度、或要和矩阵乘的 tile 边界对齐时，这种掌控力是 Triton 的 `tl.sum(axis=...)` 给不了的。

```cutedsl-norun
# CuTe 的思维：先用 Layout 描述“数据怎么摆”，再在其上做 tile 化的归约
# （示意，非完整程序；完整用法见第 15 章）
# 一行的 score 被切成若干 tile，每个 tile 是寄存器里的一小片 Tensor
tile = make_tensor(smem_ptr, make_layout(shape, stride))
row_max = reduce(tile, op=cute.max, axis=...)   # 沿指定维度归约
# 关键价值：这个 tile 可以“原地”流向下一个 gemm，无需回全局内存
```

<div class="keypoint">
<strong>四种技术，一条选择轴</strong>
从 Triton 到 CUDA 再到 CuTe，是一条<strong>"抽象 ↔ 掌控"</strong>的轴：Triton 用一句 <code>tl.sum</code> 换来开发飞快，适合快速写出正确高效的单算子；CUDA/SYCL 让你手握每一次折叠和每一块 shared memory，适合榨干性能或做 Triton 表达不了的调度；CuTe/CUTLASS 把掌控力推到 layout 层面，适合把 softmax 这样的算子<strong>无缝焊进</strong>矩阵乘的流水线——这正是下一章 fused attention 的做法。选哪件外衣，取决于你要的是"快点写完"还是"抠到极致"。
</div>

## 小结与下一步

你用四种技术写了同一个 softmax，应该已经内化了本书反复敲打的那句话：**它们是同一个编程模型的不同外衣**。算法自始至终是那条五步流水线——读入、求 max、减 max 取 exp、求 sum、归一化，核心是**每行两次归约 + 减最大值防溢出**。变的只是那两次归约谁来写：CUDA/SYCL 让你亲手用 shared memory 折叠（线程级），Triton 一句 `tl.max`/`tl.sum` 交给编译器（块级），CuTe 则让你连数据的 layout 都拿捏在手。

但 softmax 从来不是终点。它最重要的舞台，是**注意力机制**：`softmax(Q·Kᵀ / √d)·V`。下一章是全书的收官之作——从零实现一个 **fused attention 前向**。你会看到 softmax 这道"两次归约"的题，如何和 tiling（第 11 章）、归约（第 18 章）、shared memory（第 7 章）、流水线（第 19 章）全部拧到一起，逼出那个让整个领域为之一振的技巧：**online softmax（在线 softmax）**——不物化整个打分矩阵，边流式扫描边更新 running max 和 running sum。这是 softmax 的"boss 关"。

---

**这一章的要点**

- **按行 softmax = 每行两次归约**：先 max（归约①），再 sum(exp)（归约②），最后每个元素除以 sum——是第 18 章归约原语的直接应用。
- **数值稳定技巧**：先减去每行最大值再取 exp，利用分子分母同乘 $e^{-\max}$ 结果不变的恒等式，**防止 exp 溢出**，结果分毫不差。
- **CUDA/SYCL 是线程级归约**：一个 block / work-group 负责一行，线程各扫一段写进 shared memory / local_accessor，再用折叠树收成一个值；两份代码骨架完全重合，只是词汇不同。
- **Triton 是块级归约**：一个 program 处理一整行，`tl.max` / `tl.sum` 一句话完成归约，折叠细节交给编译器；尾部靠 `mask` + `other=-inf` 零污染处理。
- **CuTe/CUTLASS** 面向的是 layout 掌控与算子融合——单跑 softmax 用不上它，但要把 softmax 焊进注意力这样的大算子时，它无可替代。
- 四种技术构成一条 **"抽象 ↔ 掌控"** 的选择轴，跑出来的是同一个算法。下一章用它们的组合拳实现 **fused attention**，主角是 **online softmax**。
