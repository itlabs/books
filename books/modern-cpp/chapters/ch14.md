# 第 14 章 STL 容器与迭代器

## 这一章想让你带走什么

<div class="goal-box">
<p>你天天用 Java 的 <code>ArrayList</code> / <code>HashMap</code>、Python 的 <code>list</code> / <code>dict</code>。C++ 标准库（STL）提供对应的容器，但有两个你必须重新适应的点：容器是<strong>值类型</strong>（不是引用），以及一切遍历都建立在<strong>迭代器</strong>这个统一抽象之上。</p>
<p>读完这一章，你会知道常用容器怎么对应你熟悉的数据结构、该在什么场景选哪个，以及迭代器这套"广义指针"如何把容器和下一章的算法连接起来。</p>
</div>

## 容器速查：对应你已经会的东西

先给一张对照表，让你快速把 C++ 容器映射到已有的知识：

| C++ 容器 | 对应 Java | 对应 Python | 底层结构 |
|---|---|---|---|
| `std::vector<T>` | `ArrayList<T>` | `list` | 动态数组 |
| `std::array<T, N>` | 定长数组 | — | 定长数组（栈上） |
| `std::map<K,V>` | `TreeMap<K,V>` | — | 红黑树（有序） |
| `std::unordered_map<K,V>` | `HashMap<K,V>` | `dict` | 哈希表 |
| `std::set<T>` | `TreeSet<T>` | — | 红黑树（有序） |
| `std::unordered_set<T>` | `HashSet<T>` | `set` | 哈希表 |
| `std::deque<T>` | `ArrayDeque<T>` | `collections.deque` | 分段数组 |
| `std::list<T>` | `LinkedList<T>` | — | 双向链表 |

<div class="keypoint">
<strong>默认就用 <code>std::vector</code></strong>
90% 的情况下 <code>vector</code> 是对的选择——即使你觉得"要频繁插入删除、也许链表更好"。原因是 <code>vector</code> 的内存连续、对 CPU 缓存极其友好，实际性能常常碾压理论上更优的链表。<strong>先用 <code>vector</code>，只有在测量证明它是瓶颈时才换别的。</strong>这条经验和你在 Java/Python 里"默认 ArrayList / list"是一致的，但在 C++ 里因为缓存效应更加突出。
</div>

## 关键差异：容器是值，不是引用

这是从托管语言过来最需要重新适应的一点，也是第 3 章值语义的直接体现。在 Java / Python 里，容器变量是引用；在 C++ 里，**容器变量就是容器本身**，赋值和传参默认**拷贝整个容器**：

```cpp
#include <iostream>
#include <vector>

int main() {
    std::vector<int> a{1, 2, 3};
    std::vector<int> b = a;      // 拷贝整个容器！b 和 a 完全独立

    b[0] = 99;                   // 只改 b
    std::cout << "a[0] = " << a[0] << '\n';   // 1（没被影响）
    std::cout << "b[0] = " << b[0] << '\n';   // 99
}
```

<div class="compare">
<div class="known">
<div class="compare-head">Java / Python：引用</div>

```java
var a = new ArrayList<>(List.of(1,2,3));
var b = a;        // b、a 指向同一个 list
b.set(0, 99);
// a.get(0) 也变成 99
```

</div>
<div class="cpp-side">
<div class="compare-head">C++：值（深拷贝）</div>

```cpp-norun
std::vector<int> a{1,2,3};
auto b = a;       // b 是 a 的独立副本
b[0] = 99;
// a[0] 仍是 1
```

</div>
</div>

<div class="warn">
<strong>牢记：传容器给函数默认会拷贝</strong>
<code>void f(std::vector&lt;int&gt; v)</code> 每次调用都<strong>拷贝整个容器</strong>——这是第 3 章讲过的性能陷阱。只读就用 <code>const std::vector&lt;int&gt;&amp;</code>，要改就用 <code>std::vector&lt;int&gt;&amp;</code>，要接管就按值传再 <code>std::move</code>。这条在容器上尤其重要，因为容器往往很大。
</div>

## vector：最常用的容器

`std::vector` 是动态数组，日常操作和 Java `ArrayList` / Python `list` 类似：

```cpp
#include <iostream>
#include <vector>

int main() {
    std::vector<int> v;
    v.push_back(10);           // 尾部添加
    v.push_back(20);
    v.emplace_back(30);        // 原地构造（比 push_back 少一次移动，见下）

    std::cout << "大小: " << v.size() << '\n';        // 3
    std::cout << "首个: " << v.front() << '\n';        // 10
    std::cout << "末个: " << v.back() << '\n';         // 30
    std::cout << "下标2: " << v[2] << '\n';            // 30

    for (int x : v) std::cout << x << ' ';             // 范围 for
    std::cout << '\n';

    v.pop_back();              // 删末尾
    std::cout << "删末尾后大小: " << v.size() << '\n';  // 2
}
```

<div class="note">
<strong><code>emplace_back</code> vs <code>push_back</code></strong>
<code>push_back(x)</code> 把已有对象拷贝/移动进容器；<code>emplace_back(args...)</code> 直接在容器内部<strong>用参数原地构造</strong>对象，省掉一次临时对象和移动。对构造成本高的类型，<code>emplace_back</code> 更高效。习惯上：要放的东西现成就 <code>push_back</code>，要现场构造就 <code>emplace_back</code>。
</div>

关于 vector 有个第 4 章提过的性能要点值得重申：如果你预先知道大概要放多少元素，用 **`reserve`** 一次性备好容量，能避免反复扩容搬移：

```cpp
#include <iostream>
#include <vector>

int main() {
    std::vector<int> v;
    v.reserve(1000);           // 预留容量，避免过程中反复重新分配
    for (int i = 0; i < 1000; ++i) v.push_back(i);

    std::cout << "size=" << v.size() << " capacity>=" << v.capacity() << '\n';
}
```

## map 与 unordered_map：键值对

需要键值映射时，两个主力是 `std::map`（有序，红黑树）和 `std::unordered_map`（哈希表，平均更快但无序）：

```cpp
#include <iostream>
#include <map>
#include <string>

int main() {
    std::map<std::string, int> ages;
    ages["Alice"] = 30;                    // 插入/赋值
    ages["Bob"] = 25;
    ages.emplace("Carol", 28);

    // 查找：用 find，检查是否命中
    if (auto it = ages.find("Alice"); it != ages.end()) {
        std::cout << "Alice: " << it->second << '\n';   // it->first 是键，->second 是值
    }

    // map 有序：遍历自动按键排序（Alice < Bob < Carol）
    for (const auto& [name, age] : ages) {              // 结构化绑定拆开键值
        std::cout << name << " = " << age << '\n';
    }
}
```

预期输出：

```
Alice: 30
Alice = 30
Bob = 25
Carol = 28
```

<div class="keypoint">
<strong>map vs unordered_map，怎么选？</strong>
<ul>
<li><strong>要有序遍历、或按范围查询</strong> → <code>std::map</code>（红黑树，操作 O(log n)，遍历按键有序）。</li>
<li><strong>只要快速增删查、不在乎顺序</strong> → <code>std::unordered_map</code>（哈希表，平均 O(1)）——它才是 Java <code>HashMap</code> / Python <code>dict</code> 的真正对应物。</li>
</ul>
两者接口几乎一样，换起来只改类型名。注意 <code>map[key]</code> 若键不存在会<strong>插入一个默认值</strong>（和 Python <code>dict</code> 的 <code>[]</code> 抛异常不同），只想查不想插就用 <code>find</code>。
</div>

两段代码里出现了两个现代语法糖，值得点名：

- **结构化绑定** `auto [name, age] = ...`：一次把键值对（或结构体、tuple）拆成多个具名变量，类似 Python 的元组解包。
- **带初始化的 if** `if (auto it = ...; it != end())`：把变量的作用域限制在 if 内，很适合"查找 + 判断命中"的模式。

## 迭代器：容器与算法之间的通用语言

现在讲 STL 的灵魂——**迭代器（iterator）**。它是一个"广义的指针"：指向容器中的某个元素，能解引用（`*it` 取值）、能前进（`++it` 到下一个）。每个容器都提供 `begin()`（指向首元素）和 `end()`（指向"最后一个元素之后"的位置）。

<div class="caption">图 14-1　迭代器：begin() 指向首元素，end() 指向"尾后"（不是最后一个元素，而是它之后）。区间是左闭右开 [begin, end)</div>

```cpp
#include <iostream>
#include <vector>

int main() {
    std::vector<int> v{10, 20, 30, 40};

    // 显式用迭代器遍历（范围 for 其实就是这个的语法糖）
    for (auto it = v.begin(); it != v.end(); ++it) {
        std::cout << *it << ' ';        // *it 解引用取元素值
    }
    std::cout << '\n';

    // 迭代器像指针一样能做算术（vector 的迭代器支持随机访问）
    auto it = v.begin();
    std::cout << "第3个元素: " << *(it + 2) << '\n';   // 30
}
```

<div class="keypoint">
<strong>为什么迭代器如此重要</strong>
迭代器是容器和<strong>算法</strong>之间的<strong>统一接口</strong>。<code>std::sort</code>、<code>std::find</code> 这些算法不关心你传的是 vector、array 还是别的什么——它们只要一对迭代器 <code>[begin, end)</code>，就能干活。这种"算法只认迭代器、不认具体容器"的解耦，是 STL 设计的精髓，也是下一章的主题。区间是<strong>左闭右开</strong> <code>[begin, end)</code>：<code>end()</code> 指向最后一个元素<em>之后</em>，不可解引用，只作"到此为止"的哨兵。
</div>

## 动手：综合使用容器

下面这段程序用一个 `unordered_map` 统计词频，综合了本章的容器操作、结构化绑定、带初始化的 if。运行前先想想输出：

```cpp
#include <iostream>
#include <vector>
#include <unordered_map>
#include <string>

int main() {
    std::vector<std::string> words{"apple", "banana", "apple", "cherry", "banana", "apple"};

    // 统计词频
    std::unordered_map<std::string, int> freq;
    for (const std::string& w : words) {
        ++freq[w];        // 键不存在时先默认插入 0，再 ++
    }

    // 查询某个词
    if (auto it = freq.find("apple"); it != freq.end()) {
        std::cout << "apple 出现了 " << it->second << " 次\n";
    }

    std::cout << "共有 " << freq.size() << " 个不同的词\n";
    return 0;
}
```

预期输出（`unordered_map` 无序，只有前两行顺序固定）：

```
apple 出现了 3 次
共有 3 个不同的词
```

`++freq[w]` 是个惯用法：`freq[w]` 在键不存在时插入一个默认值 `0`，然后 `++` 加一——统计词频一行搞定。这段代码里，容器的值语义、哈希表查找、结构化访问全用上了。

有了容器和迭代器这个基础，下一章我们就能引入 STL 的另一半——**算法**：`sort`、`find`、`transform` 这些"只认迭代器"的通用操作，以及 C++20 让它们脱胎换骨的 **ranges**。

---

**这一章的要点**

- STL 容器对应你熟悉的结构：`vector`≈`ArrayList`/`list`，`unordered_map`≈`HashMap`/`dict`，`map`/`set` 是**有序**的红黑树版本。
- **默认用 `std::vector`**——内存连续、缓存友好，实际性能常胜过理论更优的结构。
- 容器是**值类型**：赋值/传参默认**拷贝整个容器**；只读传 `const T&`，别无意中拷贝大容器。
- `map`（有序，O(log n)）vs `unordered_map`（哈希，平均 O(1)）；`map[key]` 缺键会**插入默认值**，只查用 `find`。
- **迭代器**是"广义指针"：`begin()` 指首元素，`end()` 指**尾后**，区间**左闭右开** `[begin, end)`；`*it` 取值，`++it` 前进。
- 迭代器是容器与**算法**之间的统一接口——这是下一章 STL 算法与 ranges 的基础。
- 善用现代语法：**结构化绑定** `auto [k, v]`、**带初始化的 if**。
