# 第 16 章 lambda、函数对象与可调用体

## lambda 就是就地定义的函数

最简单的用法你已经见过——一个匿名的、就地写出来的函数：

```cpp
#include <iostream>

int main() {
    auto square = [](int x) { return x * x; };   // 把 lambda 存进变量
    std::cout << square(5) << '\n';               // 25

    // 也可以直接调用
    std::cout << [](int a, int b){ return a + b; }(3, 4) << '\n';   // 7
}
```

lambda 的语法结构：

```cpp-norun
[capture](params) -> ReturnType { body }
//  ↑        ↑          ↑           ↑
// 捕获列表  参数    返回类型(可省)  函数体
```

返回类型几乎总能省略（编译器自动推导），参数列表在无参时也能省。真正让 C++ lambda 与众不同的，是那个方括号 `[]`——**捕获列表**。

## 捕获：lambda 如何访问外部变量

Java / Python 的 lambda 能直接用外层变量（闭包自动捕获）。C++ 要求你**显式声明**要捕获哪些外部变量、以及**怎么捕获**——按值还是按引用。这个区分直接对应本书前半反复强调的值语义：

```cpp
#include <iostream>

int main() {
    int factor = 10;

    auto byValue = [factor](int x) { return x * factor; };   // 按值捕获：拷贝 factor
    auto byRef   = [&factor](int x) { return x * factor; };   // 按引用捕获：引用 factor

    std::cout << byValue(5) << '\n';    // 50
    std::cout << byRef(5) << '\n';      // 50

    factor = 100;                        // 改变外部的 factor
    std::cout << byValue(5) << '\n';    // 50 —— 按值捕获的是当时的拷贝，不受影响
    std::cout << byRef(5) << '\n';      // 500 —— 按引用捕获看到的是最新值
}
```

<div class="keypoint">
<strong>按值 vs 按引用捕获</strong>
<ul>
<li><code>[factor]</code> <strong>按值捕获</strong>：在 lambda 创建时把 <code>factor</code> <strong>拷贝</strong>进去。之后外部怎么变，lambda 里的副本不变。</li>
<li><code>[&factor]</code> <strong>按引用捕获</strong>：lambda 里存的是外部变量的<strong>引用</strong>，始终看到最新值——但也因此绑上了那个变量的生命周期。</li>
</ul>
还有两个"捕获全部"的简写：<code>[=]</code> 按值捕获用到的所有外部变量，<code>[&]</code> 按引用捕获所有。但显式列出（<code>[factor]</code>）通常更清晰、更安全。
</div>

## 捕获的生命周期陷阱

按引用捕获很方便，但它和第 4 章的悬垂引用是同一类风险:**如果 lambda 活得比它按引用捕获的变量久,那个引用就悬垂了。** 这是 C++ lambda 最危险的坑,Java/Python 程序员尤其容易中招——那些语言里对象有 GC 兜底,不会突然消失。

```cpp-norun
#include <functional>

std::function<int()> makeCounter() {
    int count = 0;
    return [&count]() { return ++count; };   // ❌ 按引用捕获局部变量
}   // count 在这里销毁！返回的 lambda 里的引用悬垂

int main() {
    auto counter = makeCounter();
    counter();   // 未定义行为：访问已销毁的 count
}
```

`count` 是 `makeCounter` 的局部变量，函数返回时它就销毁了，但返回的 lambda 还引用着它——悬垂。正确做法是**按值捕获**，让 lambda 拥有自己的副本：

```cpp
#include <iostream>
#include <functional>

std::function<int()> makeCounter() {
    int count = 0;
    return [count]() mutable { return ++count; };   // ✅ 按值捕获，lambda 自带副本
}

int main() {
    auto counter = makeCounter();
    std::cout << counter() << '\n';   // 1
    std::cout << counter() << '\n';   // 2
    std::cout << counter() << '\n';   // 3
}
```

<div class="warn">
<strong>经验法则：拿不准就按值捕获</strong>
如果 lambda 会被<strong>存起来、传出去、或异步执行</strong>（生命周期可能超过当前作用域），就<strong>按值捕获</strong>，让它自带数据副本。只有当你确定 lambda 用完即弃、且就在当前作用域内同步执行时（比如直接传给 <code>std::sort</code>、<code>ranges</code> 管道），按引用捕获才安全且高效。前两章那些传给算法的 lambda 就属于后者。
</div>

注意上面那个 `mutable`：按值捕获的变量在 lambda 内**默认是只读的**（lambda 的调用运算符默认是 `const` 的）。想在 lambda 里修改捕获的副本，要加 `mutable`。

## 揭秘：lambda 其实是一个对象

现在讲最关键的一点，它能解释上面所有现象。**lambda 不是"函数指针"，而是编译器自动生成的一个匿名类的对象**——一个"函数对象"（functor，即重载了 `operator()` 的类）。

<div class="keypoint">
<strong>lambda ≈ 一个带 <code>operator()</code> 的匿名类实例</strong>
当你写 <code>[factor](int x){ return x * factor; }</code>，编译器大致生成这样一个类：
<pre>class 匿名 {
    int factor;                    // 捕获的变量 → 成为成员
public:
    int operator()(int x) const {  // 函数体 → 成为调用运算符
        return x * factor;
    }
};</pre>
所以：<strong>捕获列表 = 这个类的成员</strong>（按值捕获→值成员，按引用捕获→引用成员），<strong>lambda 体 = <code>operator()</code></strong>。"调用 lambda"就是调用这个对象的 <code>operator()</code>。这解释了一切——为什么捕获有值/引用之分（成员的存储方式）、为什么默认只读（<code>operator()</code> 默认 const）、为什么零开销（就是个普通对象的成员函数调用，能被内联）。
</div>

这也说明 lambda 和第 11 章的模板一样是**零开销抽象**:把 lambda 传给 `std::sort`,编译器知道确切的类型,能把那次 `operator()` 调用完全内联进排序循环——和你手写比较代码一样快,没有任何间接调用的开销。

## std::function：统一存放"任何可调用体"

每个 lambda 都是一个**独一无二的匿名类型**(哪怕两个 lambda 长得一模一样,类型也不同)。这带来一个问题:如果你想把 lambda 存进变量、放进容器、或作为函数返回类型,却写不出它的类型名怎么办?

- 如果只是存进局部变量,`auto` 就够了(编译器知道类型)。
- 但当你需要一个**能装下各种可调用体的统一类型**时——比如一个 `vector` 里放不同的回调、或函数返回类型必须写明——用 **`std::function`**:

```cpp
#include <iostream>
#include <functional>
#include <vector>

int main() {
    // std::function<返回类型(参数类型)> 能装任何"签名匹配"的可调用体
    std::vector<std::function<int(int)>> operations;

    operations.push_back([](int x){ return x + 1; });      // lambda
    operations.push_back([](int x){ return x * 2; });      // 另一个 lambda
    int base = 100;
    operations.push_back([base](int x){ return x + base; });  // 带捕获的 lambda

    int value = 5;
    for (const auto& op : operations) {
        std::cout << op(value) << ' ';    // 6 10 105
    }
    std::cout << '\n';
}
```

`std::function<int(int)>` 是一个"类型擦除"的容器,能装下任何"接受 int、返回 int"的可调用体——不管它是 lambda、函数指针还是函数对象。三个 lambda 类型各不相同,但都能塞进同一个 `std::function<int(int)>`。

<div class="note">
<strong>std::function 有开销，别滥用</strong>
<code>std::function</code> 为了能装"任何东西"，内部用了类型擦除（常伴随一次堆分配和一次间接调用），比直接用 lambda 或模板慢。<br>
经验：<strong>能用 <code>auto</code> 就用 <code>auto</code>，能用模板参数就用模板参数</strong>（如算法那样接受任意可调用类型，零开销）；只有在<strong>确实需要把不同类型的可调用体统一存储/传递</strong>时（回调列表、返回类型必须写明），才用 <code>std::function</code>。它是灵活性和性能的权衡。
</div>

## 泛型 lambda：参数也能是模板

C++14 起，lambda 的参数可以用 `auto`，让同一个 lambda 适配多种类型——本质上它的 `operator()` 变成了模板：

```cpp
#include <iostream>
#include <string>

int main() {
    auto print = [](const auto& x) { std::cout << x << '\n'; };   // 泛型 lambda

    print(42);              // int
    print(3.14);            // double
    print(std::string("hello"));   // string
    print("world");         // const char*
}
```

一个 `print` lambda 服务了所有能打印的类型。这在需要给算法传"对任意类型都适用"的操作时很方便。

## 动手：把 lambda 的各个特性串起来

下面这段程序综合了捕获、泛型 lambda、`std::function`，做一个简单的"可配置的数据处理器"。运行前先预测输出：

```cpp
#include <iostream>
#include <vector>
#include <functional>
#include <algorithm>

int main() {
    std::vector<int> data{1, 2, 3, 4, 5};

    // 按值捕获阈值：即使 threshold 之后改变，lambda 用的仍是捕获时的值
    int threshold = 3;
    auto isBig = [threshold](int x) { return x > threshold; };

    int count = std::ranges::count_if(data, isBig);
    std::cout << "大于 " << threshold << " 的有 " << count << " 个\n";   // 2

    threshold = 100;                          // 改变外部变量
    count = std::ranges::count_if(data, isBig);
    std::cout << "改 threshold 后仍是 " << count << " 个\n";   // 2（按值捕获，不受影响）

    // 用 std::function 存一组不同的变换，依次应用到一个值
    std::vector<std::function<int(int)>> pipeline{
        [](int x){ return x + 10; },
        [](int x){ return x * 2; },
        [](int x){ return x - 5; },
    };
    int v = 1;
    for (const auto& step : pipeline) v = step(v);   // ((1+10)*2)-5 = 17
    std::cout << "流水线结果: " << v << '\n';
}
```

预期输出：

```
大于 3 的有 2 个
改 threshold 后仍是 2 个
流水线结果: 17
```

第二行是关键:`isBig` 按值捕获了 `threshold`,所以后来把外部 `threshold` 改成 100,不影响 lambda 里那个"拍照存档"的副本 `3`——这正是"lambda 是个对象、捕获是它的成员"最直观的体现。

**这一章的要点**

- **lambda** 是就地定义的匿名函数：`[捕获](参数){ 函数体 }`，返回类型通常可省。
- **捕获**决定如何访问外部变量：`[x]` 按值（拷贝，创建时定格），`[&x]` 按引用（看最新值，但绑定生命周期）；`[=]`/`[&]` 是全捕获简写。
- 按引用捕获有**悬垂风险**：lambda 若活得比被捕获变量久，引用就悬垂——存起来/传出去/异步执行时应**按值捕获**。
- lambda 本质是**编译器生成的匿名函数对象**：捕获=成员，函数体=`operator()`。这解释了值/引用捕获、默认只读（需 `mutable` 才可改）、以及零开销（可内联）。
- 每个 lambda 类型唯一；用 `auto` 存局部变量，用 **`std::function<R(Args)>`** 统一存放不同的可调用体（有类型擦除开销，别滥用）。
- **泛型 lambda**（参数用 `auto`）让一个 lambda 适配多种类型。
