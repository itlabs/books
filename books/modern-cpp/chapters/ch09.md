# 第 9 章 const、常量性与不可变

## 这一章想让你带走什么

<div class="goal-box">
<p>在 Java 里你有 <code>final</code>，在 Python 里你几乎没有真正的不可变约束。C++ 的 <code>const</code> 比这两者都强大得多——它不只是"这个变量不能改"，而是一套贯穿类型系统的<strong>常量性（const-correctness）</strong>机制，编译器会全程帮你把关。</p>
<p>读完这一章，你会看懂 C++ 代码里满屏的 <code>const</code> 到底在说什么，知道 <code>const</code> 放在不同位置的不同含义，以及为什么"该加 const 的地方都加上"是专业 C++ 代码的标志。</p>
</div>

## const 不只是"常量"

先纠正一个来自其他语言的直觉。`const` 的字面意思是"常量"，但它在 C++ 里真正的含义是**只读的承诺**：通过这个名字，你不能修改所指的对象。

```cpp
#include <iostream>

int main() {
    const int max = 100;
    // max = 200;         // ❌ 编译错误：const 对象不能修改
    std::cout << max << '\n';

    int x = 10;
    const int& r = x;     // r 是 x 的只读别名
    // r = 20;            // ❌ 不能通过 r 改
    x = 20;               // ✅ 但 x 自己还是能改
    std::cout << r << '\n';   // 20（r 看到的是 x 的当前值）
}
```

注意最后那组：`const int& r = x` 是说"**通过 r** 不能改"，并不是说 x 变成了常量。`const` 约束的是**访问路径**，不一定是对象本身。这个区分是理解 const 的关键。

<div class="compare">
<div class="known">
<div class="compare-head">Java：final</div>

```java
final int max = 100;
// max = 200; 报错

// final 只管"变量不能重新赋值"
// 对象内部照样能改：
final List<Integer> list = ...;
list.add(1);  // ✅ 允许
```

</div>
<div class="cpp-side">
<div class="compare-head">C++：const 更深入</div>

```cpp-norun
const int max = 100;
// max = 200; 报错

// const 能一路传递到成员：
const std::vector<int> v = {1,2};
// v.push_back(1); ❌ 不允许
// const 对象只能调 const 方法
```

</div>
</div>

关键差异:Java 的 `final` 只锁住"变量不能改指向别的对象",对象内部随便改;C++ 的 `const` 会**一路深入**——一个 `const` 容器,你连往里加元素都不行。这种"深"的常量性是 C++ 类型安全的重要支柱。

## const 与指针：位置决定含义

`const` 和指针搭配时，位置不同、含义不同，这是初学者最容易绕晕的地方。只需记住一个口诀：**从右往左读，`const` 修饰它左边的东西**（左边没有时才修饰右边）。

```cpp
#include <iostream>

int main() {
    int a = 1, b = 2;

    const int* p1 = &a;        // 指向 const int：不能改 *p1，能改 p1 指向
    p1 = &b;                   // ✅ 改指向
    // *p1 = 3;                // ❌ 不能改所指的值

    int* const p2 = &a;        // const 指针：能改 *p2，不能改指向
    *p2 = 10;                  // ✅ 改所指的值
    // p2 = &b;                // ❌ 不能改指向

    const int* const p3 = &a;  // 两者都不能改
    std::cout << a << ' ' << *p2 << '\n';   // 10 10
}
```

<div class="keypoint">
<strong>从右往左读指针声明</strong>
<ul>
<li><code>const int* p</code> → "p is a pointer to <strong>const int</strong>"：所指的值是只读的，指针本身可变。</li>
<li><code>int* const p</code> → "p is a <strong>const pointer</strong> to int"：指针是只读的（不能改指向），所指的值可变。</li>
<li><code>const int* const p</code> → 两者都只读。</li>
</ul>
日常最常见的是第一种（<code>const int*</code>），用来表达"我借你这个对象看看，保证不改它"。
</div>

## const 引用：最常见的只读传参方式

回忆第 3 章的性能陷阱：按值传参会拷贝整个对象。解决方案就是 **`const T&`**——按引用传递（不拷贝），同时用 `const` 承诺"我不会改它"。这是 C++ 里传递只读大对象的**标准姿势**：

```cpp
#include <iostream>
#include <string>
#include <vector>

// 只读、不拷贝：既高效又安全
long sum(const std::vector<int>& v) {   // const 引用：不拷贝这个 vector
    long total = 0;
    for (int x : v) total += x;
    // v.push_back(0);   // ❌ const 保证了函数不会改动实参
    return total;
}

int main() {
    std::vector<int> data{1, 2, 3, 4, 5};
    std::cout << sum(data) << '\n';      // 15，且没有拷贝 data
}
```

`const std::vector<int>&` 兼得两个好处:**没有拷贝开销**(引用),且**调用者放心**(const 保证函数不会偷改自己的数据)。看到一个函数参数是 `const T&`,你立刻知道:这是个只读的输入,而且传大对象也不亏。

<div class="note">
<strong>传参的经验法则</strong>
· 小对象（<code>int</code>、<code>double</code>、指针等）→ 直接<strong>按值</strong>传，拷贝比引用还便宜。<br>
· 大对象、只读 → <strong><code>const T&</code></strong>。<br>
· 需要修改实参 → <code>T&</code>（第 3 章）。<br>
· 需要接管所有权 → 按值传 <code>T</code> 或传 <code>T&&</code>（配合移动，第 7 章）。
</div>

## const 成员函数：给方法贴上"只读"标签

类的成员函数也能标记为 `const`,写在参数列表后面。它的含义是:**这个方法不会修改对象的状态**。

```cpp
#include <iostream>
#include <string>

class Account {
    std::string owner_;
    long balance_;
public:
    Account(std::string owner, long balance)
        : owner_(std::move(owner)), balance_(balance) {}

    // const 成员函数：只读，承诺不改任何成员
    long balance() const { return balance_; }
    const std::string& owner() const { return owner_; }

    // 非 const 成员函数：会修改状态
    void deposit(long amount) { balance_ += amount; }
};

int main() {
    Account a("Alice", 100);
    a.deposit(50);
    std::cout << a.owner() << ": " << a.balance() << '\n';   // Alice: 150

    const Account frozen("Bob", 999);   // const 对象
    std::cout << frozen.balance() << '\n';   // ✅ 能调 const 方法
    // frozen.deposit(1);               // ❌ const 对象不能调非 const 方法
}
```

这里藏着 const 机制最重要的一环：

<div class="keypoint">
<strong>const 对象只能调用 const 成员函数</strong>
一个 <code>const Account</code>（或通过 <code>const Account&</code> 访问的对象），编译器<strong>只允许你调用它的 <code>const</code> 方法</strong>——因为非 const 方法可能改动状态，违背了 const 承诺。<br>
这就是为什么"该标 const 的方法一定要标 const"：如果你忘了给 <code>balance()</code> 加 const，那么任何持有 <code>const Account&</code> 的代码（比如上面 <code>sum</code> 那种只读函数）就<strong>连查余额都做不到</strong>。const 正确性是会传染的——一处遗漏，处处受限。
</div>

## const 正确性：一种会传染的纪律

"const 正确性（const-correctness）"指的是:在每个"本就不该修改"的地方都老老实实加上 `const`——只读参数用 `const&`,不改状态的方法标 `const`,不变的局部变量声明 `const`。

它的价值有两层：

1. **编译期防错**:把"我不打算改它"写进类型,一旦不小心写了修改代码,编译器立刻报错——bug 在编译期就被挡住,而不是运行时。
2. **自文档化**:`const` 是给读代码的人看的契约。看到 `const T&` 参数,不用读函数体就知道它不会改你的数据。

```cpp
#include <iostream>

// const 让编译器成为你的校对员
int firstElement(const int* arr, int n) {
    if (n > 0) {
        // arr[0] = 999;    // 万一手滑写了修改，编译器立刻拦下
        return arr[0];
    }
    return -1;
}

int main() {
    int data[] = {10, 20, 30};
    std::cout << firstElement(data, 3) << '\n';   // 10
}
```

<div class="note">
<strong>为什么 Java / Python 程序员容易忽略它</strong>
Java 只有浅层的 <code>final</code>，Python 基本靠约定（下划线前缀、文档），你没有养成"处处标注只读"的习惯。但在 C++ 里，const 正确性是<strong>专业代码的基本要求</strong>：标准库全程贯彻 const，你的代码若不配合，就会在与标准库交互时处处别扭。建议从现在起养成习惯：<strong>默认加 const，只在确实需要修改时才去掉。</strong>
</div>

## const 与 constexpr 的一句话区别

你会在现代 C++ 里看到另一个相关的词 `constexpr`（第 13 章详讲），这里先厘清它和 `const` 的区别，免得混淆：

<div class="keypoint">
<strong>const = 只读，constexpr = 编译期就能算出来</strong>
<ul>
<li><code>const</code>：运行期的只读承诺。值可以来自运行时（<code>const int n = readFromUser();</code> 合法）。</li>
<li><code>constexpr</code>：更强——要求值在<strong>编译期</strong>就能确定。<code>constexpr int n = 3 * 4;</code> 会在编译时算好，可用于需要编译期常量的场景（如数组大小、模板参数）。</li>
</ul>
简单记：所有 <code>constexpr</code> 变量都隐含 <code>const</code>，但反之不然。日常先用好 <code>const</code>，编译期计算的话题留给第 13 章。
</div>

## 动手：观察 const 如何在编译期拦截错误

下面这段程序演示 const 的三种典型用法。它能正常编译运行；但每个注释掉的 `// ❌` 行，只要放开任意一行，编译就会失败。动手试试放开它们，看看编译器给出的错误信息：

```cpp
#include <iostream>
#include <string>
#include <vector>

class Book {
    std::string title_;
    int pages_;
public:
    Book(std::string t, int p) : title_(std::move(t)), pages_(p) {}
    const std::string& title() const { return title_; }   // 只读方法
    int pages() const { return pages_; }                  // 只读方法
    void addPages(int n) { pages_ += n; }                 // 修改方法
};

// 只读遍历：const 引用传参 + 只调 const 方法
void printAll(const std::vector<Book>& books) {
    for (const Book& b : books) {                 // const 引用，不拷贝
        std::cout << b.title() << " (" << b.pages() << " 页)\n";
        // b.addPages(1);   // ❌ 放开会报错：const 对象不能调非 const 方法
    }
    // books.push_back(...); // ❌ 放开会报错：const 容器不能修改
}

int main() {
    std::vector<Book> shelf;
    shelf.emplace_back("C++ 之路", 300);
    shelf.emplace_back("算法导论", 1300);

    printAll(shelf);

    const Book fixed("不可变之书", 100);
    std::cout << fixed.title() << '\n';   // ✅ const 对象调 const 方法，OK
    // fixed.addPages(10);                // ❌ 放开会报错
}
```

运行输出：

```
C++ 之路 (300 页)
算法导论 (1300 页)
不可变之书
```

这段代码把本章的要点全用上了:`const&` 只读传参、`const` 成员函数、`const` 对象。而那几个 `// ❌` 注释,正是 const 在编译期替你守住的边界——它们不是运行时才崩,而是**根本编译不过**。

---

**这一章的要点**

- `const` 的本质是**只读的承诺**：约束的是"通过这个名字/路径能否修改"，不一定是对象本身。
- 比 Java 的 `final` 更**深**：`const` 会一路传递到成员，`const` 容器连加元素都不行。
- 指针与 const 搭配**从右往左读**：`const int*`（所指只读）vs `int* const`（指针只读）。
- **`const T&`** 是传递只读大对象的标准方式：无拷贝开销 + 不可修改的保证。
- **const 成员函数**承诺不改对象状态；**const 对象只能调 const 方法**——所以该标 const 的方法务必标上。
- **const 正确性**是会传染的纪律：默认加 const，编译器帮你在编译期挡住误改，同时让代码自文档化。
- `const`（只读）与 `constexpr`（编译期可算）不同；所有 `constexpr` 隐含 `const`，反之不然。
