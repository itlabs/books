# 现代 C++ 之路

> 写给资深 Java / Python 程序员的语言、内存与编译器之旅。
> 你已经会编程了——这本书要讲的，是 C++ 为什么和你熟悉的托管语言不一样，以及这些不一样背后的设计原理。

## 这本书写给谁

你已经是一名资深工程师，能熟练用 Java 或 Python 解决真实问题，理解面向对象、泛型、异常、并发这些概念。你**不需要**别人再教你什么是变量、什么是循环。

你缺的不是编程经验，而是一套新的**心智模型**：

- 在 Java / Python 里，变量是对象的**引用**，内存由 GC 打理，你几乎不用关心对象什么时候消失；
- 在 C++ 里，变量往往**就是对象本身**，对象的生命周期由你和编译器共同精确控制，没有 GC，却也几乎不用手动 `free`。

这本书就是帮你完成这次心智模型的切换。我们不会从"Hello World 是什么"讲起，而是不断把 C++ 的做法和你已经会的 Java / Python 做法对照，讲清**差异**和**差异背后的原因**。

## 这本书的特别之处

- **对照式讲解**：每个新概念都尽量先用一段 Java 或 Python 代码建立锚点，再看 C++ 怎么做、为什么这么做。
- **看得见的内存**：栈、堆、对象生命周期、移动、vtable 等抽象概念都配有图示，而不是只有文字。
- **可运行 + 看得见编译器**：几乎每段 C++ 代码块都带两个入口——
  - **🔗 在 Compiler Explorer 打开**：点开就能在线编译、运行、改代码重跑；
  - **🔬 看汇编 / 编译器处理**：讲优化、内联、模板实例化、RVO 时，直接看编译器生成的汇编。
  - 每段代码同时附**本地编译命令**，你也可以在自己机器上 `g++`/`clang++` 跑真实结果。
- **示例全部可编译可执行**：书中的完整程序都能独立编译运行，不是伪代码。

## 怎么用这本书

1. **按顺序读第二部分。** 第 3—7 章是全书地基：值语义、生命周期、RAII、移动、所有权。这套模型立住了，后面都顺。
2. **点开代码块动手改。** 遇到"为什么"，最快的验证方式就是在 Compiler Explorer 里改一行重跑，或点"看汇编"看编译器到底做了什么。
3. **用附录 B 查术语。** 遇到陌生词，先看它对应你熟悉的 Java / Python 里的什么概念。
4. **默认现代 C++（C++20 / C++23）。** 本书讲的是今天该怎么写 C++，不是 1998 年的写法。

准备好切换心智模型了吗？从[第 1 章](chapters/ch01.md)开始。
