上一章的猜数字，是电脑出题、你来猜。这一章反过来——**你在故事里做主，程序根据你的选择往下讲**。这靠的还是老朋友：`input()` 收集选择，`if / elif / else` 分岔路，`while` 让你能重玩。真正的新本事，是学会用**函数**把一大团剧情理得井井有条。

## 先做最小的一步：一个岔路口

任何冒险故事，最核心的动作就是"给你几个选项，你挑一个"。我们先把这一步做出来：

```python
# one_choice.py
print("你走到一个黑漆漆的山洞前，隐约听到里面有声音……")
print("1) 勇敢地走进去")
print("2) 绕道从旁边小路走")

choice = input("你的选择（1/2）：")

if choice == "1":
    print("你深吸一口气，走进了山洞。")
elif choice == "2":
    print("你绕开山洞，沿着小路走了。")
else:
    print("看不懂你的选择，你在原地发呆了半天。")
```

注意 `choice == "1"` 里的 `"1"` **带引号**——因为 `input()` 拿到的是文字，玩家打的 `1` 是文字 `"1"`，不是数字。这里我们不比大小，只判断"是不是这个选项"，所以不用转成数字，直接按文字比就行。

`else` 那一支是在兜底：万一玩家手滑打了别的字符，程序也不会崩，而是好好回一句话。**给玩家的输入留一条"退路"，是好游戏的基本礼貌。**

## 剧情越长越乱：请函数来帮忙

一个岔路口好写，可冒险故事有好多岔路口，每个岔路口下面又接着新的岔路口……如果全堆在一起，代码会变成一团乱麻，你自己都看不清哪段接哪段。

这时候，上一部分学的**函数（function）**就该登场了。我们的思路是：**把每一个场景写成一个函数**。比如"洞口"是一个函数，"洞里"是一个函数，"小路"又是一个函数。玩家在洞口选了"进洞"，我们就调用"洞里"那个函数——就像翻到故事书的某一页。

这么做的好处特别大：

- 每个场景各管各的，一个函数就是一小段独立的剧情，改起来清清楚楚；
- 场景之间靠"调用"连起来，剧情怎么流转一目了然；
- 想加新场景，就多写一个函数，不会打乱原来的。

## 把一个场景写成函数

先看"洞里"这个场景怎么写成函数。**函数就是给一段代码起个名字**，以后喊这个名字，那段代码就会跑一遍。

```python
# scene_function.py

def inside():
    """洞里的场景"""
    print("洞里很黑。你摸到墙上有一支火把。")
    print("1) 点燃火把照路")
    print("2) 摸黑继续往前")
    choice = input("你的选择（1/2）：")
    if choice == "1":
        print("火光照亮了洞穴，你看见角落有一个闪光的宝箱！")
        print("【结局】你带着满满一箱金币回家，成了大英雄！🏆")
    else:
        print("摸黑走着走着，你一脚踩空，掉进了坑里……")
        print("【结局】幸好坑不深，你灰头土脸地爬回了家。")

# 喊一声函数的名字，这个场景就上演了
inside()
```

- `def inside():` 定义了一个名叫 `inside`（英语"里面"）的函数。`def` 是 define（"定义"）的缩写。
- 函数里面的代码要**缩进**（每行前面空几格），表示"这些都归 inside 管"。
- 最下面的 `inside()` 是**调用**函数——不加它，函数只是被"定义"了却从没运行。**定义是写菜谱，调用才是真正下厨。**

运行它，你会发现和之前的岔路口一模一样。只不过现在这段剧情有了名字，被打包收好了。

## 让一个场景跳到另一个场景

真正神奇的地方来了：**在一个函数里，可以调用另一个函数**。玩家在洞口选了"进洞"，我们就在洞口函数里调用洞里函数——剧情就这样从一个场景流到了下一个场景。

```python
# scene_jump.py

def inside():
    """洞里的场景"""
    print("洞里很黑。你摸到墙上有一支火把。")
    print("1) 点燃火把照路")
    print("2) 摸黑继续往前")
    choice = input("你的选择（1/2）：")
    if choice == "1":
        print("火光照亮了洞穴，你看见一个闪光的宝箱！")
        print("【结局】你抱着金币回家，成了大英雄！🏆")
    else:
        print("【结局】你在黑暗里迷了路，天亮才摸出来。")

def cave():
    """洞口的场景，整个故事从这里开始"""
    print("你走到一个黑漆漆的山洞前，隐约听到里面有声音……")
    print("1) 勇敢地走进去")
    print("2) 绕道从旁边小路走")
    choice = input("你的选择（1/2）：")
    if choice == "1":
        inside()  # 进洞，跳到洞里的场景
    else:
        print("【结局】你绕开山洞，平平安安回了家。")

# 故事从洞口开始
cave()
```

看 `cave()` 里的那行 `inside()`：玩家一旦选了"进洞"，程序立刻跳去执行 `inside` 函数，洞里的剧情就接着上演。你写故事的顺序（先写 inside 再写 cave）不重要，**重要的是最后调用了哪个函数当起点**——这里是 `cave()`。

> 小贴士：函数必须**先定义、后调用**。所以我们把所有 `def` 都写在前面，把真正启动游戏的那行 `cave()` 放在最末尾。

## 完整的冒险游戏

现在把所有场景拼成一个完整的故事。我们还加了一个 `while` 循环，让玩家通关后能选择"再玩一次"——这样一个游戏就能反复玩，每次走不同的路。

```python
# adventure.py 山洞寻宝 完整版

def treasure_room():
    """宝藏室：最深处的选择"""
    print("火光下，你看见一个宝箱和一扇写着'出口'的小门。")
    print("1) 打开宝箱")
    print("2) 直接从小门离开")
    choice = input("你的选择（1/2）：")
    if choice == "1":
        print("箱子里金币哗啦啦地流出来！")
        print("【结局】你带着满箱金币回家，成了村里的大英雄！🏆")
    else:
        print("【结局】你两手空空但平平安安地走出了山洞。")

def inside():
    """洞里：点不点火把"""
    print("洞里很黑。你摸到墙上有一支火把。")
    print("1) 点燃火把照路")
    print("2) 摸黑继续往前")
    choice = input("你的选择（1/2）：")
    if choice == "1":
        treasure_room()  # 点了火把，看清路，进入宝藏室
    else:
        print("【结局】黑暗中你撞上一头睡觉的大熊，撒腿就跑，宝藏没戏了！🐻")

def path():
    """旁边的小路"""
    print("小路尽头是一条小溪，溪边有只受伤的小狐狸。")
    print("1) 停下来帮助小狐狸")
    print("2) 不管它，继续赶路")
    choice = input("你的选择（1/2）：")
    if choice == "1":
        print("小狐狸带你找到了一处隐藏的山泉和野果。")
        print("【结局】你和小狐狸成了好朋友，满载而归。🦊")
    else:
        print("【结局】你独自走回了家，路上一直在想那只小狐狸。")

def cave():
    """洞口：整个故事的起点"""
    print("你走到一个黑漆漆的山洞前，隐约听到里面有声音……")
    print("1) 勇敢地走进去")
    print("2) 绕道从旁边小路走")
    choice = input("你的选择（1/2）：")
    if choice == "1":
        inside()
    elif choice == "2":
        path()
    else:
        print("你站在原地犹豫了太久，天黑了，只好回家。")

# ==== 游戏主循环：可以反复玩 ====
print("====== 欢迎来到《山洞寻宝》======")
again = "1"
while again == "1":
    cave()          # 玩一整局，从洞口开始
    print("--------------------")
    again = input("再来一次冒险吗？（1=好啊 / 其他=不玩了）：")

print("谢谢游玩，江湖再见！")
```

运行它玩几次，故意每次走不同的路，你会发现同一个程序能讲出好几个完全不同的故事。这就是分支剧情的魅力。

拆开看这个游戏的骨架：

- **四个场景函数**：`cave`、`inside`、`path`、`treasure_room`，各管一小段剧情，用调用彼此串起整棵故事树。
- **一个起点**：最后那段代码里的 `cave()`，是整个冒险的入口。
- **一个主循环**：`while again == "1"` 让玩家通关后能重玩；只要他还想玩，就再调一次 `cave()`。

你有没有发现——**这段完整代码，其实就是把图 8-1 那棵树"翻译"成了函数？** 先画出剧情树，再照着树写函数，是设计文字游戏最省心的办法。

> 小贴士：给函数起名要"见名知意"，`treasure_room`（宝藏室）一看就懂。你未来写大程序时，好名字能帮你省下大把读代码的时间。

## 常见错误

**1. 只定义了函数，忘了调用**

```python-norun
def cave():
    print("你走到山洞前……")

# 下面没有 cave()
```

运行后**什么都不发生**，屏幕空空。因为你只写了菜谱，没下厨。修法：在最后加一行 `cave()`。

**2. 判断选项时忘了给数字加引号**

```python-norun
choice = input("你的选择（1/2）：")
if choice == 1:      # 这里的 1 是数字，choice 却是文字
    print("进洞")
```

程序不报错，但**永远进不了这一支**——因为文字 `"1"` 永远不等于数字 `1`。修法：写成 `if choice == "1":`，给 1 加上引号。

**3. 函数里的代码没缩进**

```python-norun
def cave():
print("你走到山洞前……")   # 没缩进
```

报错：

```
IndentationError: expected an indented block after function definition
```

`IndentationError`（缩进错误）是说：`def` 这行冒号后面，函数的内容必须缩进。修法：把属于函数的每一行都往右缩进（通常按一下 Tab 或空四格）。

**4. 调用了还没定义的函数**

如果你把启动游戏的 `cave()` 写在了 `def cave():` **前面**，会报 `NameError: name 'cave' is not defined`。记住：**先定义，后调用**，启动代码放最后。

## 动手挑战

在 `adventure.py` 的基础上扩展你的冒险世界。

**★ 挑战一：加一个新结局**
在 `path()`（小路）里再加一个岔路口，写出至少一个新场景函数和新结局。让"帮助小狐狸"之后还能继续做选择。

**★★ 挑战二：给玩家起个名字**
在游戏最开始用 `input()` 问玩家叫什么名字，存进变量，然后在剧情里用上它，比如"勇敢的{name}走进了山洞"。想想怎么把这个名字传进各个场景函数里（提示：可以用函数的参数）。

**★★★ 挑战三：背包系统**
用一个列表当"背包"。玩家在某个场景捡到"钥匙"就把 `"钥匙"` 加进列表；到了另一个场景，用 `if "钥匙" in 背包:` 判断——有钥匙才能开另一扇门，通向隐藏的超级结局。这样玩家的选择就会"影响到后面"，故事更有深度！

## 本章小结

你写出了一个能讲无数个故事的分支游戏，还学会了用函数管理复杂剧情：

- 文字冒险游戏的核心，就是不断"给选项 → 收输入 → 按选择分岔"，靠 `input` 加 `if/elif/else` 实现。
- 比较 `input()` 拿到的选择时，数字选项也要**当文字比**，记得加引号：`choice == "1"`。
- 把**每个场景写成一个函数**，能让庞大的剧情变得清清楚楚、好写好改。
- 函数要**先定义、后调用**；在一个函数里调用另一个函数，就能让剧情从一个场景跳到下一个。
- 用 **`while` 循环**包住游戏，玩家就能"再玩一次"。
- 设计诀窍：**先画剧情分支树，再照着树写函数**。
- 别忘了给玩家的输入留 `else` 退路，程序才不会因为乱输入而尴尬。
