## 让程序开口问：input()

上一章我们用 `print()` 让程序"说话"。现在认识它的搭档——**`input()`**（英语"输入"），它让程序"听话"，也就是**读取用户从键盘打进来的内容**。

```python
# input_hello.py
name = input("请问你叫什么名字？")
print(f"你好，{name}！")
```

运行这个程序，屏幕会显示"请问你叫什么名字？"然后**停下来等你**。你打字输入，比如 `小明`，按回车，程序才继续往下跑。

运行结果：

```
请问你叫什么名字？小明
你好，小明！
```

拆开看这行 `name = input("请问你叫什么名字？")`：

- `input("...")` 括号里的文字叫**提示语**，会显示给用户看，告诉他该输入什么。它是可选的，但强烈建议写，不然用户看着空屏幕不知道要干嘛。
- 用户打的字，会被 `input()` "接住"，然后通过 `=` **装进左边的变量** `name`。
- 之后 `name` 盒子里就是用户输入的内容，你想怎么用都行。

这就是"和电脑对话"的核心：**`input()` 负责接话，变量负责记住，`print()` 负责回应。**

## 天大的坑：input() 收到的永远是字符串

这是本章最重要、也最容易栽跟头的一点，请画三条横线记住：

> **不管用户输入什么，`input()` 拿到的永远是字符串（str）。**

哪怕用户明明打的是数字 `13`，`input()` 收到的也是**文字 "13"**，不是**数字 13**。还记得上一章说的吗——照片上的苹果不能吃。`"13"` 就是那张照片。

不信我们用上一章的"照妖镜" `type()` 照一照：

```python
# input_type.py
age = input("你今年几岁？")
print(type(age))
```

运行结果（假设你输入 13）：

```
你今年几岁？13
<class 'str'>
```

看，是 `str`（字符串），不是 `int`。这会带来什么麻烦？看下面：

```python
# input_bug.py
age = input("你今年几岁？")
print(age + 1)    # 想算明年几岁
```

运行结果：

```
你今年几岁？13
TypeError: can only concatenate str (not "int") to str
```

报错了！因为 `age` 是字符串 `"13"`，字符串不能加数字 `1`。这个坑几乎每个初学者都会踩一次，你现在提前知道了，就能绕过去。

```python
# input_number.py
age = input("你今年几岁？")   # age 是字符串 "13"
age = int(age)                # 变身成整数 13
print(f"明年你就 {age + 1} 岁了！")
```

运行结果：

```
你今年几岁？13
明年你就 14 岁了！
```

成功！现在 `age` 是真正的数字，能参与运算了。

分两行写有点啰嗦，老手们喜欢把 `int` 直接套在 `input` 外面，一步到位：

```python
# input_number.py
age = int(input("你今年几岁？"))   # 输入、转整数，一气呵成
print(f"明年你就 {age + 1} 岁了！")
```

`int(input("..."))` 读作："先让 `input` 接住用户输入，再立刻用 `int` 把它变成整数。" 括号是从里往外算的：先执行最里面的 `input()`，它的结果交给外面的 `int()`。

如果用户输入的是小数（比如身高 1.58），就用 `float()` 代替 `int()`：

```python
height = float(input("你的身高是多少米？"))
print(f"你有 {height} 米高。")
```

> 小贴士：什么时候转、什么时候不转？只要你想拿输入去**做算术**，就得转成数字。如果只是把它当文字用（比如名字、爱好，只是打印出来），那就不用转，保持字符串就好。

## print 打印多个内容

回头再看看老朋友 `print()`，它其实比第 1 章展示的更能干。你可以在括号里放**好几样东西，用逗号 `,` 隔开**，它会一起打印，中间**自动空一格**：

```python
# print_more.py
name = "小明"
age = 13
print("姓名", name, "年龄", age)
```

运行结果：

```
姓名 小明 年龄 13
```

注意到了吗？逗号分隔的方式有个好处：**数字不用先转成字符串也能打印**，`print` 会自动处理。这和用 `+` 拼接不一样——`+` 要求两边都是字符串，逗号则来者不拒。

不喜欢那个自动空格？可以用 **`sep`**（separator"分隔符"的缩写）参数，指定用什么来隔开：

```python
print("2026", "07", "11", sep="-")
```

运行结果：

```
2026-07-11
```

还有个 **`end`** 参数，控制打印完在结尾放什么。`print` 默认打印完会换行（相当于结尾有个看不见的"回车"），如果你不想换行，可以把 `end` 改掉：

```python
print("加载中", end="")   # 打印完不换行
print("...完成！")
```

运行结果：

```
加载中...完成！
```

两个 `print` 的内容连在了一行，因为第一个 `print` 结尾没有换行。`sep` 和 `end` 现在了解个大概就行，以后需要精细排版时会用到。日常打印，我们还是最爱用 f-string。

## f-string 组合输出

其实打印多样东西，最推荐的还是上一章学的 f-string——它最清楚、最好读。逗号和 f-string 都能做同一件事，对比一下：

```python
name = "小明"
score = 95

# 方法一：逗号分隔（会有多余空格，不太好控制）
print(name, "考了", score, "分")

# 方法二：f-string（想怎么排就怎么排，推荐）
print(f"{name}考了{score}分")
```

运行结果：

```
小明 考了 95 分
小明考了95分
```

看出来了吧，f-string 让你把变量精确地嵌在句子里，一个多余空格都没有。以后组合输出，优先用 f-string。

## 完整代码一：小小采访机

把 `input()`、变量、f-string 全用上，做一个会采访人的程序。新建 `interview.py`：

```python
# interview.py

print("=== 欢迎来到小小采访机 ===")

# 依次提问，把每个回答存进对应的变量
name = input("请问你叫什么名字？")
age = int(input("你今年几岁？"))        # 年龄要算数，转成整数
hobby = input("你最喜欢做的事情是什么？")

# 用一段话把采访结果复述出来
print()   # 空一行，让输出更好看
print("=== 采访结果 ===")
print(f"这位朋友叫{name}，今年{age}岁。")
print(f"{name}最喜欢{hobby}。")
print(f"明年{name}就{age + 1}岁啦，祝你天天开心！")
```

运行结果（下划线部分是你输入的）：

```
=== 欢迎来到小小采访机 ===
请问你叫什么名字？小明
你今年几岁？13
你最喜欢做的事情是什么？打篮球

=== 采访结果 ===
这位朋友叫小明，今年13岁。
小明最喜欢打篮球。
明年小明就14岁啦，祝你天天开心！
```

看，程序真的和你"聊"起来了！它问、你答、它再根据你的答案回应。注意 `age` 用了 `int()` 转换，因为最后一行要算 `age + 1`；而 `name` 和 `hobby` 只是拿来打印，就保持字符串不动。

那个单独的 `print()`（括号里什么都没有）会打印一个空行，让输出看起来更整齐——这是个常用的小技巧。

## 完整代码二：简单加法器

再做一个能算数的程序。它问你两个数字，加起来告诉你结果。新建 `adder.py`：

```python
# adder.py

print("=== 简单加法器 ===")

# 输入两个数字，注意要用 int() 转成整数，否则会变成字符串拼接
num1 = int(input("请输入第一个数字："))
num2 = int(input("请输入第二个数字："))

result = num1 + num2   # 现在是真正的加法

print(f"{num1} + {num2} = {result}")
```

运行结果：

```
=== 简单加法器 ===
请输入第一个数字：25
请输入第二个数字：17
25 + 17 = 42
```

这里 `int()` 是关键中的关键。如果你忘了转，把 `num1 = int(input(...))` 写成 `num1 = input(...)`，那输入 25 和 17 后，得到的会是 `2517`（两串文字粘在一起），而不是 `42`。不信你去掉 `int()` 试试，亲眼看看这个坑长什么样，印象会更深。

> 小贴士：想让加法器算小数（比如 3.5 + 2.1），把 `int()` 换成 `float()` 就行。想减、乘、除，把 `+` 换成 `- * /` 即可。动手改改，它就是你自己的计算器了。

## 常见错误

**1. 忘了转类型，数字变成了字符串拼接**

```python
a = input("第一个数：")
b = input("第二个数：")
print(a + b)
```

输入 2 和 3，运行结果：

```
第一个数：2
第二个数：3
23
```

没报错，但结果是 `23` 不是 `5`！因为 `a`、`b` 是字符串，`+` 把它们粘起来了。要算数就得 `int(a) + int(b)`，或者输入时就 `int(input(...))`。

**2. 用户输入的不是数字，int() 转换失败**

```python
age = int(input("你几岁？"))
```

如果用户手滑输入了 `十三` 或 `abc`，会报错：

```
ValueError: invalid literal for int() with base 10: '十三'
```

`ValueError`（值错误）表示：`int()` 没法把"十三"这种不是阿拉伯数字的东西变成整数。现在你只要知道这个错长啥样就行——等学了第 4 章的条件判断，就能优雅地处理"用户乱输入"的情况了。

**3. f-string 忘了写前面的 `f`**

```python
name = "小明"
print("你好，{name}")
```

运行结果：

```
你好，{name}
```

大括号和里面的字被原样打印出来了！因为引号前面少了 `f`，Python 就把它当普通文字了。记住：**要替换变量，引号前必须有 `f`。**

**4. 把 `input` 拼错或漏了括号**

```python-norun
name = input "请输入名字"
```

报错：

```
SyntaxError: invalid syntax
```

`input` 后面必须紧跟圆括号 `()`，提示语放在括号里。函数调用永远离不开那对括号。

## 本章小结

这一章，你的程序学会了和人对话，回顾一下：

- **`input()`** 读取用户从键盘输入的内容，括号里可以写提示语；输入的内容用 `=` 存进变量。
- **最大的坑：`input()` 拿到的永远是字符串**，哪怕用户打的是数字，也是文字 `"13"` 而不是数字 13。
- 想拿输入做算术，先转类型：**`int(input(...))`** 转整数，**`float(input(...))`** 转小数。
- **`print()` 能打印多个内容**，用逗号 `,` 分隔会自动加空格；`sep` 控制分隔符，`end` 控制结尾。
- 组合输出最推荐 **f-string**：清楚、好读、排版随心。
- 遇到 `TypeError` 多半是类型没转对，遇到 `ValueError` 多半是 `int()` 收到了不是数字的东西。

现在程序会问、会记、会回应了。但它还很"死板"——不管你回答什么，它都走同样的流程。下一章，我们要教程序**做选择**：用**条件判断**（`if`），让程序根据不同的输入走不同的路。比如"如果分数大于 60，就说及格；否则说加油"。程序从此有了"脑子"，会变得聪明得多。翻页吧！
