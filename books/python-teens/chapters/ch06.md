## 列表：把一堆东西排成队

假设你要记住三种水果。用前面学的办法，得开三个变量：

```python
fruit1 = "苹果"
fruit2 = "香蕉"
fruit3 = "橙子"
```

要是有 100 种水果呢？这也太笨了。这时候就该请出**列表（list）**——它像一个能装很多东西的**长盒子**，一格挨一格，把同类的东西排成一队：

```python
# list_basic.py
fruits = ["苹果", "香蕉", "橙子"]   # 用方括号，元素之间用逗号隔开
print(fruits)
```

运行结果：

```
['苹果', '香蕉', '橙子']
```

创建列表用**方括号 `[ ]`**，里面的每样东西叫一个**元素（element）**，元素之间用逗号隔开。一个变量 `fruits`，就装下了全部三种水果。

## 用编号取出列表里的元素：索引

列表里每个元素都有一个**编号**，叫**索引（index）**。这里有个特别要记牢的规矩：

> **索引从 0 开始数，不是从 1！** 第一个元素的编号是 0，第二个是 1，以此类推。

想取出某个元素，就在列表名后面用方括号写上它的索引：

```python
# list_index.py
fruits = ["苹果", "香蕉", "橙子"]

print(fruits[0])    # 第一个元素
print(fruits[1])    # 第二个元素
print(fruits[2])    # 第三个元素
```

运行结果：

```
苹果
香蕉
橙子
```

记住：`fruits[0]` 才是第一个"苹果"。这个"从 0 开始"是全世界初学者都要适应一阵的地方，别嫌它奇怪，习惯就好。

## 往列表里添东西：append

列表最方便的一点，是**随时能往里加东西**。用 **`append`**（英语"添加"）把新元素接到队尾：

```python
# list_append.py
fruits = ["苹果", "香蕉"]
print("加之前：", fruits)

fruits.append("橙子")    # 在末尾添加一个
fruits.append("西瓜")    # 再添加一个
print("加之后：", fruits)
```

运行结果：

```
加之前： ['苹果', '香蕉']
加之后： ['苹果', '香蕉', '橙子', '西瓜']
```

`fruits.append("橙子")` 读作"让 fruits 这个列表，把'橙子'添到末尾"。中间那个点 `.` 你先理解成"的"——"fruits 的 append 功能"。列表可以从空开始，一点点往里加，这在写程序时特别常用。

## 遍历列表：一个一个拿出来处理

想把列表里每样东西都拿出来处理一遍（比如全打印出来），用上一章的 `for` 循环最合适。这叫**遍历（traverse）**列表：

```python
# list_loop.py
fruits = ["苹果", "香蕉", "橙子"]

for fruit in fruits:          # 把 fruits 里的元素一个个拿出来，装进 fruit
    print(f"我爱吃{fruit}")
```

运行结果：

```
我爱吃苹果
我爱吃香蕉
我爱吃橙子
```

注意这里 `for fruit in fruits`——它不再需要 `range`，而是**直接从列表里一个个取元素**。每转一圈，`fruit` 就变成列表里的下一样东西。这比用索引方便多了。

## 列表有多长：len

想知道列表里有几个元素，用 **`len`**（英语 length"长度"的缩写）：

```python
# list_len.py
fruits = ["苹果", "香蕉", "橙子"]
print(f"一共有 {len(fruits)} 种水果")
```

运行结果：

```
一共有 3 种水果
```

`len()` 数出来是 3。小心：因为索引从 0 开始，长度为 3 的列表，最大索引是 **2**（就是 `len - 1`），可没有 `fruits[3]`。

## 字典：给每样东西贴上标签

列表用**编号**找东西。但有时编号不好用——比如记全班同学的分数，你更想用"名字"去查分数，而不是"第 3 个同学"。这时就该用**字典（dict）**。

字典就像现实里的字典：你用一个**词**（叫**键，key**）去查它对应的**解释**（叫**值，value**）。它把数据存成一对一对的"**键值对**"：

```python
# dict_basic.py
# 记录三位同学的分数：名字是键，分数是值
scores = {"小明": 85, "小红": 92, "小刚": 78}
print(scores)
```

运行结果：

```
{'小明': 85, '小红': 92, '小刚': 78}
```

创建字典用**花括号 `{ }`**，每一对里 **键在前、值在后，中间用冒号 `:` 隔开**，对与对之间用逗号。

## 用键取值、增改

查字典，就用**键**去取**值**——在字典名后面用方括号写上键（注意不是索引编号，而是键本身）：

```python
# dict_get.py
scores = {"小明": 85, "小红": 92, "小刚": 78}

print(scores["小明"])          # 用键"小明"查它的分数
print(f"小红考了 {scores['小红']} 分")
```

运行结果：

```
85
小红考了 92 分
```

想**改**某个值，或**加**一对新的键值，都用同样的方括号写法：

```python
# dict_change.py
scores = {"小明": 85, "小红": 92}

scores["小明"] = 90         # 键已存在 → 修改它的值
scores["小刚"] = 78         # 键不存在 → 新增一对

print(scores)
```

运行结果：

```
{'小明': 90, '小红': 92, '小刚': 78}
```

规律很简单：`字典[键] = 值` 这句话，如果键已经在字典里，就是**修改**；如果键还没有，就是**新增**。

> 小贴士：列表和字典怎么选？如果你的数据讲究**顺序**、用**编号**取（比如排队名单、步骤清单），用列表；如果你想用一个**名字/标签**去查对应的信息（比如姓名查分数、单词查意思），用字典。

## 函数：把常用代码打包成一个按钮

假设程序里好几个地方都要打印一条漂亮的分割线：

```python-norun
print("=" * 20)
print("欢迎")
print("=" * 20)
# ……过一会儿又要一次
print("=" * 20)
print("再见")
print("=" * 20)
```

同样的 `print("=" * 20)` 抄来抄去，又累又容易错。有没有办法把它"打包"，用的时候喊一声就行？有——这就是**函数（function）**。

其实你早就用过函数了：`print()`、`input()`、`len()` 都是函数。它们是 Python 内置的"按钮"，按一下就干一件事。现在，你要学会**自己造按钮**。

用关键词 **`def`**（define"定义"的缩写）来定义一个函数：

```python
# func_basic.py
def say_hello():              # 定义一个叫 say_hello 的函数
    print("你好！")
    print("欢迎来学 Python！")

# 定义好之后，用"函数名()"来调用它
say_hello()
say_hello()                   # 想用几次就调几次
```

运行结果：

```
你好！
欢迎来学 Python！
你好！
欢迎来学 Python！
```

拆开看：

- **`def say_hello():`** 定义一个名叫 `say_hello` 的函数，后面跟括号和冒号。
- 缩进的部分是**函数体**——这个函数被调用时要执行的代码。
- 光"定义"函数，它**不会自己运行**，就像按钮装好了没人按。要**调用（call）**它，写 `say_hello()` ——加上那对括号，才算"按下按钮"。

## 给函数传参数

上面的函数每次都说一样的话。想让它更灵活——比如跟不同的人打招呼——就给它加**参数（parameter）**，也就是"调用时可以传进去的信息"：

```python
# func_param.py
def greet(name):              # name 是参数，像个占位的空盒子
    print(f"你好，{name}！")

greet("小明")                 # 把"小明"传给 name
greet("小红")
```

运行结果：

```
你好，小明！
你好，小红！
```

括号里的 `name` 是参数。调用时你在括号里给的东西（"小明"），就会装进 `name`，函数体里就能用它了。同一个函数，传不同的参数，就有不同的效果——这才叫灵活。

## 让函数返回结果：return

前面的函数只是"打印"。但更常见的是：让函数**算出一个结果，交还给你**，好让你接着用。这要用 **`return`**（英语"返回"）：

```python
# func_return.py
def add(a, b):                # 两个参数，用逗号隔开
    result = a + b
    return result             # 把结果交回去

# 函数返回的值，可以存进变量，也可以直接用
total = add(3, 5)
print(f"3 加 5 等于 {total}")
print(f"10 加 20 等于 {add(10, 20)}")
```

运行结果：

```
3 加 5 等于 8
10 加 20 等于 30
```

`return result` 的意思是"把 `result` 送出函数，交给调用它的地方"。有了 `return`，`add(3, 5)` 整个就**代表 8 这个值**，你可以像用普通数字一样用它。

`return` 和 `print` 的区别很多人搞混：`print` 只是**显示在屏幕上**，看完就没了；`return` 是把结果**交回去让程序接着用**。想让函数的结果参与后面的计算，就必须 `return`。

## 为什么要用函数？

你可能会想：不用函数也能写程序呀，干嘛多此一举？函数至少有三大好处：

1. **不重复**：一段代码写一次，到处调用，改起来也只改一处。
2. **好理解**：给一堆代码起个名字（比如 `check_answer`），别人一看名字就懂它干嘛，程序像搭积木一样清晰。
3. **好分工**：把大问题拆成一个个小函数，各管一摊，再拼起来——这是专业程序员写大程序的秘诀。

记住一句话：**当你发现某段代码要用好几次，或者想给一段逻辑起个名字时，就该把它写成函数。**

## 完整代码：背单词小程序

现在把列表、字典、函数**全用上**，做一个像模像样的背单词程序。它用字典存"单词→中文意思"，用列表把要背的单词排好队，再用函数把"检查答案"这段逻辑打包起来。新建 `word_card.py`：

```python
# word_card.py

# 用字典存单词卡片：键是英文单词，值是中文意思
word_bank = {
    "apple": "苹果",
    "book": "书",
    "cat": "猫",
}

# 用列表决定考哪些词、按什么顺序考
quiz_list = ["apple", "book", "cat"]


def check(word, user_answer):
    """检查一个单词答得对不对，答对返回 True，答错返回 False"""
    correct = word_bank[word]      # 从字典查出正确答案
    if user_answer == correct:
        print("答对了！✔")
        return True
    else:
        print(f"答错了，{word} 的意思是 {correct}")
        return False


print("=== 背单词小程序 ===")

right = 0                          # 记录答对了几张
total = len(quiz_list)             # 一共几张卡片

# 遍历列表，一张一张地考
for i in range(total):
    word = quiz_list[i]
    answer = input(f"第 {i + 1} 张：{word} 是什么意思？")
    if check(word, answer):        # 调用函数检查，顺便拿到对错
        right = right + 1          # 答对就加一分

# 算出正确率并汇报
rate = int(right / total * 100)
print(f"一共 {total} 张卡片，你答对了 {right} 张，正确率 {rate}%！")
```

运行结果（下划线部分是你输入的）：

```
=== 背单词小程序 ===
第 1 张：apple 是什么意思？苹果
答对了！✔
第 2 张：book 是什么意思？书
答对了！✔
第 3 张：cat 是什么意思？狗
答错了，cat 的意思是 猫
一共 3 张卡片，你答对了 2 张，正确率 66%！
```

看看这个程序，本章三样工具配合得多好：

- **字典 `word_bank`** 存着每个单词的正确意思，用单词当键，一查就到。
- **列表 `quiz_list`** 决定考哪些词、什么顺序，用 `for` 遍历它，一张张考。
- **函数 `check`** 把"对比答案、打印对错、返回结果"这段逻辑打包，主程序里只要 `check(word, answer)` 一句就调用，干净利落。

想加新单词？往字典和列表里各添一项就行，主程序一个字都不用改——这就是好结构的威力。

> 小贴士：那个用三引号 `"""..."""` 写的句子叫**文档字符串（docstring）**，是专门给函数写的说明，放在函数体第一行，解释这个函数是干嘛的。它像注释一样不影响运行，但能让别人（和以后的你）秒懂函数的用途。

## 常见错误

**1. 列表索引越界**

```python-norun
fruits = ["苹果", "香蕉", "橙子"]
print(fruits[3])       # 想取第 4 个，可只有 3 个
```

报错：

```
IndexError: list index out of range
```

`IndexError`（索引错误）表示编号超出了列表范围。3 个元素的列表，索引只有 0、1、2，没有 3。记住**最大索引是 `len - 1`**。

**2. 字典里查一个不存在的键**

```python-norun
scores = {"小明": 85}
print(scores["小红"])      # 字典里没有"小红"
```

报错：

```
KeyError: '小红'
```

`KeyError`（键错误）表示字典里没有这个键。检查一下键名有没有拼错，或者这个键是不是还没加进去。

**3. 定义了函数却忘了调用**

```python
def say_hi():
    print("嗨！")
# 只定义，没调用
```

运行结果：

```

```

什么都不打印！因为函数只是"定义好了"，没人"按按钮"。得加上 `say_hi()` 才会执行。定义 ≠ 运行，别忘了那对调用的括号。

**4. 函数忘了 return，结果拿不到**

```python
def add(a, b):
    result = a + b
    # 忘了 return result

print(add(3, 5))
```

运行结果：

```
None
```

`None` 是 Python 里表示"什么都没有"的特殊值。函数算是算了，但没 `return`，结果没送出来，外面拿到的就是"空"。要拿结果，函数里必须 `return`。

**★★ 挑战二：宠物名册**
用字典存几只宠物的信息（键是宠物名字，值是它的种类，比如 `"旺财": "狗"`）。让用户输入一个名字，程序查字典告诉他这是什么动物；如果字典里没有，就提示"查无此宠物"。（提示：可以先用 `if 名字 in 字典:` 判断键在不在。）

**★★★ 挑战三：班级成绩管家**
做一个小程序：用字典存全班同学的分数。写至少两个函数——一个 `add_student(name, score)` 往字典里加一名同学，一个 `average()` 算出全班平均分。再让程序打印出最高分是谁。（提示：遍历字典求平均和最大值；算平均要用到 `len`。）

## 本章小结

这一章你拿到了三样整理代码的利器，程序从此能又大又整齐。回顾要点：

- **列表（list）** 用方括号 `[ ]` 创建，把一堆东西排成队；用**索引**取元素，**索引从 0 开始**。
- 列表常用操作：`append()` 加元素、`len()` 求长度、用 `for` **遍历**每个元素。
- **字典（dict）** 用花括号 `{ }` 创建，存"**键值对**"；用**键**取值 `d[键]`，`d[键] = 值` 可修改或新增。
- 列表看**顺序、用编号**，字典看**标签、用键**，按需选用。
- **函数（def）** 把常用代码打包成"按钮"：定义后要**调用**才运行；用**参数**传信息进去，用 **`return`** 把结果交出来。
- 函数的好处：不重复、好理解、好分工。发现代码要重复用，就写成函数。
