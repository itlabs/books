## 一个大问题：AI 其实"没有记性"

先做个实验。回想上一章的代码，它每次运行只发一条消息。如果你连着问两次，AI 会记得第一次吗？

答案是：**不会**。每次调用 `messages.create()`，对 AI 来说都是**全新的、互不相干的一次对话**。它就像一个每次醒来都失忆的人——你刚说完的话，下一句它就忘了。

这怎么能聊天呢？秘密在于：**记性要靠我们的程序自己维护**。

我们的办法是：准备一个**列表**，把每一句话（你说的、AI 说的）都**依次存进去**。每次要 AI 回答时，就把**到目前为止的全部对话**一起发过去。这样 AI 一看："哦，原来我们之前聊了这些"，就能接上话了。

## 一句话对应一个字典

还记得上一章 `messages` 里的写法吗？每条消息是一个**字典**：

```python-norun
{"role": "user", "content": "我叫小美。"}
```

- `"role"` 是"谁说的"：`"user"` 是你（用户），`"assistant"`（助手）就是 AI。
- `"content"` 是"说了什么"。

所以一段对话，就是一串这样的字典组成的**列表**：

```python-norun
messages = [
    {"role": "user", "content": "我叫小美。"},
    {"role": "assistant", "content": "你好呀小美！很高兴认识你～"},
    {"role": "user", "content": "你还记得我叫什么吗？"},
]
```

把这整个列表发给 AI，它就能根据前面几句，答出"你叫小美"。**这就是记性的全部秘密。**

## 搭建聊天机器人

思路理清了，代码其实就是把第 5 章的 `while` 循环、第 6 章的列表，和上一章的 AI 调用**拼在一起**：

1. 准备一个空的 `messages` 列表。
2. 进入 `while` 循环：
   - 让用户输入一句话（`input`）。
   - 如果用户输入"再见"，就跳出循环，结束。
   - 把用户这句话**追加**进 `messages`。
   - 把整个 `messages` 发给 AI，拿到回答。
   - 打印回答，并把回答也**追加**进 `messages`（这样下一轮它就记得自己说过啥）。

新建 `chatbot.py`：

```python-norun
# chatbot.py
import anthropic

client = anthropic.Anthropic()

# 这个列表就是机器人的"记忆"，一开始是空的
messages = []

print("聊天机器人已上线！（输入'再见'结束）")

while True:
    # 1. 用户说一句
    user_says = input("你：")

    # 2. 想结束就退出
    if user_says == "再见":
        print("机器人：拜拜，下次再聊！")
        break

    # 3. 把用户这句存进记忆
    messages.append({"role": "user", "content": user_says})

    # 4. 把全部对话发给 AI，拿回答
    response = client.messages.create(
        model="claude-haiku-4-5",
        max_tokens=300,
        messages=messages,
    )

    # 5. 取出 AI 的回答文字
    reply = ""
    for block in response.content:
        if block.type == "text":
            reply += block.text

    # 6. 打印，并把 AI 的回答也存进记忆
    print("机器人：" + reply)
    messages.append({"role": "assistant", "content": reply})
```

设置好密钥（见上一章）后运行 `python chatbot.py`，你就能和它连续聊天了！试试先告诉它你的名字，聊几句，再问它"我叫什么"，看它记不记得。

> 💡 小贴士：第 5 步为什么用 `reply += block.text` 而不是直接 `print`？因为我们既要**打印**回答，又要把回答**存进记忆**（第 6 步）。所以先把文字拼进 `reply` 变量攒起来，再一次用掉。

## 给机器人一个"人设"

现在的机器人挺普通。我们可以给它一个**人设（角色设定）**，让它扮演成任何角色——比如一只傲娇的猫、一位耐心的数学老师、一个海盗船长。

方法是用一个叫 **system（系统提示）** 的设定。它像是给演员的"剧本说明"，在对话开始前悄悄告诉 AI："你要扮演谁、用什么口气说话"。

在 `messages.create` 里加一个 `system` 参数：

```python-norun
# cat_bot.py
import anthropic

client = anthropic.Anthropic()

# 人设：让它扮演一只可爱又有点傲娇的小猫
persona = "你是一只可爱又有点傲娇的小猫，名叫团子。说话要软萌，偶尔在句尾加'喵'，喜欢卖萌，但也很关心和你聊天的主人。"

messages = []
print("小猫团子已经趴在键盘上啦！（输入'再见'结束）")

while True:
    user_says = input("你：")
    if user_says == "再见":
        print("团子：主人要走了吗…那团子先睡觉觉了喵～")
        break

    messages.append({"role": "user", "content": user_says})

    response = client.messages.create(
        model="claude-haiku-4-5",
        max_tokens=300,
        system=persona,          # 把人设告诉 AI
        messages=messages,
    )

    reply = ""
    for block in response.content:
        if block.type == "text":
            reply += block.text

    print("团子：" + reply)
    messages.append({"role": "assistant", "content": reply})
```

运行看看，它会全程用小猫的口气跟你说话！

> 💡 小贴士：`system` 只需要设一次，它管着整场对话的"风格"。注意：`system` 是单独的参数，**不要**把人设塞进 `messages` 列表里。

## 常见错误

**1. 忘了把 AI 的回答存回 messages**

如果你只 `append` 用户的话、忘了 `append` AI 的回答，机器人就会"半失忆"——它记得你说的，却不记得自己说过什么，对话会变得很奇怪。**一定要一问一答都存进列表。**

**2. 死循环出不来**

```python-norun
while True:
    user_says = input("你：")
    # 忘了写 if ... break，永远退不出去！
```

`while True` 是死循环，必须有 `break` 才能停。我们靠"输入'再见'就 break"来退出。（还记得第 5 章的安全提醒吗？）

**3. 把 system 写进了 messages**

```python-norun
# 错误示范
messages = [{"role": "system", "content": persona}]
```

在这套写法里，人设要用**单独的 `system` 参数**传，不要放进 `messages`。放错了可能报错或不生效。

**4. 聊太久变慢、变贵**

对话越长，`messages` 列表越大，每次发给 AI 的内容越多——会**变慢**，也**更费钱**。练习时聊几轮就好；真正的产品会用一些办法来"精简旧对话"，那是更进阶的话题了。

## 本章小结

你做出了一个有记性、有个性的聊天机器人！回顾关键点：

- AI 本身**没有记性**，每次调用都是全新对话；**记性靠我们的程序用列表维护**。
- 把每一句对话（`user` 和 `assistant`）都 `append` 进 `messages` 列表，每次把**整个列表**发给 AI。
- 用 **`while` 循环** 实现"一直聊"，用 **`break`**（如输入"再见"）来退出。
- 用 **`system` 参数**给 AI 设定**人设**，一句话就能让它变身不同角色。
- 对话越长越慢越贵，练习时适度就好。
