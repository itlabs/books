## 一只会画画的小海龟

想象一只小海龟，尾巴上绑着一支笔，趴在一张大白纸的正中央。你喊一声"前进 100 步"，它就朝着头对的方向爬 100 步，笔一路拖着，纸上就留下一条线。你再喊"右转 90 度"，它原地把头扭向右边。就这样，一句一句地指挥，它爬过的轨迹拼起来，就成了一幅画。

这只海龟就藏在 Python 自带的一个**模块（module）**里，模块就是"别人提前写好、打包送给你用的一堆工具"。这个模块的名字叫 **turtle**（英语"海龟"的意思）。要用它，先在程序开头把它"请"进来：

```python-norun
import turtle
```

**`import`** 是英语"引入、导入"的意思。`import turtle` 这行的意思就是："把 turtle 这盒工具搬进我的程序，接下来我要用它。"以后凡是要用海龟的功能，都写成 `turtle.` 加上具体动作。

## 让海龟走第一步

新建一个文件，命名为 `first_turtle.py`，抄进下面这几行：

```python-norun
# first_turtle.py
import turtle

turtle.forward(100)   # 向前爬 100 步（像素）
turtle.done()         # 画完了，让窗口停住别关
```

在自己电脑上运行（终端里输入 `python first_turtle.py`）。一个窗口会弹出来，中间出现一只小海龟（有的电脑上是一个小箭头），它向右爬了一段，留下一条横线。

我们来认识这两行新东西：

- **`turtle.forward(100)`**：`forward` 是"前进"的意思。括号里的 `100` 是走多少步——这里的"步"其实是屏幕上的**像素（pixel）**，也就是一个个小点。数字越大，线越长。
- **`turtle.done()`**：告诉程序"画完了"。有了它，窗口才会乖乖停在那儿等你看；不写的话，窗口可能一闪就没了。**它一般放在整个程序的最后一行。**

> 小贴士：海龟一开始头朝右边（东边），所以 `forward` 默认是往右爬。别急，马上就教它转弯。

## 转弯：right 和 left

只会直着走可画不出图形，得会拐弯。海龟转弯用这两个工具：

- **`turtle.right(90)`**：原地向**右**转 90 度（`right` 是"右"）。
- **`turtle.left(90)`**：原地向**左**转 90 度（`left` 是"左"）。

注意：转弯时海龟**不动地方，只扭头**，改变的是它下一步要爬的方向。度数就是转多少——90 度是转个直角，180 度是掉头往回。

下面让海龟走一个"L"形：先向前，再右转，再向前。

```python-norun
# corner.py
import turtle

turtle.forward(100)   # 先向右爬 100
turtle.right(90)      # 向右转 90 度，现在头朝下
turtle.forward(100)   # 再向下爬 100
turtle.done()
```

运行一下，你会看到一条拐了个直角的线。**"前进 + 转向"这两个动作反复组合，几乎能画出任何形状。** 下面这张图帮你看清海龟每一步在做什么：

## 完整程序一：画一个正方形

正方形有四条一样长的边，每画完一条边就右转 90 度。你当然可以老老实实写四遍"前进、右转"：

```python-norun
# square_long.py
import turtle

turtle.forward(100)
turtle.right(90)
turtle.forward(100)
turtle.right(90)
turtle.forward(100)
turtle.right(90)
turtle.forward(100)
turtle.right(90)
turtle.done()
```

能画出来，但你有没有觉得——**这四组代码几乎一模一样，抄四遍好烦？** 还记得第 6 章学的**循环**吗？重复的事，正是循环最擅长的！"前进 + 右转"重复 4 次，用 `for` 循环写就清爽多了：

```python-norun
# square.py
import turtle

for i in range(4):        # 重复做 4 次
    turtle.forward(100)   # 画一条边
    turtle.right(90)      # 右转 90 度，准备画下一条

turtle.done()
```

运行它，海龟就会画出一个端端正正的正方形。是不是短多了？这就是循环的威力：**同样的动作，交给循环去重复。**

> 小贴士：为什么是右转 90 度？因为正方形四个角加起来转一整圈是 360 度，4 个角平分，每个角就转 `360 ÷ 4 = 90` 度。记住这个"360 除以边数"，等下画别的多边形要用到。

## 举一反三：任意多边形

正方形转 90 度，那三角形、五边形呢？按上面的规律：**转的度数 = 360 ÷ 边数**。想画几边形，就把循环次数和转角一起改掉。比如画一个正五边形（5 条边，每个角转 `360 ÷ 5 = 72` 度）：

```python-norun
# pentagon.py
import turtle

for i in range(5):        # 五条边，循环 5 次
    turtle.forward(100)
    turtle.right(72)      # 360 ÷ 5 = 72 度

turtle.done()
```

把 `5` 和 `72` 换成 `3` 和 `120`，就是三角形；换成 `6` 和 `60`，就是六边形。**你现在已经能画出任意正多边形了**，试试看！

## 给海龟换个颜色

黑白线条太朴素了。海龟的画笔可以换颜色，用 **`turtle.color`**：

```python-norun
# color_square.py
import turtle

turtle.color("red")       # 把画笔换成红色（red 是"红"）
for i in range(4):
    turtle.forward(120)
    turtle.right(90)
turtle.done()
```

括号里放颜色的英文名，都要用引号包起来。常用的有：`"red"`（红）、`"blue"`（蓝）、`"green"`（绿）、`"orange"`（橙）、`"purple"`（紫）、`"pink"`（粉）、`"black"`（黑）。多试几个，挑你喜欢的。

## 抬笔和落笔：penup 与 pendown

有时你想让海龟**走过去但不留线**——比如画完一个图形，想挪到别处再画下一个。这时就得把笔"抬起来"：

- **`turtle.penup()`**：抬笔（pen up，笔抬起）。之后海龟移动**不画线**。
- **`turtle.pendown()`**：落笔（pen down，笔放下）。之后海龟移动**又开始画线**了。

就像你写字，抬起笔挪到下一行，再落笔继续写。看这个例子：画一条线，空一段，再画一条线。

```python-norun
# jump.py
import turtle

turtle.forward(100)   # 画第一条线
turtle.penup()        # 抬笔
turtle.forward(50)    # 悄悄往前挪 50，不留痕迹
turtle.pendown()      # 落笔
turtle.forward(100)   # 画第二条线
turtle.done()
```

运行后你会看到两条线中间空了一段——那就是海龟抬着笔走过的地方。

## 完整程序二：画一朵花

现在来点更酷的。一朵花，可以看成"很多个正方形（或多边形）绕着中心转一圈"。做法是：画一个多边形，**转一个小角度**，再画一个，再转……转满一圈，一朵花就出来了。

思路拆开：
1. 外层循环转一圈（比如转 36 次，每次转 10 度，`36 × 10 = 360`，正好一整圈）。
2. 每次循环里，先用内层循环画一个多边形（一片"花瓣"）。
3. 顺便每片花瓣换个颜色，更好看。

```python-norun
# flower.py
import turtle

# 准备一串颜色，画每片花瓣时轮流用
colors = ["red", "orange", "purple", "blue", "green", "pink"]

turtle.speed(0)              # 把海龟速度调到最快（0 表示最快），不然画得太慢

for i in range(36):          # 外层：转一圈，画 36 片花瓣
    turtle.color(colors[i % 6])   # 用 i 除以 6 的余数，在 6 种颜色间轮换
    for j in range(4):       # 内层：画一个正方形当花瓣
        turtle.forward(100)
        turtle.right(90)
    turtle.right(10)         # 画完一片，整体转 10 度，再画下一片

turtle.done()
```

在电脑上运行它，你会看到海龟飞快地转着圈，一片一片地叠出一朵五彩缤纷的花。**这就是你写出的第一幅真正的图画！**

这里出现了两个小新招，顺带认识一下：

- **`turtle.speed(0)`**：调海龟爬行速度。数字 1（最慢）到 10（快），特别地，`0` 表示**最快**。画复杂图形时调快点，省得等半天。
- **`colors[i % 6]`**：`colors` 是一个**列表**（第 7 章学过），里面装了 6 种颜色。`i % 6` 是"i 除以 6 的余数"，结果总在 0～5 之间，正好能从列表里轮流取出这 6 种颜色。这样每片花瓣颜色都不同。

试着把颜色列表、花瓣形状（把内层改成三角形或五边形）、转的角度换一换，你会得到完全不同的花。

## 常见错误

海龟画图是你第一次在自己电脑上跑程序，除了老朋友"中文符号""拼写错误"，还会遇到几个新坑：

**1. 点了网页的"运行"按钮，什么都没发生**

这不是 bug！本章开头就说过：**turtle 的窗口浏览器里弹不出来**，网页运行按钮跑不了它。你必须把代码存成 `.py` 文件，在自己电脑上用 `python 文件名.py` 运行。没装好 Python 的话，回到[附录 A](appendix-a.md) 一步步来。

**2. 忘了写 `import turtle`**

```python-norun
turtle.forward(100)
```

报错：

```
NameError: name 'turtle' is not defined
```

`NameError`（名字错误）说电脑不认识 `turtle`。因为你还没把这盒工具搬进来。**程序第一行永远先写 `import turtle`。**

**3. 忘了写 `turtle.done()`，窗口一闪就关了**

如果运行后窗口"啪"地闪一下就消失，多半是漏了最后的 `turtle.done()`。加上它，窗口就会停住等你欣赏。

**4. 颜色名写错或没加引号**

```python-norun
turtle.color(red)
```

报错：

```
NameError: name 'red' is not defined
```

颜色名是**字符串**，必须用引号包起来：`turtle.color("red")`。另外颜色要用英文名，写 `"红色"` 电脑也不认识，会报另一种错。

## 动手挑战

下面的程序都要在**你自己电脑上**运行才看得到效果哦。

**★ 挑战一：画个三角形**
仿照正方形的写法，用 `for` 循环画一个正三角形。想一想：三条边，每个角该转多少度？（提示：360 ÷ 3。）

**★★ 挑战二：五角星**
画一个五角星。它有 5 条边，但每画完一条边要转的角度不是 72 度，而是 **144 度**（试试就知道为什么星星和五边形不一样）。用 `range(5)` 的循环搞定它。

**★★★ 挑战三：彩色螺旋**
画一条越转越大的彩色螺旋线。思路：用一个循环，每一圈都比上一圈爬得**更远一点**，同时**换个颜色**。提示——让前进的步数跟着循环变量变大，比如第 `i` 次就 `forward(i * 5)`，每次还稍微转个角度（比如右转 91 度，故意不整齐，会更像螺旋）。多调几个数字，做出你自己的螺旋。

## 本章小结

太棒了，你从"让电脑说话"跨到了"让电脑画画"！回顾这一章：

- 用 **`import turtle`** 把海龟工具箱搬进程序，这是画图程序的第一行。
- **`forward()`／`backward()`** 让海龟前进／后退，**`right()`／`left()`** 让它右转／左转。
- **前进 + 转向反复组合**就能画出各种形状；正多边形每个角转 `360 ÷ 边数` 度。
- 用**循环**代替重复的"前进、转弯"，代码又短又清楚。
- **`color("颜色名")`** 换画笔颜色，**`penup()`／`pendown()`** 控制走的时候画不画线。
- **`turtle.done()`** 放在最后，让窗口停住；**`speed(0)`** 让海龟画得飞快。
- 最重要的一点：**turtle 要在自己电脑上运行**，网页运行按钮跑不了它——你已经迈进真正的编程环境了。
