## 第一步：让 AI 出一道题

我们先单独练一个"出题"的小工具。关键在于**把要求说清楚**——告诉 AI 主题是什么、只要题目本身、别把答案也说出来（不然就剧透了）。

```python-norun
# ask_question.py
import anthropic

client = anthropic.Anthropic()

topic = "太空"

# 把我们的要求写清楚，这段话叫"提示词（prompt）"
prompt = f"请出一道关于「{topic}」的知识问答题，适合初中生。只输出题目本身，不要给出答案，不要加'题目：'之类的前缀。"

response = client.messages.create(
    model="claude-haiku-4-5",
    max_tokens=150,
    messages=[{"role": "user", "content": prompt}],
)

question = ""
for block in response.content:
    if block.type == "text":
        question += block.text

print("AI 出的题：", question)
```

注意那句 `prompt` 用了 **f-string**（第 2 章学的）把 `topic` 拼进去。这样只要换 `topic`，就能出不同主题的题。**你对 AI 的要求越具体，它给的结果越合你意**——这是使用 AI 的第一条黄金法则。

## 第二步：让 AI 判分

玩家答完，怎么知道对不对？我们**再问一次 AI**，把"题目 + 玩家的答案"一起发过去，让它当裁判。

这里有个小技巧：我们让 AI 的回答**以"对"或"错"开头**，这样程序就能靠开头这个字判断得分，同时后面还能带上讲解给玩家看。

```python-norun
# judge.py
import anthropic

client = anthropic.Anthropic()

question = "太阳系中最大的行星是哪一颗？"
player_answer = "木星"

prompt = f"""这是一道题：{question}
玩家的答案是：{player_answer}

请判断玩家答得对不对。如果对，回答以"对"字开头；如果错，回答以"错"字开头。
然后用一两句话简短讲解正确答案，语气鼓励，适合初中生。"""

response = client.messages.create(
    model="claude-haiku-4-5",
    max_tokens=200,
    messages=[{"role": "user", "content": prompt}],
)

verdict = ""
for block in response.content:
    if block.type == "text":
        verdict += block.text

print(verdict)

# 靠开头那个字判断得分
if verdict.startswith("对"):
    print(">> 记 1 分！")
else:
    print(">> 这题没得分，继续加油！")
```

> 💡 小贴士：让 AI 用固定格式开头（这里是"对"/"错"），程序就能"读懂"它的判断。这是一个很常用的技巧——**给 AI 定一个好检查的回答格式**。当然 AI 偶尔也会不完全照格式来，所以真正严谨的程序还会做更多检查，但对我们的游戏来说，这样已经够用又好玩了。

## 第三步：拼成完整游戏

把"出题"和"判分"两个工具，用第 5 章的 `for` 循环串起来，再加上记分，一个完整的游戏就诞生了。为了让代码整洁，我们用第 6 章学的**函数**，把"出题"和"判分"各包成一个函数。

```python-norun
# quiz_game.py
import anthropic

client = anthropic.Anthropic()


def ask_ai(prompt):
    """一个小助手函数：把 prompt 发给 AI，返回它的文字回答。"""
    response = client.messages.create(
        model="claude-haiku-4-5",
        max_tokens=250,
        messages=[{"role": "user", "content": prompt}],
    )
    text = ""
    for block in response.content:
        if block.type == "text":
            text += block.text
    return text


def make_question(topic):
    """让 AI 出一道关于 topic 的题。"""
    prompt = f"请出一道关于「{topic}」的知识问答题，适合初中生。只输出题目本身，不要给答案，不要加前缀。"
    return ask_ai(prompt)


def judge(question, answer):
    """让 AI 判断答案对不对，返回它的评语（以'对'或'错'开头）。"""
    prompt = f"""这是一道题：{question}
玩家的答案是：{answer}
请判断对错。对就以"对"开头，错就以"错"开头，然后用一两句话鼓励性地讲解正确答案。"""
    return ask_ai(prompt)


# ===== 游戏开始 =====
print("🎯 欢迎来到 AI 智能问答闯关！")
topic = input("选个主题（如：太空 / 动物 / 历史 / 电影）：")

score = 0                      # 得分
total = 3                      # 一共出几题

for i in range(total):         # 循环出 total 道题
    print(f"\n—— 第 {i + 1} 题 ——")
    question = make_question(topic)
    print(question)

    answer = input("你的答案：")

    verdict = judge(question, answer)
    print(verdict)

    if verdict.startswith("对"):
        score += 1
    print(f"当前得分：{score} / {i + 1}")

# ===== 公布成绩 =====
print(f"\n🏁 闯关结束！你答对了 {score} / {total} 题。")
if score == total:
    print("太厉害了，全对！你就是这个主题的小专家！🎉")
elif score >= total / 2:
    print("不错哦，继续加油！")
else:
    print("没关系，多玩几次就越来越强啦！")
```

运行 `python quiz_game.py`，你就拥有了一个**永远出新题、还会讲解**的问答游戏。想玩历史就输历史，想玩动物就输动物，题目全是 AI 现编的！

## 拆开看看

这个游戏其实是把你学过的东西**全用上了**：

- **函数**（第 6 章）：`ask_ai`、`make_question`、`judge` 把重复的活儿打包，代码干净好读。注意 `ask_ai` 被另外两个函数调用——函数可以调用函数。
- **f-string**（第 2 章）：把 `topic`、`question`、`answer` 拼进提示词。
- **for 循环 + range**（第 5 章）：控制出几道题。
- **if / elif / else**（第 4 章）：判断得分、公布评语。
- **变量累加**（第 5 章）：`score += 1` 记分。
- **input / print**（第 3 章）：和玩家互动。
- **AI 调用**（第 12、13 章）：让内容变"活"。

看，你不是在学一堆孤立的知识点——它们最后会**汇合成一个真正好玩的作品**。这就是编程最迷人的地方。

## 常见错误

**1. AI 偶尔不按"对/错"开头**

AI 不是每次都 100% 听话，可能回答"这个答案是正确的……"而不是"对……"开头，导致明明答对了却没记分。这不是你的 bug，是 AI 的特点。可以在提示词里**再强调一遍格式**（"必须严格以'对'或'错'这个字开头"），能改善很多。

**2. 每题都要联网、要花一点点钱**

这个游戏每出一题、每判一次分，都要"打电话"给 AI。所以玩的时候要联网，也会消耗一点点额度。把 `total` 设小一点（比如 3）练习就好。

**3. 忘了 return，函数没把结果给出来**

```python-norun
def make_question(topic):
    prompt = f"..."
    ask_ai(prompt)      # 少了 return！
```

函数算完了却不 `return`，外面就拿不到结果，`question` 会变成 `None`。记得把结果 `return` 出去。（第 6 章讲过。）

**4. 把 max_tokens 设太小**

判分要带讲解，如果 `max_tokens` 太小（比如 20），讲解会被"说一半就断掉"。给判分留够长度（比如 200 以上）。

## 动手挑战

**★ 挑战一：加长闯关**
把 `total` 改成 5 或 10 题。想一想题目变多后，玩一局要花多长时间、多问几次 AI。

**★★ 挑战二：难度选择**
在开始时多问一句玩家想要的难度（简单/普通/困难），把难度也拼进 `make_question` 的提示词里，让 AI 按难度出题。

**★★★ 挑战三：出选择题**
改造 `make_question`，让 AI 出**带 A/B/C/D 四个选项的选择题**（在提示词里要求它给出题目和四个选项）。玩家输入 A/B/C/D 作答，再让 AI 判分。这样答案更好判断，游戏也更规范。
