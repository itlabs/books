## 第一步：让电脑随机出拳

石头剪刀布有三种出法。我们先把它们放进一个**列表（list）**里——列表就像一个能装很多东西的收纳盒，东西排着队待在里面。

然后请出上上一章认识的 `random` 工具箱。这次用的是它的另一个工具 **`random.choice`**（choice 是英语"选择"）：给它一个列表，它就**随机从里面挑一个**出来，就像从三张牌里随手抽一张。

```python
# computer_move.py
import random

moves = ["石头", "剪刀", "布"]  # 三种拳，装进列表

# 从列表里随机抽一个，就是电脑出的拳
computer = random.choice(moves)
print("电脑出了：", computer)
```

多运行几次，电脑出的拳每次都不一样。`random.choice(moves)` 就是"从 moves 这个盒子里，随机拿一个出来"。

## 第二步：让玩家也出拳

玩家用数字选拳更方便：`1` 石头、`2` 剪刀、`3` 布。我们用 `input()` 收玩家的选择，再把数字对应到拳。

```python
# player_move.py
moves = ["石头", "剪刀", "布"]

print("1) 石头  2) 剪刀  3) 布")
choice = input("你的选择：")

if choice == "1":
    player = "石头"
elif choice == "2":
    player = "剪刀"
elif choice == "3":
    player = "布"
else:
    player = "石头"   # 乱输入就当出了石头，保证游戏不卡住

print("你出了：", player)
```

这里也可以用一个更聪明的写法，直接从列表里取：玩家打 `1`，我们就取列表的第 1 个……不过列表是从 0 开始数的，所以要 `moves[int(choice) - 1]`。这一章我们先用清楚好懂的 `if/elif` 写法，等你熟了再试那种简洁写法。

## 第三步：判断谁赢了

这是游戏的"大脑"。石头剪刀布的规则你早就滚瓜烂熟：

- **石头**砸**剪刀**
- **剪刀**剪**布**
- **布**包**石头**
- 出得一样，就是**平局**

我们把它翻译成 `if / elif / else`。先看"平局"最好判断——两人出的一样就是平。剩下的，就把玩家能赢的三种情况列出来：

```python
# judge.py
player = "石头"
computer = "剪刀"

if player == computer:
    print("平局！")
elif (player == "石头" and computer == "剪刀") or \
     (player == "剪刀" and computer == "布") or \
     (player == "布" and computer == "石头"):
    print("你赢了这一局！")
else:
    print("电脑赢了这一局！")
```

第二支的条件有点长，我们拆开看：它把玩家会赢的**三种情况**用 `or`（英语"或者"）连起来——只要**其中任意一种**成立，玩家就赢了。

- `player == "石头" and computer == "剪刀"`：你石头砸它剪刀，赢；
- `player == "剪刀" and computer == "布"`：你剪刀剪它布，赢；
- `player == "布" and computer == "石头"`：你布包它石头，赢。

行末那个 `\` 是"这行还没写完，下一行接着"的意思，只是为了让长条件换行看着舒服，你也可以全写在一行。

> 想一想：为什么剩下的情况都归 `else`（电脑赢）？因为平局和"你赢"都已经被前面挑走了，剩下的必然是你输。**把情况一层层筛掉，是判断胜负的常用思路。**

## 第四步：记分板——用字典记分

比赛得记分。我们用一个**字典（dictionary）**当记分板。字典就像一张贴着标签的储物柜：每样东西都有自己的"标签"（叫**键**），顺着标签就能找到对应的"值"。这里我们贴两个标签——`"玩家"` 和 `"电脑"`，值就是各自的得分。

```python
# scoreboard.py
# 记分板：一开始两边都是 0 分
score = {"玩家": 0, "电脑": 0}

# 假设玩家赢了一局，就给玩家那栏加 1
score["玩家"] = score["玩家"] + 1

print(score)
print("玩家得分：", score["玩家"])
print("电脑得分：", score["电脑"])
```

运行结果：

```
{'玩家': 1, '电脑': 0}
玩家得分： 1
电脑得分： 0
```

- `score["玩家"]` 就是顺着 `"玩家"` 这个标签，找到玩家的分数。
- `score["玩家"] = score["玩家"] + 1` 是"把玩家现在的分加 1，再存回去"，和上一章计数器一个道理。

用字典比用两个单独的变量 `player_score`、`computer_score` 更整齐——两边的分都收在同一个记分板里，一目了然。

## 完整的石头剪刀布游戏

零件齐了，拼成完整游戏！我们让玩家决定玩几局，用 `while` 循环一局一局打，每局判胜负、更新记分板，最后公布总比分。

```python
# rps_game.py 石头剪刀布 完整版
import random

moves = ["石头", "剪刀", "布"]
score = {"玩家": 0, "电脑": 0}   # 记分板

total = int(input("想玩几局？请输入局数："))

round_no = 1  # 现在是第几局
while round_no <= total:
    print(f"\n第 {round_no} 局 —— 请出拳：")
    print("1) 石头  2) 剪刀  3) 布")
    choice = input("你的选择：")

    # 把玩家的数字选择变成拳
    if choice == "1":
        player = "石头"
    elif choice == "2":
        player = "剪刀"
    elif choice == "3":
        player = "布"
    else:
        print("输入无效，这一局算电脑赢！")
        player = None

    # 电脑随机出拳
    computer = random.choice(moves)

    if player is not None:
        print(f"你出【{player}】，电脑出【{computer}】。")
        # 判断胜负
        if player == computer:
            print("平局！这一局不计分。")
        elif (player == "石头" and computer == "剪刀") or \
             (player == "剪刀" and computer == "布") or \
             (player == "布" and computer == "石头"):
            print("你赢了这一局！")
            score["玩家"] += 1
        else:
            print("电脑赢了这一局！")
            score["电脑"] += 1
    else:
        # 玩家乱输入，直接判电脑赢
        score["电脑"] += 1

    print(f"当前比分  你 {score['玩家']} : {score['电脑']} 电脑")
    round_no += 1   # 进入下一局

# 所有局打完，公布总成绩
print("\n========== 比赛结束 ==========")
print(f"最终比分  你 {score['玩家']} : {score['电脑']} 电脑")
if score["玩家"] > score["电脑"]:
    print("🏆 恭喜你，你是最终赢家！")
elif score["玩家"] < score["电脑"]:
    print("电脑赢了这场比赛，再来一次挑战它吧！")
else:
    print("平分秋色，你和电脑打成平手！")
```

```
你赢了这一局！

第 3 局 —— 请出拳：
1) 石头  2) 剪刀  3) 布

```

拆开看这个游戏的结构：

- **准备阶段**：定义拳的列表 `moves`、记分板字典 `score`，问玩家要玩几局。
- **循环阶段**：`while round_no <= total` 让游戏打满指定局数。每一局都做三件事：收玩家出拳、电脑随机出拳、判胜负并记分。
- **收尾阶段**：循环结束后公布总比分，比较两边的分决定冠军。

一个真正的多局对战游戏，就这样成型了！

> 小贴士：字符串里的 `\n` 是一个**换行符**，会让输出空出一行，比如 `print(f"\n第 {round_no} 局")` 会先空一行再打印，让每局之间看着更清爽。

## 常见错误

**1. 字典的键写错或没加引号**

```python-norun
score = {"玩家": 0, "电脑": 0}
print(score[玩家])   # 玩家没加引号
```

报错：

```
NameError: name '玩家' is not defined
```

`玩家` 不加引号，会被当成一个变量名。修法：键是文字，要加引号——`score["玩家"]`。如果标签打错字（比如写成 `score["玩家们"]`），则会报 `KeyError`，表示字典里没这个标签。

**2. 忘了把局数用 `int()` 转成数字**

```python-norun
total = input("想玩几局？")   # 少了 int()
while round_no <= total:      # 数字和文字比大小
```

报错：

```
TypeError: '<=' not supported between instances of 'int' and 'str'
```

和第 7 章一样的坑：`input()` 拿到的是文字，比大小前要用 `int()` 转成整数——`total = int(input("想玩几局？"))`。

**3. 胜负条件里 `and` 和 `or` 用反了**

```python-norun
# 想表达"石头砸剪刀"，却把 and 写成了 or
if player == "石头" or computer == "剪刀":
    print("你赢了")
```

这样只要玩家出石头（不管电脑出什么）就判赢，全乱套了。记住：**`and` 是"两个都要成立"，`or` 是"有一个成立就行"。** 判断"石头砸剪刀"必须是"我出石头**并且**它出剪刀"，用 `and`。

**4. 循环里忘了让局数增加**

如果漏写了最后的 `round_no += 1`，`round_no` 永远是 1，`while` 条件永远成立，游戏会**卡在第一局无限循环**。这叫"死循环"。修法：确保循环里有一步让 `round_no` 变大，循环才能走到头。

## 动手挑战

在 `rps_game.py` 的基础上，把游戏做得更专业。

**★ 挑战一：统计平局数**
在记分板字典里再加一个键 `"平局"`，每次平局就给它加 1，最后一起公布：你赢了几局、输了几局、平了几局。

**★★ 挑战二：三局两胜**
不再固定局数，而是**谁先赢满 2 局谁就夺冠**，立刻结束比赛。（提示：把 `while` 的条件改成"两边都还没到 2 分"，或者在有人达到 2 分时用别的办法跳出循环。）

**★★★ 挑战三：升级成"石头剪刀布蜥蜴斯波克"**
这是《生活大爆炸》里的加强版，一共五种手势，规则更多。把 `moves` 扩成五个，重新设计胜负判断。想想：当选项变多，长长的 `if` 条件会不会很难写？能不能用字典记下"谁能赢谁"，让判断更简洁？（这是个真正的挑战，做出来你就很厉害了！）

## 本章小结

你完成了一个有记分板的完整对战游戏，还把前面学的知识全用上了：

- **`random.choice(列表)`** 从列表里随机抽一个，正好用来让电脑"随机出拳"。
- 用**列表**存放一组同类选项（三种拳），代码更整齐。
- 用**字典**当记分板，`{"玩家": 0, "电脑": 0}`，靠标签（键）存取每一方的分数，比零散的变量清楚。
- 判断胜负时，用 `and`（并且）组合条件、用 `or`（或者）合并多种获胜情况；先筛掉平局和获胜，剩下的就是失败。
- 用 **`while` 循环**打多局，别忘了让局数每次增加，否则会死循环。
- 游戏的通用结构：**准备 → 循环对战 → 收尾公布**。
