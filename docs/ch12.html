第 12 章 贪吃蛇大作战——你的第一个完整游戏

本章你会做出什么

上一章你让一个方块动了起来。这一章，你要做出一个完整的、能玩上瘾的经典游戏——贪吃蛇：一条小蛇在窗口里爬，你用方向键指挥它去吃食物，每吃一个就变长一节、加一分；一不小心撞到墙、或者撞到自己的身子，游戏就结束。这是你学编程以来第一个真正意义上的"大作"，做完它，你就正式入门游戏开发了。

成品预览：一条绿色的蛇在黑色画面里游走，追着红色的食物；左上角实时显示分数；撞墙或咬到自己时，屏幕中央弹出"游戏结束"。

│ 得分：5                     │
│        ●●●●●                │
│            ●        ◆       │
│                            │
└────────────────────────────┘
   ●=蛇身  ◆=食物  用方向键控制

重要！和上一章一样：本章代码网页上的"▶ 运行"按钮跑不了，因为游戏窗口浏览器里弹不出来。所有代码都要在自己电脑上运行，而且要先装好 pygame。装 pygame 的方法上一章讲过，就一句 pip install pygame（在终端里敲）；装 Python 和终端怎么用，看附录 A。

先回忆一下：游戏的心脏还是主循环

别紧张，贪吃蛇看着复杂，其实骨架和上一章那个"移动方块"一模一样。每一圈主循环，游戏还是在重复做那三件事：

  1. 处理输入：看你按了哪个方向键、有没有想关窗口。
  2. 更新状态：让蛇往前爬一步，检查它吃到食物没、撞墙没、咬到自己没。
  3. 画出画面：把蛇、食物、分数重新画一遍。

上一章的方块，位置是两个变量 x、y；这一章不一样的地方只有一个——蛇有很多节身子，一个变量存不下。那用什么存呢？你其实早就学过了：列表。

核心思想：用一个列表存整条蛇

我们不再让蛇满屏乱跑，而是把窗口划成一格一格的网格，就像方格纸。每一格 20 像素宽。蛇的每一节身子，正好占一格。

蛇是由好几节身子连成的一串，对不对？那就用一个列表（list），把每一节身子的格子坐标按顺序装进去。列表里第一个是蛇头，最后一个是蛇尾。

我们用格子坐标来记，不用像素。比如 [5, 3] 表示"从左数第 5 格、从上数第 3 格"。一条有 3 节的蛇，就长这样：

Python
# 一条 3 节长的蛇，蛇头在最前面（列表第一个）
snake = [[7, 5], [6, 5], [5, 5]]
#         蛇头     身子     蛇尾
图 12-1　蛇就是一个装着格子坐标的列表；把每个坐标画成一个绿方块，连起来就是一条蛇

想通了这一点，贪吃蛇就没什么可怕的了——它不过是一个列表 + 一些让列表变化的规则。下面我们一条一条把规则加上去。

蛇怎么"爬"？加一个头，去掉尾巴

这是整个游戏最巧妙的地方，一定要弄懂。蛇怎么往前爬一步呢？你可能想的是"让每一节都往前挪"——太麻烦了。有个特别聪明的办法：

在蛇前进的方向上，新添一个头；然后把最后的尾巴去掉。 就这两步，蛇看起来就整整齐齐地前进了一格，而且长度不变。

原来： 头 尾

① 前面加新头： 新头 变长了1节

② 去掉旧尾： 删掉 长度不变，向前挪一格

图 12-2　蛇每走一步：先在头前面插入一个新格子，再删掉尾巴那一格

用列表操作来写，就是这么两行。假设蛇正在往右走（横坐标每次 +1）：

Python
head = snake[0]                    # snake[0] 是当前的头
new_head = [head[0] + 1, head[1]]  # 往右一格：x 加 1，y 不变
snake.insert(0, new_head)          # 把新头插到列表最前面
snake.pop()                        # 删掉列表最后一个（尾巴）
  • snake[0]：列表的第一个元素，也就是蛇头。
  • snake.insert(0, new_head)：insert 是"插入"。insert(0, ...) 表示插到第 0 位（最前面）。这样新头就成了新的 snake[0]。
  • snake.pop()：pop 不带参数时，删掉并"吐出"列表最后一个元素——正好就是尾巴。

记住这个"加头去尾"，它是贪吃蛇的灵魂。等下你会看到：吃到食物时，只要不去尾，蛇就自然变长了。

方向控制：方向键改变前进方向，但不能掉头

上一章按住方向键方块才动。贪吃蛇不一样——蛇会自己一直往前爬，方向键只是改变它前进的方向。松开手它也不会停。

我们用两个变量 dx、dy 记住"当前每一步往哪个方向挪"：

  • 往右：dx = 1, dy = 0（x 加 1）
  • 往左：dx = -1, dy = 0
  • 往上：dx = 0, dy = -1（y 减 1，因为 y 向下增大）
  • 往下：dx = 0, dy = 1

按了方向键，就改这两个数。但有一条重要规则：蛇不能直接掉头。想想真实的蛇——它正往右爬，你不可能让它一瞬间往左，那样头会直接撞进自己的脖子。所以按左键时要先检查："我现在不是正往右吗？是的话就不理这个左键。"

Python
if event.key == pygame.K_LEFT and dx != 1:   # 想往左，但只要现在不是往右就行
    dx, dy = -1, 0

dx != 1 的意思是"现在不是在往右走"。只有满足这个条件才允许转向左。其他三个方向同理。

小贴士：这里我们用 pygame.KEYDOWN 事件（键被按下的那一下），而不是上一章的 get_pressed()。因为方向是"按一下就定下来"，不需要一直按着。

食物：随机出现，吃到就变长 + 加分

食物也是网格上的一个格子，位置用 random 随机决定。你在讲 random 那章学过 random.randint，它能给你一个范围内的随机整数：

Python
import random
food = [random.randint(0, 29), random.randint(0, 19)]  # 随机一个格子坐标

我们的网格宽 30 格（编号 0～29）、高 20 格（编号 0～19），所以随机范围就是这么写。

怎么算"吃到了"？ 特别简单——蛇的新头，正好落在食物那一格，就是吃到了：

Python
if snake[0] == food:      # 蛇头和食物在同一格
    score = score + 1     # 加分
    food = [random.randint(0, 29), random.randint(0, 19)]  # 生成新食物
    # 注意：吃到时【不做 pop()】，蛇就留下了那一节，自然变长

看到没？吃食物和普通爬行的唯一区别，就是吃到时不删尾巴。加了头又没去尾，蛇就长了一节。这就是前面埋的伏笔。

判定游戏结束：撞墙、撞到自己

蛇有两种死法，都在"更新状态"这一步检查蛇头 snake[0]：

1. 撞墙——蛇头跑出了网格范围（格子坐标小于 0，或大于等于边界）：

Python
head = snake[0]
if head[0] < 0 or head[0] >= 30 or head[1] < 0 or head[1] >= 20:
    game_over = True      # 出界了，游戏结束

2. 撞到自己——蛇头这一格，和身子的某一节重合了。判断"蛇头是否出现在身子里"，用 in 就行：

Python
if snake[0] in snake[1:]:   # 蛇头出现在【除了头以外】的身子里
    game_over = True

snake[1:] 是"从第 1 个到最后"，也就是除去蛇头的全部身子。in 判断蛇头那一格是不是在这堆身子里。是的话，就是咬到自己了。

显示分数：pygame 的文字

最后给游戏加个分数显示。在 pygame 里写字，要先做一个"字体"，再用它把文字"渲染"成图片，画到屏幕上：

Python
font = pygame.font.SysFont(None, 36)   # 做一个字体，None=系统默认字体，36=字号
text = font.render("得分：" + str(score), True, (255, 255, 255))  # 渲染成白色文字
screen.blit(text, (10, 10))            # 把文字画到屏幕左上角 (10,10)
  • pygame.font.SysFont(None, 36)：SysFont 用系统自带字体，None 表示用默认那款，36 是字号。
  • font.render(文字, True, 颜色)：把一段文字"画"成一张小图片。中间的 True 是"抗锯齿"，让字更平滑，照抄即可。
  • screen.blit(图片, 位置)：blit 是把一张图片"贴"到屏幕的指定位置。文字渲染出来就是张图片，所以用 blit 贴上去。

小贴士：str(score) 是把数字 score 变成文字，才能和"得分："拼在一起。数字和文字不能直接用 + 相加，这个坑你在前面章节踩过。

完整程序：贪吃蛇大作战

好了，所有零件都齐了，拼成完整的游戏！新建文件 snake.py，把下面的代码抄进去（或复制过去），在自己电脑上运行。慢慢读注释，你会发现每一块都是前面讲过的东西拼起来的。

Python
# snake.py
import pygame
import random

pygame.init()

# ===== 游戏的基本设定 =====
CELL = 20                    # 每个格子 20 像素
COLS = 30                    # 横向 30 格
ROWS = 20                    # 纵向 20 格
WIDTH = CELL * COLS          # 窗口宽 = 20 × 30 = 600
HEIGHT = CELL * ROWS         # 窗口高 = 20 × 20 = 400

screen = pygame.display.set_mode((WIDTH, HEIGHT))
pygame.display.set_caption("贪吃蛇大作战")
clock = pygame.time.Clock()
font = pygame.font.SysFont(None, 36)     # 显示分数用的字体

# ===== 颜色（红, 绿, 蓝，各 0~255）=====
BLACK = (0, 0, 0)
GREEN = (63, 155, 92)        # 蛇身
ORANGE = (224, 106, 59)      # 蛇头，突出一点
RED = (220, 60, 60)          # 食物
WHITE = (255, 255, 255)      # 文字


def random_food(snake):
    """随机生成一个不在蛇身上的食物格子"""
    while True:
        food = [random.randint(0, COLS - 1), random.randint(0, ROWS - 1)]
        if food not in snake:          # 别正好长在蛇身上
            return food


def run_game():
    # ===== 初始状态 =====
    # 蛇一开始 3 节，横躺在中间，头在右边
    snake = [[15, 10], [14, 10], [13, 10]]
    dx, dy = 1, 0                # 一开始往右走
    food = random_food(snake)
    score = 0
    game_over = False

    running = True
    while running:
        # ---- ① 处理输入 ----
        for event in pygame.event.get():
            if event.type == pygame.QUIT:            # 点了关闭按钮
                running = False
            elif event.type == pygame.KEYDOWN:       # 按下了某个键
                # 改变方向，但不能直接掉头
                if event.key == pygame.K_LEFT and dx != 1:
                    dx, dy = -1, 0
                elif event.key == pygame.K_RIGHT and dx != -1:
                    dx, dy = 1, 0
                elif event.key == pygame.K_UP and dy != 1:
                    dx, dy = 0, -1
                elif event.key == pygame.K_DOWN and dy != -1:
                    dx, dy = 0, 1

        if not game_over:
            # ---- ② 更新状态：让蛇走一步 ----
            head = snake[0]
            new_head = [head[0] + dx, head[1] + dy]  # 按方向算出新头的格子
            snake.insert(0, new_head)                # 加新头

            # 撞墙判定：新头跑出网格了吗
            if (new_head[0] < 0 or new_head[0] >= COLS or
                    new_head[1] < 0 or new_head[1] >= ROWS):
                game_over = True
            # 撞到自己判定：新头是否落在剩下的身子里
            elif new_head in snake[1:]:
                game_over = True
            elif new_head == food:
                # 吃到食物：加分、换新食物，【不去尾】所以蛇变长了
                score = score + 1
                food = random_food(snake)
            else:
                # 没吃到：去掉尾巴，长度不变（正常爬行）
                snake.pop()

        # ---- ③ 画出画面 ----
        screen.fill(BLACK)                           # 背景涂黑

        # 画食物（红方块）
        pygame.draw.rect(screen, RED,
                         (food[0] * CELL, food[1] * CELL, CELL, CELL))

        # 画蛇：列表里每一格都画成一个方块，头用橙色
        for i in range(len(snake)):
            part = snake[i]
            color = ORANGE if i == 0 else GREEN      # 第 0 个是头
            pygame.draw.rect(screen, color,
                             (part[0] * CELL, part[1] * CELL, CELL - 1, CELL - 1))
            # 宽高用 CELL-1，留一条缝，能看清一节一节

        # 画分数
        text = font.render("得分：" + str(score), True, WHITE)
        screen.blit(text, (10, 10))

        # 游戏结束时，屏幕中央显示提示
        if game_over:
            over_text = font.render("游戏结束！按关闭退出", True, WHITE)
            rect = over_text.get_rect(center=(WIDTH // 2, HEIGHT // 2))
            screen.blit(over_text, rect)

        pygame.display.flip()                        # 显示出来
        clock.tick(10)                               # 每秒 10 步：贪吃蛇要慢一点

    pygame.quit()


run_game()

在自己电脑上运行，一条绿蛇（橙色的头）就开始往右爬。按方向键指挥它去吃红色食物，每吃一个变长一节、左上角分数加一。撞到墙或咬到自己，中央就显示"游戏结束"。这是你独立做出的第一个完整游戏，去玩两把犒劳一下自己吧！

有两个小地方值得说一说：

  • 格子坐标 × CELL = 像素坐标：蛇和食物我们都用格子坐标（比如 [15, 10]）来算逻辑，又清楚又不容易错；只有真正画到屏幕上时，才乘以 CELL（20）换成像素。part[0] * CELL 就是这一格左上角的横坐标像素。
  • clock.tick(10)：上一章我们用 60，画面很流畅。但贪吃蛇每一圈就走一整格，60 太快根本没法玩。改成 10，蛇每秒走 10 步，刚刚好。想让游戏更刺激，把这个数调大就更快。

常见错误

贪吃蛇比前面的程序复杂，新手容易在这几处栽跟头：

1. 蛇一转向就"自杀"

如果你发现蛇往右走时按一下左键立刻就"游戏结束"，多半是漏掉了"不能掉头"的判断（那几个 and dx != 1 之类）。没有它，蛇头会一步退到自己脖子那一格，判定成"撞到自己"。把四个方向的条件都补齐就好。

2. 把 pop() 写在了 if 外面，蛇永远长不长

Python
snake.insert(0, new_head)
snake.pop()                # 错！这样不管吃没吃到都去尾
if new_head == food:
    score = score + 1

这样每一圈都去尾，蛇吃了食物也不会变长。正确做法是：只有"没吃到"时才 pop()（放进 else 里），像完整代码那样。

3. 直接把两条蛇身列表指向同一个

Python
new_head = head        # 错！这样 new_head 和 head 是同一个东西
new_head[0] = new_head[0] + dx

这会连原来的头也一起改掉，画面全乱。要新建一个列表 new_head = [head[0] + dx, head[1] + dy]，这才是一个全新的格子。

4. 数字和文字直接相加

Python
text = font.render("得分：" + score, True, WHITE)   # 错！score 是数字

报错 TypeError: can only concatenate str (not "int") to str。文字不能直接加数字，要先用 str(score) 把分数变成文字。

动手挑战

下面的任务都在你自己电脑上运行 snake.py 的基础上改。一步一步来，别急着一次全做完。

★ 挑战一：调节难度与配色 把 clock.tick(10) 里的 10 改成别的数，找出你觉得最好玩的速度。再把蛇、食物、背景换成你喜欢的颜色（记得颜色是 (红, 绿, 蓝) 三个 0～255 的数）。

★★ 挑战二：越吃越快 让游戏随分数变难：每吃到一个食物，蛇就走得快一点点。（提示：用一个变量 speed 代替 tick 里写死的 10，吃到食物时 speed = speed + 1，把 clock.tick(10) 改成 clock.tick(speed)。想想上限要不要设一个，别快到没法玩。）

★★★ 挑战三：加障碍物 / 重新开始 挑一个（或都做）：

  • 障碍物：在网格里放几块固定的"石头"（存成一个坐标列表），蛇头撞上它们也算游戏结束，记得把石头也画出来。
  • 重新开始：游戏结束后，按空格键就重开一局，不用关窗口重跑。（提示：run_game() 本身就是一局，想办法在游戏结束后侦测空格键，重置 snake、score 等变量再来一遍。）

这些都不简单，做出来任何一个，你都已经是能独立改游戏的人了。

本章小结

你做出了人生第一个完整游戏，这是一个了不起的里程碑！回顾这一章把学过的东西怎么拼到了一起：

  • 贪吃蛇的核心思想：用一个列表存整条蛇，列表里每个元素是一节身子的格子坐标，snake[0] 是头。
  • 蛇的移动 = 加头去尾：snake.insert(0, new_head) 加新头，snake.pop() 删尾巴；吃到食物时不去尾，蛇就变长了。
  • 方向用 dx、dy 两个数表示，靠 pygame.KEYDOWN 事件改变，并且要判断不能直接掉头。
  • 食物用 random.randint 随机放在网格上；蛇头 snake[0] 和食物同格就是吃到了。
  • 两种死法：蛇头出界（撞墙），或蛇头 in snake[1:]（撞到自己）。
  • 用 font.render + screen.blit 显示分数；clock.tick(10) 把贪吃蛇的速度放慢到能玩。
  • 格子坐标算逻辑，乘以 CELL 换成像素再画，这个分工让代码又清楚又不出错。
  • 老规矩：pygame 要在自己电脑上运行，网页运行按钮跑不了；没装先 pip install pygame。
