第 21 章 综合项目：从零实现一个 vector

这一章想让你带走什么

这是全书的收官。我们要从零实现一个能用的 SimpleVector<T>——一个动态数组。它会用上你这一路学到的几乎所有东西：模板、RAII、拷贝与移动、所有权、placement new、异常安全、noexcept。

做完你会明白：std::vector 不是魔法。第 4 章说它是"栈上管理堆资源的小管家"，这一章你将亲手把这个管家造出来，让全书的知识在一个真实组件里融会贯通。

目标与设计

我们要实现的 SimpleVector<T> 支持：动态增长（push_back/emplace_back）、下标访问、拷贝与移动、范围 for 遍历。核心数据结构就三个字段——这正是第 4 章图 4-2 画过的"小管家"：

图 21-1　SimpleVector 的结构：一个栈上的小对象（三个字段），管理一块堆上的元素数组
  • size_：当前元素个数。
  • capacity_：已分配的容量（≥ size_，留出增长空间）。
  • data_：指向堆上元素数组的指针。

一个关键设计点：容量和元素个数要分开。如果每加一个元素就重新分配，性能会很差。所以我们预留容量，满了才扩容（通常翻倍），把平均插入成本摊薄到 O(1)。

第一块拼图：内存与元素分开管理

这是自己实现容器最烧脑、也最能体现 C++ 内存模型的一点。普通的 new T[n] 会立即构造 n 个 T 对象，但我们要的是"先要一块够大的生内存，之后再逐个在里面构造元素"——分配和构造必须分离。

分配内存 ≠ 构造对象
  • ::operator new(bytes)：只分配一块原始字节，不构造任何对象（类似 C 的 malloc）。
  • placement new new (ptr) T(args...)：不分配内存，而是在已有的 ptr 位置上构造一个 T。
  • ptr->~T()：显式调用析构函数，销毁对象但不释放内存。
容器就是靠这三样，把"内存管理"和"对象生命周期"解耦——预留一大块内存，按需在其中构造/析构元素。这正是第 4 章"生命周期"和第 5 章"RAII"在库层面的底层实现。

逐步构建：骨架 + 析构

先搭骨架——三个字段、内存分配辅助函数、以及负责清理的析构函数。析构要做两件事：先逐个析构元素，再释放内存（顺序不能反）。

C++ 🔗 在 Compiler Explorer 打开
#include <iostream>
#include <new>
#include <cstddef>

template <typename T>
class SimpleVector {
    T* data_ = nullptr;
    std::size_t size_ = 0;
    std::size_t capacity_ = 0;

    // 只分配原始内存，不构造元素
    static T* allocate(std::size_t n) {
        return static_cast<T*>(::operator new(n * sizeof(T)));
    }
    static void deallocate(T* p) { ::operator delete(p); }

public:
    SimpleVector() = default;

    ~SimpleVector() {
        for (std::size_t i = 0; i < size_; ++i) data_[i].~T();  // 1) 逐个析构元素
        deallocate(data_);                                       // 2) 再释放内存
    }

    std::size_t size() const { return size_; }
    std::size_t capacity() const { return capacity_; }
};

int main() {
    SimpleVector<int> v;   // 空 vector，析构时什么也不用做（data_ 为 nullptr 也安全）
    std::cout << "size=" << v.size() << " cap=" << v.capacity() << '\n';
}
本地编译运行：g++ -std=c++23 -O2 -Wall main.cpp && ./a.out

输出 size=0 cap=0。这个析构函数就是 RAII 的核心——只要 SimpleVector 离开作用域，它管理的堆内存和所有元素都被自动清理。这正是它比裸 new[]/delete[] 安全的原因。

第二块拼图：扩容（reserve）

reserve 是容器的心脏——申请更大的新内存，把现有元素搬过去，释放旧内存。"搬"这个字暗示了第 7 章的移动语义：

C++
void reserve(std::size_t newCap) {
    if (newCap <= capacity_) return;               // 只增不减
    T* newData = allocate(newCap);                 // 1) 分配更大的新内存
    for (std::size_t i = 0; i < size_; ++i)
        new (newData + i) T(std::move_if_noexcept(data_[i]));  // 2) 把元素搬到新内存
    for (std::size_t i = 0; i < size_; ++i) data_[i].~T();     // 3) 析构旧元素
    deallocate(data_);                             // 4) 释放旧内存
    data_ = newData;                               // 5) 接管新内存
    capacity_ = newCap;
}
为什么是 move_if_noexcept 而不是 move？ 搬移元素时，我们想用移动（快）而非拷贝。但有个异常安全的考量：如果搬到一半某个元素的移动构造抛了异常，旧数据可能已经被破坏、无法回滚。
std::move_if_noexcept 的策略是：如果 T 的移动构造是 noexcept 的（保证不抛），就移动；否则退而拷贝（拷贝失败时旧数据还完好，能安全回滚）。这就是第 17 章说的"移动构造标 noexcept 才能让容器扩容用移动"的底层机制——你现在看到它在库里是怎么用的了。

第三块拼图：push_back 与 emplace_back

有了 reserve，添加元素就简单了：容量满了就翻倍扩容，然后在末尾用 placement new 构造新元素。emplace_back 用完美转发（第 11、16 章的可变参数模板）直接在容器内原地构造，push_back 只是它的两个便捷包装：

C++
void push_back(const T& value) { emplace_back(value); }        // 拷贝进来
void push_back(T&& value) { emplace_back(std::move(value)); }  // 移动进来

template <typename... Args>
T& emplace_back(Args&&... args) {
    if (size_ == capacity_)
        reserve(capacity_ == 0 ? 1 : capacity_ * 2);  // 满了：翻倍（0 则起步为 1）
    new (data_ + size_) T(std::forward<Args>(args)...);  // 原地构造
    return data_[size_++];
}

std::forward<Args>(args)... 把参数原样（保持左值/右值属性，第 6 章值类别）转发给 T 的构造函数——这样 emplace_back 既能转发左值触发拷贝、也能转发右值触发移动，还能直接传构造参数原地构造，一份代码通吃。

亲眼观察扩容时发生了什么

把上面的部件组装起来，用第 10 章那种"会报告生死"的探针类型，观察 emplace_back 触发扩容时的移动与析构。这是理解容器行为最直观的方式：

C++ 🔗 在 Compiler Explorer 打开
#include <iostream>
#include <new>
#include <utility>
#include <cstddef>

template <typename T>
class SimpleVector {
    T* data_ = nullptr;
    std::size_t size_ = 0, capacity_ = 0;
    static T* allocate(std::size_t n) { return static_cast<T*>(::operator new(n * sizeof(T))); }
    static void deallocate(T* p) { ::operator delete(p); }
public:
    SimpleVector() = default;
    ~SimpleVector() { for (std::size_t i=0;i<size_;++i) data_[i].~T(); deallocate(data_); }
    void reserve(std::size_t nc) {
        if (nc <= capacity_) return;
        T* nd = allocate(nc);
        for (std::size_t i=0;i<size_;++i) new (nd+i) T(std::move_if_noexcept(data_[i]));
        for (std::size_t i=0;i<size_;++i) data_[i].~T();
        deallocate(data_); data_ = nd; capacity_ = nc;
    }
    template <typename... Args>
    void emplace_back(Args&&... args) {
        if (size_ == capacity_) reserve(capacity_ == 0 ? 1 : capacity_ * 2);
        new (data_ + size_) T(std::forward<Args>(args)...); ++size_;
    }
    std::size_t size() const { return size_; }
    std::size_t capacity() const { return capacity_; }
};

struct Probe {
    int id;
    Probe(int i) : id(i) { std::cout << "  构造 #" << id << '\n'; }
    Probe(const Probe& o) : id(o.id) { std::cout << "  拷贝 #" << id << '\n'; }
    Probe(Probe&& o) noexcept : id(o.id) { std::cout << "  移动 #" << id << '\n'; }
    ~Probe() { std::cout << "  析构 #" << id << '\n'; }
};

int main() {
    SimpleVector<Probe> v;
    for (int i = 1; i <= 3; ++i) {
        std::cout << "emplace_back(#" << i << ")  [size=" << v.size()
                  << " cap=" << v.capacity() << "]\n";
        v.emplace_back(i);
    }
    std::cout << "结束，逐个析构：\n";
}
本地编译运行：g++ -std=c++23 -O2 -Wall main.cpp && ./a.out

运行输出：

emplace_back(#1)  [size=0 cap=0]
  构造 #1
emplace_back(#2)  [size=1 cap=1]
  移动 #1
  析构 #1
  构造 #2
emplace_back(#3)  [size=2 cap=2]
  移动 #1
  移动 #2
  析构 #1
  析构 #2
  构造 #3
结束，逐个析构：
  析构 #1
  析构 #2
  析构 #3

逐行读这个输出，容器的行为一览无余：

  • 加 #1 时容量 0→1，构造 #1；
  • 加 #2 时容量 1 满了→扩到 2，把 #1 移动到新内存、析构旧的 #1，再构造 #2；
  • 加 #3 时容量 2 满了→扩到 4，把 #1、#2 移动过去、析构旧的，再构造 #3。

因为 Probe 的移动构造标了 noexcept，move_if_noexcept 选择了移动而非拷贝——这就是第 17 章那条规则的实际后果。扩容有搬移成本，这也解释了为什么第 4、14 章反复建议：预先知道大小就先 reserve，避免反复扩容。

第四块拼图：拷贝与移动（Rule of Five）

因为 SimpleVector 直接管理裸资源（data_ 指针），它落入第 10 章的 Rule of Five——必须正确实现拷贝和移动，否则默认的浅拷贝会导致双重释放（第 7 章的灾难）。

  • 拷贝构造：分配自己的新内存，深拷贝每个元素；
  • 移动构造：接管对方的指针，把对方置空；
  • 赋值：用一个漂亮的技巧 copy-and-swap 同时搞定拷贝赋值和移动赋值。
C++
// 拷贝构造：深拷贝
SimpleVector(const SimpleVector& other)
    : data_(allocate(other.capacity_)), size_(0), capacity_(other.capacity_) {
    try {
        for (; size_ < other.size_; ++size_)
            new (data_ + size_) T(other.data_[size_]);  // 逐个拷贝构造
    } catch (...) {                    // 异常安全：构造到一半失败要清理已建的
        for (std::size_t i = 0; i < size_; ++i) data_[i].~T();
        deallocate(data_);
        throw;
    }
}

// 移动构造：接管资源，置空对方（noexcept！）
SimpleVector(SimpleVector&& other) noexcept
    : data_(other.data_), size_(other.size_), capacity_(other.capacity_) {
    other.data_ = nullptr;
    other.size_ = other.capacity_ = 0;
}

// copy-and-swap：一个赋值运算符同时处理拷贝赋值和移动赋值
SimpleVector& operator=(SimpleVector other) noexcept {  // 按值传参！
    swap(other);
    return *this;
}

void swap(SimpleVector& other) noexcept {
    std::swap(data_, other.data_);
    std::swap(size_, other.size_);
    std::swap(capacity_, other.capacity_);
}
copy-and-swap：一石二鸟的赋值 注意 operator= 的参数是按值传的 SimpleVector other。这意味着：
  • 赋值来源是左值时，按值传参触发拷贝构造，other 是一份副本；
  • 来源是右值时，触发移动构造，other 廉价接管。
然后只需把 *this 和这份 other 的内容 swap 一下——旧内容随 other 析构时自动清理。一个函数同时正确处理了拷贝赋值和移动赋值，还天然异常安全（副本先建好才交换）。这是 C++ 里非常经典的惯用法，把第 6、7 章的值类别与移动语义用到了极致。

完整实现与试用

把五块拼图（骨架、reserve、emplace_back、拷贝/移动、访问接口）全部组装，再加上下标访问、at（带越界检查）、以及让范围 for 能工作的 begin/end（第 14 章：迭代器就是指针的泛化，对连续内存直接用 T* 即可）：

C++ 🔗 在 Compiler Explorer 打开
#include <iostream>
#include <new>
#include <utility>
#include <stdexcept>
#include <cstddef>

template <typename T>
class SimpleVector {
    T* data_ = nullptr;
    std::size_t size_ = 0;
    std::size_t capacity_ = 0;

    static T* allocate(std::size_t n) { return static_cast<T*>(::operator new(n * sizeof(T))); }
    static void deallocate(T* p) { ::operator delete(p); }

public:
    SimpleVector() = default;

    ~SimpleVector() {
        for (std::size_t i = 0; i < size_; ++i) data_[i].~T();
        deallocate(data_);
    }

    SimpleVector(const SimpleVector& other)
        : data_(allocate(other.capacity_)), size_(0), capacity_(other.capacity_) {
        try {
            for (; size_ < other.size_; ++size_)
                new (data_ + size_) T(other.data_[size_]);
        } catch (...) {
            for (std::size_t i = 0; i < size_; ++i) data_[i].~T();
            deallocate(data_);
            throw;
        }
    }

    SimpleVector(SimpleVector&& other) noexcept
        : data_(other.data_), size_(other.size_), capacity_(other.capacity_) {
        other.data_ = nullptr;
        other.size_ = other.capacity_ = 0;
    }

    SimpleVector& operator=(SimpleVector other) noexcept { swap(other); return *this; }

    void swap(SimpleVector& other) noexcept {
        std::swap(data_, other.data_);
        std::swap(size_, other.size_);
        std::swap(capacity_, other.capacity_);
    }

    void reserve(std::size_t newCap) {
        if (newCap <= capacity_) return;
        T* newData = allocate(newCap);
        std::size_t i = 0;
        try {
            for (; i < size_; ++i) new (newData + i) T(std::move_if_noexcept(data_[i]));
        } catch (...) {
            for (std::size_t j = 0; j < i; ++j) newData[j].~T();
            deallocate(newData);
            throw;
        }
        for (std::size_t j = 0; j < size_; ++j) data_[j].~T();
        deallocate(data_);
        data_ = newData;
        capacity_ = newCap;
    }

    template <typename... Args>
    T& emplace_back(Args&&... args) {
        if (size_ == capacity_) reserve(capacity_ == 0 ? 1 : capacity_ * 2);
        new (data_ + size_) T(std::forward<Args>(args)...);
        return data_[size_++];
    }
    void push_back(const T& v) { emplace_back(v); }
    void push_back(T&& v) { emplace_back(std::move(v)); }
    void pop_back() { data_[--size_].~T(); }

    T& operator[](std::size_t i) { return data_[i]; }
    const T& operator[](std::size_t i) const { return data_[i]; }
    T& at(std::size_t i) {
        if (i >= size_) throw std::out_of_range("索引越界");
        return data_[i];
    }

    std::size_t size() const { return size_; }
    std::size_t capacity() const { return capacity_; }
    bool empty() const { return size_ == 0; }

    T* begin() { return data_; }              // 迭代器即指针（连续内存）
    T* end() { return data_ + size_; }
    const T* begin() const { return data_; }
    const T* end() const { return data_ + size_; }
};

int main() {
    SimpleVector<int> v;
    for (int i = 1; i <= 5; ++i) v.push_back(i * 10);

    std::cout << "size=" << v.size() << " cap=" << v.capacity() << '\n';
    for (int x : v) std::cout << x << ' ';         // 范围 for（靠 begin/end）
    std::cout << '\n';

    SimpleVector<int> copy = v;                    // 拷贝构造：深拷贝
    copy[0] = 999;
    std::cout << "改 copy 后：v[0]=" << v[0] << " copy[0]=" << copy[0] << '\n';

    SimpleVector<int> moved = std::move(v);        // 移动构造：接管
    std::cout << "移动后：moved.size=" << moved.size() << " v.size=" << v.size() << '\n';

    moved.at(2) = 333;                             // 带越界检查的访问
    std::cout << "moved: ";
    for (int x : moved) std::cout << x << ' ';
    std::cout << '\n';
}
本地编译运行：g++ -std=c++23 -O2 -Wall main.cpp && ./a.out

运行输出：

size=5 cap=8
10 20 30 40 50 
改 copy 后：v[0]=10 copy[0]=999
移动后：moved.size=5 v.size=0
moved: 10 20 333 40 50

逐条印证了全书的核心概念：容量按 1→2→4→8 翻倍增长；范围 for 靠 begin/end 工作；copy = v 是深拷贝（改 copy 不动 v）；std::move(v) 后 v 被清空（size 变 0，所有权转移给 moved）。你亲手造出了一个 vector。

回望：这个 vector 用到了全书什么

一个容器，串起整本书
  • 模板（第 11 章）：SimpleVector<T> 对任意元素类型工作。
  • RAII（第 5 章）：析构函数自动清理内存和元素，无需手动 delete。
  • 值语义与深拷贝（第 3 章）：拷贝得到独立副本。
  • 移动语义与所有权（第 6、7 章）：移动转移资源、置空源；move_if_noexcept 择优搬移。
  • Rule of Five / copy-and-swap（第 10 章）：管理裸资源时正确实现五个特殊成员函数。
  • 完美转发（第 11、16 章）：emplace_back 原地构造。
  • 异常安全 / noexcept（第 17 章）：构造失败清理、noexcept 影响搬移策略。
  • 迭代器（第 14 章）：begin/end 返回指针，接入范围 for 与标准算法。
真正的 std::vector 还处理了分配器、异常保证的更多细节、iterator 类型等，但骨架就是你手上这个。它不再是黑盒。

这一章的要点

  • std::vector 的核心是三个字段（data_ / size_ / capacity_）：一个栈上小对象管理一块堆数组，容量与元素数分开以摊薄插入成本。
  • 自己实现容器要分离内存分配与对象构造：::operator new 分配生内存，placement new 就地构造，显式 ~T() 析构。
  • reserve 是心脏：分配更大内存、搬移元素、清理旧内存；用 move_if_noexcept 在移动与拷贝间择优（异常安全）。
  • 直接管理裸资源 → Rule of Five：正确实现拷贝/移动，copy-and-swap 用一个按值参数的 operator= 同时搞定拷贝赋值与移动赋值。
  • emplace_back 靠完美转发原地构造；begin/end 返回指针即成迭代器。
  • 这个 vector 串起了全书：模板、RAII、值语义、移动、所有权、异常安全、noexcept、迭代器——std::vector 不再是魔法。
