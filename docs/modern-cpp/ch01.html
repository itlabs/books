第 1 章 你已经会编程了——那 C++ 到底不一样在哪

这一章想让你带走什么

你会写 Java 或 Python，理解类、泛型、异常、并发。这一章不教你任何语法，而是先讲清一件事：当你从托管语言走进 C++，真正改变的不是关键字，而是"变量是什么"这个最底层的假设。

读完这一章，你会明白后面每一章为什么这么安排，也会知道该带着哪几个问题往下读。

先说结论：变量的含义变了

在 Java 和 Python 里，你写了无数遍这样的代码：

java
Point a = new Point(1, 2);
Point b = a;
b.x = 99;
System.out.println(a.x);   // 99

你早就知道结果是 99，而且从不觉得奇怪：a 和 b 是同一个对象的两个名字。变量装的不是对象，而是指向对象的引用。对象活在堆上，由垃圾回收器决定什么时候销毁。

现在看 C++ 里"长得几乎一样"的代码：

C++ 🔗 在 Compiler Explorer 打开
#include <iostream>
struct Point { int x, y; };

int main() {
    Point a{1, 2};
    Point b = a;      // 注意：这里发生了什么？
    b.x = 99;
    std::cout << a.x << '\n';   // 输出 1，不是 99
}
本地编译运行：g++ -std=c++23 -O2 -Wall main.cpp && ./a.out

点开上面的 🔗 在 Compiler Explorer 打开 亲自跑一遍——结果是 1。

差别不在语法（两段代码几乎一模一样），而在赋值这个动作的含义：

Java / Python 的心智模型
java
Point b = a;
// b 和 a 指向同一个对象
// 复制的是"引用"（一个地址）
C++ 的心智模型
C++
Point b = a;
// b 是一个全新的对象
// a 的内容被逐字节拷贝进 b
本书第一个、也是最重要的一个观念 在 Java / Python 里，变量默认是对象的引用；在 C++ 里，变量默认就是对象本身。b = a 不是"让 b 也指向 a"，而是"造一个新对象，把 a 复制过来"。这叫值语义（value semantics）。
图 1-1　同一句 b = a，两种世界观：托管语言里 a、b 共享一个对象；C++ 里 b 是 a 的独立拷贝

这一个差别，会像多米诺骨牌一样推倒你对很多事情的默认预期：函数传参会不会改动原对象？两个变量会不会互相影响？对象什么时候被销毁？拷贝一个大容器贵不贵？——在 C++ 里，答案常常和你的直觉相反。第 3 章会把值语义彻底讲透，它是全书的地基。

第二个大不同：没有 GC，但也几乎不用手动释放

托管语言的另一根支柱是垃圾回收。你 new 一个对象，然后……就不管了。GC 会在某个你不关心的时刻回收它。

C++ 没有垃圾回收器。但请先放下你可能听过的那个刻板印象——"写 C++ 就是到处 new / delete，一不小心就内存泄漏"。那是上世纪的 C++。

现代 C++ 的做法是把资源的生命周期绑定到对象的生命周期上：对象诞生时获取资源，对象销毁时自动释放。这套机制叫 RAII，是 C++ 最核心的设计思想，没有之一。

C++
#include <vector>
#include <string>

void demo() {
    std::vector<std::string> names;   // 在栈上诞生
    names.push_back("Alice");         // 内部在堆上分配内存
    names.push_back("Bob");
}   // demo 结束：names 被销毁，它持有的堆内存被自动释放
    // 没有 GC，也没有一句 delete

names 是一个栈上的局部变量。当 demo() 返回、names 离开作用域时，它的析构函数自动运行，把内部在堆上分配的内存还回去。你没写 delete，也不会泄漏。

类比：这其实很像你熟悉的东西 Java 的 try-with-resources、Python 的 with 语句，都是"离开作用域自动清理"的思路。区别在于：在那些语言里，这是你要特意为资源（文件、锁）加的语法；而在 C++ 里，这是每一个对象的默认行为。第 5 章专门讲 RAII。

第三个大不同：很多事情发生在"运行之前"

Java 和 Python 也要编译（.class 字节码、.pyc），但你通常把它当成一个黑箱：源码进去，能跑的东西出来。运行时的 JIT、解释器才是主角。

C++ 把大量工作前移到了编译期。编译器不只是翻译代码，它还会：

  • 把泛型（模板）在编译期实例化成一份份具体类型的代码；
  • 在编译期就算出一些常量的值（constexpr）；
  • 做激进的优化：内联、消除拷贝、把循环展开甚至整段算掉。

这意味着：读懂 C++，很多时候要读懂编译器对你的代码做了什么。这也是为什么本书几乎每段代码都给了一个 🔬 看汇编 / 编译器处理 的入口。举个例子——下面这个函数，编译器会直接算出结果，运行时什么都不算：

C++ 🔬 看汇编 / 编译器处理
constexpr int square(int x) { return x * x; }

int main() {
    return square(9);   // 编译器直接生成 "return 81"
}
本地编译运行：g++ -std=c++23 -O2 -Wall main.cpp && ./a.out

点开"看汇编"，你会看到 main 里根本没有乘法指令，只有一个 81。这种"代码在编译期就跑完了"的能力，是 Java / Python 给不了你的。第 13 章（编译期计算）和第 19 章（编译器优化）会带你深入。

那 C++ 换来了什么？

既然要放弃 GC 的省心、接受值语义的"反直觉"，图什么？

  • 可预测、可控的性能：没有 GC 停顿，对象在栈上还是堆上、拷贝还是移动，都由你精确决定。
  • 零成本抽象（zero-overhead abstraction）：高层抽象（模板、RAII、range）编译后往往和手写底层代码一样快。你不必在"写得优雅"和"跑得快"之间二选一。
  • 确定性的资源管理：对象销毁的时刻是确定的，所以文件、锁、连接的释放时机也是确定的——不用等 GC 心情好。

这也是为什么在游戏引擎、数据库、高频交易、操作系统、编译器本身这些地方，C++ 至今不可替代。

这本书怎么帮你完成切换

你不缺编程能力，缺的是一套新的心智模型。所以本书的策略始终是对照：先用一段你熟悉的 Java / Python 建立锚点，再看 C++ 怎么做、为什么这么做。

全书的路线：

  1. 换心智模型（第 1–2 章）：先建立值语义的直觉，再看清"源码怎么变成能跑的程序"。
  2. 对象、内存与生命周期（第 3–10 章，核心）：值语义、栈与堆、RAII、值类别、拷贝与移动、指针与所有权、const、类。这一部分立住了，C++ 就不再神秘。
  3. 泛型与编译期（第 11–13 章）：模板、concepts、constexpr——C++ 泛型和 Java 泛型是两个世界。
  4. 标准库的现代用法（第 14–17 章）：容器、算法与 range、lambda、错误处理。
  5. 深入运行时与编译器（第 18–21 章）：多态与 vtable、编译器优化、并发与内存模型，最后从零实现一个 vector 把所有知识串起来。
本书默认现代 C++（C++20 / C++23） 如果你以前翻过老的 C++ 教程，看到过 new/delete 满天飞、裸指针到处传的写法——请把那些暂时忘掉。本书教的是今天专业团队实际在写的 C++：以值和 RAII 为默认，裸 new/delete 几乎不出现。

动手：把心智模型的差别跑出来

光看不够。下面这段程序把本章的第一个观念做成了可运行的实验。点开 🔗 在 Compiler Explorer 打开，先猜输出，再运行对答案：

C++ 🔗 在 Compiler Explorer 打开
#include <iostream>
#include <vector>

struct Box { int value; };

void tryToModify(Box b) {   // 注意：按值传参
    b.value = 999;          // 改的是一份拷贝
}

int main() {
    Box original{1};
    Box copy = original;    // 值语义：copy 是独立的新对象
    copy.value = 42;

    tryToModify(original);  // original 会被改吗？

    std::cout << "original.value = " << original.value << '\n';
    std::cout << "copy.value     = " << copy.value << '\n';
}
本地编译运行：g++ -std=c++23 -O2 -Wall main.cpp && ./a.out

如果你带着 Java / Python 的直觉，可能会预期 original 被 tryToModify 改成了 999。但输出是：

original.value = 1
copy.value     = 42

copy = original 造了独立的新对象，所以改 copy 不影响 original；tryToModify 按值接收参数，改的也只是一份拷贝。在 C++ 里，"复制"是默认行为，"共享"才需要你特意用引用或指针去要求——这和你熟悉的语言正好相反。

搞清楚了这一点，我们就可以进入下一章：先看清一段 C++ 源码是怎么变成一个能运行的程序的。这不只是背景知识——理解编译与链接模型，能帮你读懂 C++ 里一大半令人困惑的报错。

这一章的要点

  • C++ 变量默认是对象本身（值语义），不是引用；b = a 是拷贝，不是共享。
  • 没有 GC，但通过 RAII 把资源生命周期绑定到对象生命周期，几乎不用手动释放。
  • 大量工作发生在编译期：模板实例化、常量计算、激进优化——读 C++ 常常要读懂编译器做了什么。
  • 换来的是可预测的性能、零成本抽象和确定性的资源管理。
  • 本书用对照的方式，帮你把 Java / Python 的心智模型平滑切换到 C++。
