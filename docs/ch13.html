第 13 章 AI 是什么？让程序连上大脑

本章你会做出什么

从这一章开始，你的程序要变"聪明"了。你会让自己写的 Python 程序连上一个真正的 AI（人工智能），然后问它一个问题、收到它的回答——就像给程序装了一个会思考的大脑。

成品预览：

你问 AI：用一句话介绍一下你自己。
AI 回答：你好！我是一个由 Anthropic 训练的 AI 助手，很高兴认识你，随时可以帮你解答问题、一起写代码或聊聊天。

前面 12 章，你已经会用 Python 做出好几个游戏了，包括一个完整的贪吃蛇。这一部分，我们要解锁一个全新的超能力：调用 AI。

AI 到底是什么？

你可能听过"人工智能""大模型""ChatGPT""Claude"这些词。简单来说：

AI（Artificial Intelligence，人工智能） 是一种能"读懂"人类语言、并用人类语言回答的程序。我们这本书用到的是一种叫大语言模型（Large Language Model，简称 LLM） 的 AI。你可以把它想象成一个读过海量书籍和文章的"超级学霸"——你用文字问它，它就用文字答你。

它能做的事非常多：回答问题、写文章、翻译、讲笑话、帮你检查代码、陪你玩游戏……而最神奇的地方是：这个"大脑"不在你的电脑里，它运行在很远的一台超级强大的服务器上。你的程序只要通过网络"打个电话"过去，把问题送过去，再把答案收回来就行了。

图 12-1　你的程序通过互联网向远方的 AI 大脑一问一答

这种"打电话过去用它的能力"的方式，有个专门的名字，叫 API（应用程序接口）。你不用懂 AI 内部是怎么工作的，就像你用微波炉不用懂里面的电路——你只要会按按钮（调用 API）就行。

我们要用的 AI：Claude

这本书里，我们用的 AI 叫 Claude（读作"克劳德"），它是一家叫 Anthropic 的公司做的。Claude 很擅长聊天、写代码、讲道理，而且用起来对新手很友好。

要让程序用上 Claude，需要三样东西：

  1. 一个"钥匙"：叫 API 密钥（API key），是一长串字母数字。它证明"是你在打这个电话"。
  2. 一个工具包：一个叫 anthropic 的 Python 库，帮你把打电话的复杂细节都处理好。
  3. 联网：因为 AI 在远方，运行时电脑必须能上网。

🔑 超级重要的安全提醒（请一定读） API 密钥就像你家的钥匙，或者你的银行卡密码——它是秘密，只能你自己知道。

  • 绝对不要把密钥直接写在代码里，然后把代码发给别人、发到群里、传到网上（比如 GitHub）。
  • 绝对不要把密钥截图分享。
  • 密钥可能和花钱有关（见下一节），泄露了可能被别人拿去乱用。 本章后面会教你一种安全的做法：把密钥存在电脑的"环境变量"里，而不是写进代码。

关于花钱：请和家长或老师一起

调用 Claude 的 API 通常不是免费的——用得越多，费用越高（不过每次问答其实很便宜，常常是几分钱）。Anthropic 一般会给新账号一点免费额度试用。

因为涉及注册账号和可能的费用，申请密钥这一步，请一定和你的家长或老师一起完成。让他们帮你：

  1. 打开 Anthropic 的官网（https://console.anthropic.com），注册一个账号。
  2. 在控制台里找到 "API Keys"，创建一个新密钥。
  3. 把那串密钥复制并妥善保存（它通常只完整显示一次）。

💡 小贴士：如果暂时拿不到密钥，也没关系。你可以先把这几章当故事读，理解 AI 编程是怎么回事；等条件具备了再动手。代码原理是一样的。

安装工具包

拿到密钥后，先装好 anthropic 工具包。打开终端（怎么打开见附录 A），输入：

终端
pip install anthropic

看到类似 Successfully installed anthropic-... 就装好了。

把密钥安全地告诉程序

我们不把密钥写进代码，而是存进电脑的环境变量（environment variable）——你可以理解成"电脑的一个秘密抽屉"，程序能悄悄打开它拿钥匙，但钥匙不会出现在代码里。

Windows（命令提示符），运行程序前先输入（把后面那串换成你自己的密钥）：

终端
set ANTHROPIC_API_KEY=你的密钥粘贴到这里

Mac / Linux（终端）：

终端
export ANTHROPIC_API_KEY=你的密钥粘贴到这里

💡 小贴士：这样设置的密钥只在当前这个终端窗口有效，关掉窗口就没了。这其实更安全。每次要运行 AI 程序，就在同一个窗口里先设置、再运行。

第一个 AI 程序：问它一个问题

一切就绪，见证奇迹的时刻到了。新建 first_ai.py：

Python
# first_ai.py
import anthropic   # 导入刚装好的工具包

# 创建一个"客户端"，它会自动从环境变量里读取你的密钥
client = anthropic.Anthropic()

# 向 Claude 发一条消息
response = client.messages.create(
    model="claude-haiku-4-5",       # 用哪个 AI 型号（haiku 又快又便宜，适合练手）
    max_tokens=200,                 # 最多让它回复多长（token 约等于字数上限）
    messages=[
        {"role": "user", "content": "用一句话介绍一下你自己。"}
    ],
)

# 回复藏在 response.content 里，把文字取出来打印
for block in response.content:
    if block.type == "text":
        print("AI 回答：", block.text)

设置好密钥后运行 python first_ai.py，你会看到 Claude 亲口（亲"字"）回答你！

运行结果（每次可能略有不同，这很正常——AI 的回答不是固定的）：

AI 回答： 你好！我是 Claude，一个由 Anthropic 打造的 AI 助手，很高兴能帮你答疑解惑、一起写代码或聊天。

拆开看看这段代码

这段代码有几个新面孔，我们一个个认识：

  • import anthropic：把工具包请进来，就像玩游戏前先加载模组。
  • client = anthropic.Anthropic()：创建一个"客户端"，理解成"你手里的那部电话"。括号里什么都没写，它就会自动去环境变量的秘密抽屉里找密钥。
  • client.messages.create(...)：真正"拨号"的一步。里面几个关键设置：
    • model：选哪个 AI。"claude-haiku-4-5" 是又快又便宜的型号，最适合练习。等你想要更聪明的回答，可以换成更强的型号（比如 "claude-opus-4-8"），但它更贵、更慢。
    • max_tokens：限制回答的最大长度。token（词元） 是 AI 数长度的单位，你先粗略理解成"字数上限"。设 200 就够短问答用了。
    • messages：你要说的话。它是一个列表（还记得第 6 章的列表吗？），里面每条消息是一个字典，"role" 是谁说的（"user" 就是"我，用户"），"content" 是说的内容。
  • response：AI 的完整回复，是个有点复杂的对象。
  • for block in response.content:：AI 的回复内容放在 response.content 里，可能分成好几"块"。我们遍历它，只挑出文字块（block.type == "text"），打印它的 .text。

💡 小贴士：为什么要 for 循环 + if 判断才能拿到文字？因为 AI 的回复以后可能不只有文字（还可能有别的东西）。现在你只要记住这个"取文字"的固定写法就好：遍历 response.content，遇到 type == "text" 的就取它的 .text。

动手改一改

把 messages 里 content 的那句话，换成你想问的任何问题，运行看看：

Python
# ask_ai.py
import anthropic

client = anthropic.Anthropic()

response = client.messages.create(
    model="claude-haiku-4-5",
    max_tokens=300,
    messages=[
        {"role": "user", "content": "给我讲一个关于程序员的冷笑话。"}
    ],
)

for block in response.content:
    if block.type == "text":
        print(block.text)

试试问它："帮我把'早上好'翻译成 5 种语言"、"用小学生能懂的话解释什么是黑洞"、"给我的小猫起 10 个可爱的名字"……你会发现，同一段代码，换一句话，就能干完全不同的事。这就是 AI 编程的魔力：程序的能力，来自你和 AI 的对话。

常见错误

1. 没设置密钥就运行

报错大概长这样：

AuthenticationError: 缺少 API 密钥 / could not resolve authentication

意思是程序没找到钥匙。检查：你是不是忘了在同一个终端窗口里先 set / export 密钥？密钥有没有粘贴完整？

2. 没装工具包

ModuleNotFoundError: No module named 'anthropic'

ModuleNotFoundError（找不到模块）表示 anthropic 没装上。回去运行 pip install anthropic。

3. 没联网 / 网络不通

APIConnectionError: 连接失败 / Connection error

AI 在远方，运行时必须联网。检查一下网络。如果在校园网或用了某些网络，可能需要请老师帮忙。

4. 模型名字写错

NotFoundError: model: ...

型号名字要一字不差，比如 "claude-haiku-4-5"。抄的时候别漏字符。

动手挑战

★ 挑战一：你的 AI 百科 写一个程序，问 AI 一个你真正好奇的问题（比如"为什么天是蓝色的"），把答案打印出来。

★★ 挑战二：换个型号比一比 把同一个问题分别用 claude-haiku-4-5 问一次。然后（如果家长同意花一点点钱）把 model 改成 "claude-opus-4-8" 再问一次，比较两次回答有什么不同。想一想：更强的型号回答得更好吗？

★★★ 挑战三：控制回答长度 把 max_tokens 从 200 改成 20，再改成 500，分别运行同一个问题。观察回答长度的变化。想一想 max_tokens 太小会发生什么？（提示：回答可能会被"说到一半就断掉"。）

本章小结

你的程序第一次连上了 AI 的大脑，这是一个巨大的里程碑！回顾一下：

  • AI / 大语言模型是能读懂人话、用人话回答的程序，它运行在远方的服务器上。
  • 我们通过 API 来"打电话"使用它，用的是 Claude 这个 AI。
  • 需要三样东西：API 密钥（秘密！）、anthropic 工具包、联网。
  • 密钥是秘密，绝不写进代码、绝不外传；我们把它存进环境变量这个秘密抽屉。
  • 申请密钥和花费问题，一定要和家长或老师一起。
  • 核心三步：client = anthropic.Anthropic() → client.messages.create(model=..., max_tokens=..., messages=[...]) → 遍历 response.content 取出 text。
