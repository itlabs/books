第 16 章 AI 故事工厂——综合大项目

本章你会做出什么

这是本书的压轴大项目。你要和 AI 一起，做一个互动故事游戏：AI 当"讲故事的人"，实时编出一个冒险故事；每讲到关键处，就停下来给你几个选择；你选了什么，故事就往哪个方向发展——每一次玩，都是一个独一无二、由你和 AI 共同创作的故事。

成品预览：

🏰 你的冒险开始了……
你站在一座古老城堡的大门前，门缝里透出微弱的蓝光。
你要怎么做？


    

  1. 推开大门走进去
    2. 

  2. 绕到城堡后面找别的入口
你的选择（1/2）：1
    3. 



你推开吱呀作响的大门，一只会说话的猫头鹰落在你肩上……

这一章会把你在这本书里学的几乎所有东西都用上。做完它，你就是一个能独立做出 AI 小应用的人了。准备好了吗？

这个项目难在哪、妙在哪

第 8 章你做过文字冒险游戏，但那时候所有剧情、所有分支都是你自己一句句写死的——写起来很累，故事也是固定的。

这次不一样：故事由 AI 现编。你只给它一个开头设定（比如"一场城堡探险"），剩下的它来讲。它会：

  1. 讲一段故事。
  2. 在关键处停下，给你 2 个选择。
  3. 等你选，然后接着你选的方向继续讲。

要做到"接着你选的方向讲"，靠的还是第 14 章的核心秘密——用列表维护对话记忆。AI 记得整个故事讲到哪了，所以能接得上。

图 15-1　故事树：AI 讲一段并给出选择，你的选择决定故事的走向，如此循环

关键点：用 system 定好"游戏规则"

要让 AI 每次都"讲一段 + 给两个选择"，我们得在 system（第 14 章学的人设/规则）里把游戏规则讲清楚。这段规则很重要，我们让 AI：

  • 当一个讲故事的人，语言生动、适合初中生；
  • 每次只讲一小段（别一口气讲完）；
  • 每段结尾给出 2 个选择，并明确标上"1." 和 "2."；
  • 如果故事该结束了，就给个结局，不再给选择。

这段 system 就像游戏的"说明书"，AI 会全程照着做。

完整代码

这个项目会用到函数、循环、条件、列表、f-string、AI 调用——全在这儿了。新建 story_factory.py：

Python
# story_factory.py
import anthropic

client = anthropic.Anthropic()

# ===== 游戏规则：讲给 AI 听的"说明书" =====
rules = """你是一个互动冒险故事的讲述者，读者是初中生。请遵守以下规则：
1. 每次只讲一小段故事（3-6 句话），生动有画面感，适合青少年，内容积极健康。
2. 每段故事结尾，给读者两个不同的行动选择，必须严格用这样的格式另起两行：
1. 第一个选择
2. 第二个选择
3. 当故事进行到足够精彩的结局时（大约 5-6 段之后），给出一个完整的结局，
   并在结尾单独写一行"【故事结束】"，这时就不要再给选择了。
4. 直接开始讲故事，不要说"好的""当然"之类的开场白。"""


def tell_story(messages):
    """把当前对话发给 AI，返回它讲的这一段故事（文字）。"""
    response = client.messages.create(
        model="claude-haiku-4-5",
        max_tokens=500,
        system=rules,
        messages=messages,
    )
    text = ""
    for block in response.content:
        if block.type == "text":
            text += block.text
    return text


# ===== 游戏开始 =====
print("🏰 欢迎来到 AI 故事工厂！")
theme = input("你想要一个什么主题的冒险故事？（如：城堡探险 / 太空历险 / 海底寻宝）\n> ")

# messages 是故事的"记忆"。第一句用户消息用来开启故事。
messages = [
    {"role": "user", "content": f"请开始一个关于「{theme}」的互动冒险故事。"}
]

print("\n你的冒险开始了……\n")

# 主循环：一直讲，直到故事结束
while True:
    # 1. 让 AI 讲一段
    story = tell_story(messages)
    print(story)

    # 2. 把 AI 讲的这段存进记忆
    messages.append({"role": "assistant", "content": story})

    # 3. 如果 AI 说故事结束了，就收尾退出
    if "【故事结束】" in story:
        print("\n🎬 这个故事就到这里啦，谢谢你的冒险！")
        break

    # 4. 让玩家做选择
    choice = input("\n你的选择（输入 1 或 2，或输入'退出'结束）：")
    if choice == "退出":
        print("下次再来冒险哦！")
        break

    # 5. 把玩家的选择存进记忆，AI 下一轮就会接着这个方向讲
    messages.append({"role": "user", "content": f"我选择 {choice}，请接着讲下去。"})
    print()   # 空行，让排版好看一点

设置好密钥（见第 13 章）后运行 python story_factory.py，一个属于你的互动冒险就开始了！输入不同的主题、做不同的选择，你会得到完全不同的故事。把这个游戏分享给同学，看看谁能玩出最离奇的结局！

常见错误

1. AI 有时不给选择，或格式乱

AI 不是机器般严格，偶尔可能忘了给两个选择，或者结束时忘了写"【故事结束】"。这不是你的 bug。改善办法：把 rules 里的格式要求写得更死、更清楚，或者在提示词里多强调一遍。这也是在教你一个真相——和 AI 合作，要靠清晰的规则去"引导"它，而不是指望它永远完美。

2. 故事永远不结束

如果 AI 一直不写"【故事结束】"，while 循环就停不下来。我们已经留了一个"后门"：玩家随时可以输入"退出"来结束。这也是好习惯——永远给死循环留一个出口。（第 5 章的安全提醒，还记得吧？）

3. 玩得越久越慢、越费一点点钱

故事越长，messages 记忆越大，每次发给 AI 的内容越多，就越慢、越费额度。玩一局尽兴就好。

4. 把 rules 放进了 messages

规则要用单独的 system 参数传（system=rules），不要塞进 messages 列表。这点和第 14 章一样。

动手挑战

这是最后一章的挑战，放开手脚，把这个故事工厂改造成你自己的作品吧！

★ 挑战一：给故事配个开场 在冒险开始前，先让程序打印一段有气氛的开场白（用 print 和一点 emoji），让玩家更有代入感。

★★ 挑战二：三个选择 把规则改成每次给3 个选择，让玩家输入 1/2/3。想一想：选择变多，故事会不会更有趣？

★★★ 挑战三：给你的故事配一张"角色卡" 在游戏开始时，让玩家先输入自己的角色名字和一个特长（比如"会魔法""跑得快"），把这些信息拼进第一条消息告诉 AI，让 AI 把玩家本人写成故事的主角。这样每个人玩的都是"以自己为主角"的专属冒险！

超级挑战（选做）：把学过的东西再组合一次 你已经会做问答游戏（第 15 章）、聊天机器人（第 14 章）、故事工厂（本章）了。试着把它们混搭：比如做一个"冒险途中会遇到 AI 出的谜题，答对才能继续前进"的游戏。这已经是一个相当完整的作品了——如果你能做出来，真的非常了不起。
