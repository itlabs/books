第 7 章 猜数字大作战

本章你会做出什么

这一章你要做出你的第一个完整游戏——猜数字。电脑会偷偷想一个 1 到 100 之间的数，你来猜。每猜一次，电脑就提示你"太大了"还是"太小了"，帮你一步步逼近答案。猜中时它会喝彩，还会告诉你一共用了几次。玩起来又紧张又上瘾！

成品预览（下划线是你打进去的数）：

我想好了一个 1 到 100 的数，来猜猜看！
请输入你猜的数：50
太大了！往小里猜。
请输入你猜的数：25
太小了！往大里猜。
请输入你猜的数：37
太大了！往小里猜。
请输入你猜的数：31
猜对了！你用了 4 次，太厉害了！

好消息：这一章的每个完整程序，你都可以点右上角的 ▶ 运行 按钮，直接在网页里玩起来。程序问你数字时，会弹出一个输入框，把数字打进去按确定就行。

前面几章你学会了 print、变量、input、if、循环、列表、字典、函数——这些全是造游戏的零件。今天我们就把它们拼起来，造出一个真正能玩的东西。别急，我们一块一块地搭。

第一块零件：让电脑"想"一个数

游戏的关键是：电脑得先想一个数，而且这个数每次玩都不一样，不然你一下就背下来了。可是电脑很笨，它不会自己"随便想"。我们得请一个专门的帮手——random 模块（module）。

模块就是别人提前写好、打包送给你的一箱工具。Python 自带很多这样的工具箱，random（英语"随机"）就是其中一个，专门负责"制造随机、说不准的结果"，比如掷骰子、抽签、洗牌。

要用工具箱，先得把它"搬进来"，这一步叫 import（导入）：

Python 
# guess_random.py
import random

# random.randint(1, 100) 会随机给出一个 1 到 100 之间的整数（两头都算）
secret = random.randint(1, 100)
print("电脑想的数是：", secret)

运行几次看看，你会发现每次打印出来的数都不一样——有时 7，有时 88。这就是"随机"。

拆开看这行 secret = random.randint(1, 100)：

  • random. 表示"到 random 这个工具箱里"，randint 是箱子里的一个工具（rand 是 random 的缩写，int 是 integer"整数"，合起来就是"随机整数"）。
  • 括号里的 1, 100 是范围的两端，表示"从 1 到 100 里挑一个，1 和 100 都可能被挑中"。
  • 挑出来的数，用 = 装进变量 secret（英语"秘密"）。
图 7-1　random.randint 随机挑一个整数，装进 secret 变量里

小贴士：import random 一般写在程序最上面，只需写一次，后面整份代码都能用这个工具箱了。

第二块零件：让玩家猜一次

现在电脑有了秘密数字，该让玩家出手了。用上一章就熟悉的 input() 接住玩家打的字。但要小心一个大坑：input() 接到的永远是"文字"，不是"数字"。

也就是说，玩家打了 50，程序拿到的其实是文字 "50"，没法直接和数字比大小。我们得用 int() 把它"翻译"成真正的整数（int 是 integer"整数"的缩写）。

Python 
# guess_once.py
import random

secret = random.randint(1, 100)

# input 拿到的是文字，套上 int() 变成整数
guess = int(input("请输入你猜的数："))

if guess == secret:
    print("猜对了！")
else:
    print("没猜中。电脑想的是", secret)

这就是一次完整的猜测：想数 → 玩家输入 → 转成整数 → 比一比。运行试试，虽然现在只能猜一次，但游戏的"心脏"已经跳起来了。

第三块零件：告诉玩家"大了还是小了"

只说"对/不对"太没意思了。真正好玩的地方在于提示——猜大了就说大了，猜小了就说小了，玩家才能一步步逼近答案。这正好用 if / elif / else 三岔路：

Python 
# guess_hint.py
import random

secret = random.randint(1, 100)
guess = int(input("请输入你猜的数："))

if guess > secret:
    print("太大了！往小里猜。")
elif guess < secret:
    print("太小了！往大里猜。")
else:
    print("猜对了！")
  • 如果猜的比秘密数大，提示"太大了"；
  • 否则如果比它小，提示"太小了"；
  • 剩下的情况（不大也不小，那就是相等），就是"猜对了"。

三种情况刚好被 if / elif / else 分得干干净净，不重不漏。

第四块零件：让玩家一直猜到对为止

一次机会太少了。我们想让玩家反复猜，直到猜中才停。"重复做某件事，直到条件满足"——这正是 while 循环的拿手好戏。

我们让循环的条件是"还没猜中"。用一个变量记录玩家最近一次猜的数，只要它还不等于 secret，就继续循环、继续问。

Python 
# guess_loop.py
import random

secret = random.randint(1, 100)
print("我想好了一个 1 到 100 的数，来猜猜看！")

guess = 0  # 先随便放一个不可能等于 secret 的初始值

# 只要还没猜对，就一直循环
while guess != secret:
    guess = int(input("请输入你猜的数："))
    if guess > secret:
        print("太大了！往小里猜。")
    elif guess < secret:
        print("太小了！往大里猜。")
    else:
        print("猜对了！")

这里有两个要点：

  1. 循环开始前，先给 guess 一个初始值 0。因为 while 一上来就要检查 guess != secret，guess 得先存在才行。0 不在 1~100 范围内，肯定不等于 secret，保证循环至少跑一次。
  2. != 是"不等于"的意思。while guess != secret 读作"当 guess 还不等于 secret 时，继续循环"。一旦猜中，guess 就等于 secret，条件变成假，循环自动结束。
图 7-2　while 循环：猜错就回头再猜，猜对才走出循环

第五块零件：数一数猜了几次

游戏得有点"成绩"才带劲。我们加一个计数器（counter），每猜一次就加 1，猜中时报出总次数。

计数器就是一个普通变量，套路永远是：循环开始前设为 0，循环里每做一次就 + 1。

Python 
# guess_count.py
import random

secret = random.randint(1, 100)
print("我想好了一个 1 到 100 的数，来猜猜看！")

guess = 0
count = 0  # 计数器：记录猜了几次

while guess != secret:
    guess = int(input("请输入你猜的数："))
    count = count + 1  # 每猜一次就加 1
    if guess > secret:
        print("太大了！往小里猜。")
    elif guess < secret:
        print("太小了！往大里猜。")
    else:
        print(f"猜对了！你用了 {count} 次，太厉害了！")

count = count + 1 这行是重点，读作"把 count 现在的值加 1，再存回 count"。比如 count 原来是 3，这行执行完就变成 4。这样每循环一次，count 就长大 1，猜中时它正好等于总猜测次数。

小贴士：count = count + 1 可以简写成 count += 1，效果完全一样，程序员们更爱用这种短写法。

第六块零件：限制猜的次数

无限次太容易了。给游戏加点难度：最多只能猜 7 次，超过就算输，还要揭晓答案。这样玩家每一次都得动脑筋。

我们把循环条件改成"还没猜中，并且次数还没用完"。两个条件用 and（英语"并且"）连起来——两个都成立，循环才继续。

Python 
# guess_game.py 猜数字大作战 完整版
import random

secret = random.randint(1, 100)
max_times = 7  # 最多能猜的次数
count = 0
guess = 0

print("我想好了一个 1 到 100 的数。")
print(f"你只有 {max_times} 次机会，来挑战吧！")

# 还没猜中，并且次数没用完，就继续
while guess != secret and count < max_times:
    guess = int(input("请输入你猜的数："))
    count += 1
    if guess > secret:
        print("太大了！往小里猜。")
    elif guess < secret:
        print("太小了！往大里猜。")
    else:
        print(f"猜对了！你只用了 {count} 次，神了！")

# 循环结束后，判断是猜中出来的，还是次数用光了
if guess != secret:
    print(f"机会用完啦！我想的数是 {secret}，下次加油！")

这就是我们的完整游戏了！它比第一块零件复杂了很多，但你看——它其实就是六块小零件一点点拼起来的。任何复杂程序，都是这样从小往大搭出来的。 这是编程最重要的思路，请你记住它。

想一想：为什么最后要用 if guess != secret 再判断一次？因为循环有两种结束方式——猜中了、或次数用完了。这一步是分辨"你到底是赢了还是输了"。

一个聪明的玩法：折半猜

跟你分享一个小策略：每次都猜当前范围的正中间。1100 就先猜 50，太大就猜 25，太小就猜 75……每猜一次，范围就砍掉一半。这样最多 7 次，就能猜中 1100 里的任何数！这也是为什么我们把机会设成 7 次——刚好够聪明的人赢。这个"每次砍一半"的思路，在编程里叫二分查找，以后你会经常见到它。

常见错误

1. 忘了用 int()，直接拿文字比大小

Python
guess = input("请输入你猜的数：")  # 少了 int()
if guess > secret:  # 文字和数字比大小
    print("太大了")

报错：

TypeError: '>' not supported between instances of 'str' and 'int'

TypeError（类型错误）是说：文字（str）和整数（int）没法比大小。修法是把 input(...) 用 int(...) 包起来：guess = int(input("..."))。

2. 忘了给循环变量设初始值

Python
import random
secret = random.randint(1, 100)
while guess != secret:  # guess 还没出生就被用了
    guess = int(input("请猜："))

报错：

NameError: name 'guess' is not defined

while 一开始就要检查 guess，可这时 guess 还不存在。修法是在循环前加一行 guess = 0。

3. 玩家输入了不是数字的东西

Python
guess = int(input("请输入你猜的数："))  # 玩家却打了 "abc"

报错：

ValueError: invalid literal for int() with base 10: 'abc'

int() 没法把"abc"变成数字。目前只要提醒玩家老老实实输数字就行；等你学了更多，可以用别的办法拦住乱输入。

4. 把 == 写成了 =

判断"相等"要用两个等号 ==。一个等号 = 是"赋值"（装进盒子），意思完全不同。if guess = secret 会直接报 SyntaxError。记住：比较用 ==，赋值用 =。

动手挑战

在完整版 guess_game.py 的基础上改造，让游戏更好玩。

★ 挑战一：改改范围和次数 把秘密数字的范围改成 1 到 1000，机会改成 10 次。想一想：范围变大了，为什么 10 次还够用？（提示：想想"折半猜"。）

★★ 挑战二：贴心的差距提示 在"太大了/太小了"之外，再根据猜的数和答案差多远，给出"就差一点点"（差 5 以内）或"差得远呢"（差 20 以上）的提示。（提示：可以用 abs(guess - secret) 算出两个数相差多少，abs 会去掉负号只留大小。）

★★★ 挑战三：换人来出题 做一个"双人版"：先让玩家 A 输入一个秘密数字（想办法别让玩家 B 看见），再让玩家 B 来猜。还可以记录两人各自猜中用的次数，比比谁更厉害。

本章小结

你做出了人生第一个完整游戏，这一步意义重大！回顾一下：

  • 模块是打包好的工具箱，用 import 导入。random 模块能制造随机结果。
  • random.randint(a, b) 随机返回 a 到 b 之间的一个整数（两端都可能取到）。
  • input() 拿到的是文字，比大小前要用 int() 转成整数。
  • while 循环让程序反复做同一件事，直到条件不再成立；用它实现"猜到对为止"。
  • 计数器变量（count += 1）能统计某件事发生了几次。
  • 多个条件可以用 and（并且）连接，都成立时才算真。
  • 最重要的心法：大程序是由一块块小零件慢慢拼起来的，先让每一小块跑通，再往上加。
