## 一个秘密：一套套路，无数应用

回想一下，从第 13 章到现在，每个 AI 程序是不是都长这样：

1. `import anthropic`，创建客户端 `client = anthropic.Anthropic()`；
2. 用 `system` 给 AI 定个**人设**（你是翻译官 / 你是学习搭子……）；
3. 把用户的话拼成**提示词**，用 `client.messages.create(...)` 发过去；
4. 遍历 `response.content`，取出 `text` 打印出来。

**骨架不变，变的只是"人设"和"提示词"这两处。** 就像同一台游戏机，换一张不同的卡带，就能玩不同的游戏。这一章的四个应用，全都是"换卡带"换出来的。

## 应用一：多语言翻译官

**思路**：让 AI 当一个专业翻译。用户输入一句中文，再说要翻译成哪种语言，我们把这两样拼成提示词发给 AI。为了能一句接一句地翻，外面套一个 `while` 循环（第 5 章学的），输入"退出"才停。

新建 `translator.py`：

```python-norun
# translator.py
import anthropic

client = anthropic.Anthropic()

# 给 AI 定个"翻译官"人设：只翻译，不啰嗦
system_prompt = "你是一个专业的翻译官。用户给你一句中文和一个目标语言，你只需给出翻译结果，不要加任何解释或多余的话。"

print("=== 多语言翻译官 ===")

# 主循环：可以一句接一句地翻译
while True:
    text = input('\n请输入要翻译的中文（输入"退出"结束）：')
    if text == "退出":
        print("再见！")
        break

    language = input("翻译成哪种语言？（如 英语/日语/法语）：")

    # 用 f-string 把用户的输入拼成一句清楚的指令
    question = f"请把这句话翻译成{language}：{text}"

    response = client.messages.create(
        model="claude-haiku-4-5",
        max_tokens=200,
        system=system_prompt,
        messages=[
            {"role": "user", "content": question}
        ],
    )

    # 取出翻译结果
    for block in response.content:
        if block.type == "text":
            print("翻译结果：", block.text)
```

运行结果（你的可能不同）：

```
=== 多语言翻译官 ===

请输入要翻译的中文（输入"退出"结束）：我喜欢编程
翻译成哪种语言？（如 英语/日语/法语）：日语
翻译结果： プログラミングが好きです。

请输入要翻译的中文（输入"退出"结束）：退出
再见！
```

看，是不是很简单？还是那套熟悉的骨架，只是 `system` 换成了"翻译官"，提示词换成了"请把这句话翻译成……"。

**可以怎么改：**

- 反过来翻：让它把英语翻译成中文，只要改改提示词。
- 一次翻多种：把提示词改成"请把这句话同时翻译成英语、日语、法语，分行列出"。
- 加上拼音或读法：提示词里加一句"并在翻译后附上读音"。

## 应用二：作业小助手（你的学习搭子）

**思路**：把 AI 设成一个耐心的**学习搭子**，帮你检查一段话里的**错别字**和**病句**，并给出修改建议。

但这里有一件更重要的事要先说清楚 👇

> 💡 **重要：AI 是帮你"学"，不是替你"做"**
> 用 AI 检查作文，是为了让你**看懂哪里错了、为什么错**，下次自己就能写对——这叫**学习**。
> 如果只是让 AI 把整篇作文替你写好、你照抄交上去，那你其实什么都没学到，考试时还是不会，这叫**抄袭**，一点意义都没有。
> 所以我们特意让这个助手：**只指出问题、给建议、讲原因，但不替你重写整篇**。真正动手改的人，永远是你自己。

新建 `study_buddy.py`：

```python-norun
# study_buddy.py
import anthropic

client = anthropic.Anthropic()

# 人设：一个只帮忙、不代劳的学习搭子
system_prompt = """你是一个耐心友善的学习搭子，帮助初中生修改作文。请遵守：
1. 找出文字里的错别字和病句，逐条列出来，并简单说明为什么错。
2. 给出修改建议，但不要替对方把整篇文章重写好，要引导他自己改。
3. 语气鼓励、温和，先肯定写得好的地方，再提建议。"""

print("=== 作文小助手（你的学习搭子）===")
print("把你写的一段话贴进来，我帮你看看有没有错别字和病句。\n")

my_text = input("请输入你的文字：")

# 把用户的文字拼进提示词
question = f"请帮我检查下面这段话，找出错别字和病句并给修改建议：\n{my_text}"

response = client.messages.create(
    model="claude-haiku-4-5",
    max_tokens=600,
    system=system_prompt,
    messages=[
        {"role": "user", "content": question}
    ],
)

print("\n--- 学习搭子的建议 ---")
for block in response.content:
    if block.type == "text":
        print(block.text)
```

运行结果（示例）：

```
=== 作文小助手（你的学习搭子）===
把你写的一段话贴进来，我帮你看看有没有错别字和病句。

请输入你的文字：今天我和同学去公园完，我们玩的很开心，在那里我们观查了很多小动物。

--- 学习搭子的建议 ---
你这段话写得很有生活气息，能看出你玩得很开心！我帮你挑几个小问题：
1. "去公园完" —— "完"应该是"玩"，写错字啦。
2. "玩的很开心" —— 这里的"的"要用"得"，因为它修饰"玩"这个动作。
3. "观查" —— 正确写法是"观察"。
你自己试着把这三处改一改，改完再读一遍，看看是不是通顺多啦？
```

**可以怎么改：**

- 改成"英语作文助手"：帮你检查英语句子的语法（记得在 `system` 里说明）。
- 加个"打分+鼓励"：让它先夸一句，再给一个 1—10 分的进步分数。
- 做成循环：像翻译官那样套个 `while`，让你能一段一段地检查。

## 应用三：创意点子机 / 取名神器

**思路**：AI 特别擅长"头脑风暴"。给它一个主题，让它一口气生成一批点子或名字。这个应用几乎就是纯好玩——给社团取名、给宠物起名、给作文想标题，都能用。

新建 `idea_machine.py`：

```python-norun
# idea_machine.py
import anthropic

client = anthropic.Anthropic()

# 人设：一个脑洞大开的点子专家
system_prompt = "你是一个脑洞大开、创意十足的点子专家，擅长为各种主题想出有趣又不俗套的名字和创意。"

print("=== 创意点子机 ===")
topic = input("你想为什么想点子？（如：给编程社团取名 / 给我的机器人起名）：")
count = input("要几个？（直接回车默认 8 个）：")

# 如果用户没输入数量，就用默认值 8
if count == "":
    count = "8"

# 把主题和数量都拼进提示词
question = f"请围绕「{topic}」，给我 {count} 个有创意的点子，用编号列出，每个后面用一句话说明为什么好。"

response = client.messages.create(
    model="claude-haiku-4-5",
    max_tokens=600,
    system=system_prompt,
    messages=[
        {"role": "user", "content": question}
    ],
)

print("\n--- 为你想到的点子 ---")
for block in response.content:
    if block.type == "text":
        print(block.text)
```

运行结果（示例）：

```
=== 创意点子机 ===
你想为什么想点子？（如：给编程社团取名 / 给我的机器人起名）：给编程社团取名
要几个？（直接回车默认 8 个）：5

--- 为你想到的点子 ---
1. 代码星球 —— 每个人都是探索代码宇宙的小小宇航员。
2. Bug 猎人 —— 听起来又酷又有使命感，抓虫子就是我们的日常！
3. 键盘乐队 —— 敲代码就像合奏一首歌，团队感十足。
4. 0 和 1 的魔法师 —— 强调编程像变魔法一样神奇。
5. 未来工程队 —— 大气、正能量，适合想做大项目的社团。
```

**可以怎么改：**

- 换主题：给游戏角色起名、给短视频想标题、为班级设计口号……
- 限定风格：提示词里加"要求名字都很中二/很温柔/带动物"，风格立刻变。
- 让它二选一：把生成的点子再发给 AI，问"哪个最好，为什么"。

## 应用四：每日鼓励语生成器（轻松一下）

**思路**：最后来个特别轻松的。给 AI 一个你今天的心情或处境，让它送你一句量身定做的鼓励语。累了、考砸了、紧张了，运行一下，收一句暖心话。

新建 `daily_cheer.py`：

```python-norun
# daily_cheer.py
import anthropic

client = anthropic.Anthropic()

# 人设：一个温暖的加油鼓励专家
system_prompt = "你是一个温暖、真诚的鼓励大师。根据用户当下的心情，送他一句简短有力、发自内心的鼓励，不超过两句话，不要说教。"

print("=== 每日鼓励语 ===")
mood = input("说说你现在的心情或遇到的事吧：")

question = f"我现在的情况是：{mood}。请送我一句鼓励的话。"

response = client.messages.create(
    model="claude-haiku-4-5",
    max_tokens=150,
    system=system_prompt,
    messages=[
        {"role": "user", "content": question}
    ],
)

print()
for block in response.content:
    if block.type == "text":
        print("💪", block.text)
```

运行结果（示例）：

```
=== 每日鼓励语 ===
说说你现在的心情或遇到的事吧：数学考试没考好，有点难过

💪 一次考试说明不了你有多厉害，它只是告诉你下一步往哪儿使劲。你已经在认真对待了，这就很棒。
```

**可以怎么改：**

- 做成"表情包文案生成器"：让它根据一个场景，生成一句适合配图的搞笑文字。
- 每天不重样：多运行几次，因为 AI 每次回答都不同，收集你最喜欢的贴在书桌上。
- 做成"夸夸机"：让它专门找出你身上的优点，逢人就夸（自己给自己打气也行）。

| 学习搭子 | 你是耐心的学习搭子 | 请帮我检查错别字和病句…… |

**骨架完全一样，只换了两处。** 这意味着：只要你能想清楚"让 AI 扮演谁"和"让它做什么"，你就能造出属于自己的新应用。AI 编程的天花板，其实是你的**想象力**。

## 常见错误

**1. AI 回答里带了多余的话**

比如翻译时它还礼貌地说"好的，为您翻译如下："。这不是 bug，是它太"热情"了。解决办法：在 `system` 里把要求写死一点，比如"只给结果，不要任何多余的话"（翻译官那段就是这么写的）。

**2. 输入数量却写成了数字**

在点子机里，`input()` 拿到的永远是**文字**（字符串），哪怕你输入 5，它也是 `"5"`。好在我们是把它拼进提示词给 AI 看的，AI 认得字符串里的数字，所以没问题。但如果你想用它做数学运算（比如 `count * 2`），就要先 `int(count)` 转成整数——这是第 3 章的老朋友了。

**3. 忘了设密钥 / 忘了联网**

报错 `AuthenticationError`（没密钥）或 `APIConnectionError`（没联网）。检查：有没有在**同一个终端窗口**先设好密钥？网络通不通？（详见第 13 章的"常见错误"。）

**4. 每个程序单独跑**

这一章有四个 `.py` 文件，它们是**四个独立的小程序**，各跑各的，不要指望在一个文件里定义、另一个文件里用。

## 动手挑战

**★ 挑战一：给翻译官加个"反向档"**
改造 `translator.py`，让它既能中译外，也能外译中。（提示：可以先问用户"要中译外还是外译中"，再拼不同的提示词。）

**★★ 挑战二：合体！万能小助手**
做一个带菜单的程序：开头让用户选"1. 翻译 2. 改作文 3. 想点子"，选几就跑对应的功能。（提示：用 `if / elif` 判断，把这一章的代码拼起来就行。）

**★★★ 挑战三：发明你自己的 AI 应用**
这才是真正的挑战——**别抄这四个，想一个全新的**。先问自己两个问题：
1. 我想让 AI 扮演**谁**？（这决定 `system`）
2. 我想让它帮我做**什么**？（这决定提示词）

比如："菜谱推荐官"（告诉它冰箱里有啥，它给你想菜）、"梦境解读员"、"古诗接龙搭子"、"英语单词造句机"……想到了就动手写，用这一章的骨架，把 `system` 和提示词换成你自己的。**做出来的那一刻，你就不只是在学编程了——你在用编程创造。**

## 本章小结

这一章你没学什么全新的语法，却做出了四个能用的 AI 应用。回顾一下：

- AI 应用的**骨架是固定的**：创建客户端 → 用 `system` 定人设 → 拼提示词发送 → 遍历 `response.content` 取 `text`。
- **换个人设 + 换段提示词 = 一个全新应用**。翻译官、学习搭子、点子机、鼓励师，都是这么来的。
- 好用的提示词要**清楚**：让 AI 演谁、做什么、按什么格式回答，说得越明白，结果越好。
- **AI 是帮你学的工具，不是替你做的枪手**。用它看懂问题、学会本领，才是真本事；照抄一点意义都没有。
- 造新应用的关键，是先想清楚两件事：**让 AI 扮演谁**、**让它做什么**。
