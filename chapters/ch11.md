## 什么是 pygame

上一章的海龟只能画静止的图。要做游戏，画面得**动起来、还能响应你的按键**。这就需要一个更专业的工具——**pygame**。

pygame 是一盒专门用来做游戏的工具箱：它能开一个窗口、在里面画方块画图、听你按了哪个键、每秒刷新几十次画面让东西看起来在动。很多小游戏都是用它做的。

和 turtle 不同，pygame **不是 Python 自带的**，你得先把它"下载安装"到电脑上，才能用。

## 第一步：安装 pygame

**`pip`** 是 Python 的"应用商店店员"，专门帮你下载安装别人写好的工具包。`install` 是"安装"，`pygame` 是要装的东西。这行命令的意思就是："pip，帮我把 pygame 装上。"

## 游戏的心脏：主循环

在写代码前，先弄懂一个所有游戏都逃不开的道理。你玩的每一个游戏——不管是超级马里奥还是手机上的消消乐——其实都在**飞快地重复做同样三件事**：

1. **处理输入**：看看玩家按了什么键、点了哪里、有没有想关窗口。
2. **更新状态**：根据输入算出新情况——方块该往哪挪、得分加没加、谁赢了。
3. **画出画面**：把最新的样子画到屏幕上。

做完这三步，马上从头再来一遍。一秒钟转个几十圈，快到你眼睛看不出"一圈一圈"，只觉得画面在流畅地动。这个不停打转的循环，就是**游戏主循环（game loop）**，是整个游戏的心脏。

## 最小骨架：开一个窗口

先做出最简单的游戏——一个能打开、能关掉的空窗口。别看它什么都没有，主循环该有的架子全在这儿了。新建文件 `window.py`：

```python-norun
# window.py
import pygame

pygame.init()                              # 启动 pygame，做好准备工作

screen = pygame.display.set_mode((600, 400))  # 开一个 600×400 的窗口
pygame.display.set_caption("我的第一个游戏")    # 设置窗口标题

running = True                             # 游戏是否在运行，True 表示"是"
while running:                             # 游戏主循环：只要 running 为真就一直转
    for event in pygame.event.get():       # ① 处理输入：看看发生了什么事件
        if event.type == pygame.QUIT:      # 如果玩家点了窗口的关闭按钮
            running = False                # 就让 running 变假，循环会停下

    screen.fill((0, 0, 0))                 # ③ 画面：把整个屏幕涂成黑色
    pygame.display.flip()                  # 把刚画的东西真正显示出来

pygame.quit()                              # 循环结束后，关闭 pygame
```

在自己电脑上运行，会弹出一个 600×400 的黑窗口，标题写着"我的第一个游戏"。点右上角的关闭按钮，窗口就正常关掉。逐个认识这些新东西：

- **`pygame.init()`**：`init` 是 "initialize"（初始化）的缩写，意思是"启动、做好开工前的准备"。用 pygame 前必须先叫它一次。
- **`pygame.display.set_mode((600, 400))`**：开一个窗口，宽 600、高 400 像素。它会把这个窗口"交给"你，我们存进变量 `screen`（屏幕），以后往上面画东西都用它。注意里面是**两层括号**——`(600, 400)` 是一对绑在一起的宽和高。
- **`while running:`**：这就是主循环。`running` 是一个开关，为 `True`（真）时循环一直转，一旦变 `False`（假）循环就停。
- **`pygame.event.get()`**：拿到"这一圈里发生的所有事件"——按键、点鼠标、关窗口都算**事件（event）**。用 `for` 循环挨个检查。
- **`event.type == pygame.QUIT`**：判断这个事件是不是"玩家想关窗口"（QUIT 是"退出"）。是的话就把开关关掉。
- **`screen.fill((0, 0, 0))`**：把屏幕整个涂满颜色。这里的 `(0, 0, 0)` 是颜色的另一种写法——三个数分别代表红、绿、蓝的多少（0～255），`(0,0,0)` 全是 0 就是黑色。
- **`pygame.display.flip()`**：把这一圈画好的画面"翻"到屏幕上，让你真正看到。**没有它，画了也白画。**

> 小贴士：几乎每个 pygame 游戏都长这个样子——初始化、开窗口、`while` 主循环里处理事件+画画、最后 `pygame.quit()`。把这个骨架记熟，以后做任何游戏都从它起步。

## 画一个方块

空窗口太寂寞，往里画个方块。画矩形用 **`pygame.draw.rect`**，把上面 `screen.fill` 那行下面加一句：

```python-norun
# 在 screen.fill((0, 0, 0)) 之后、flip() 之前加这行
pygame.draw.rect(screen, (224, 106, 59), (100, 100, 50, 50))
```

拆开看 `pygame.draw.rect` 的括号里放了什么：

- `screen`：画在哪个屏幕上。
- `(224, 106, 59)`：什么颜色，这三个数是橙色（红多、绿中、蓝少）。
- `(100, 100, 50, 50)`：**位置和大小**，四个数依次是——左上角横坐标 x、纵坐标 y、宽、高。所以这是一个左上角在 (100,100)、50×50 的小方块。

> 小贴士：pygame 的坐标和数学课上不太一样：**原点 (0,0) 在窗口左上角**，x 越大越往**右**，y 越大越往**下**。想让方块往下走，就把 y 变大。

## 让方块动起来：处理键盘

现在到最激动人心的部分——**用方向键控制方块**。思路是：

1. 用两个变量 `x`、`y` 记住方块**现在**在哪。
2. 每一圈主循环里，看玩家按没按方向键；按了就改一改 `x` 或 `y`。
3. 用最新的 `x`、`y` 把方块画出来。

方块的位置一变，下一圈画的地方就变了，连起来看就是它在"动"。

怎么知道当前按着哪个键？用 **`pygame.key.get_pressed()`**，它一次性告诉你**所有键此刻是不是被按着**。然后这样判断：

```python-norun
keys = pygame.key.get_pressed()
if keys[pygame.K_LEFT]:    # 左方向键正被按着
    x = x - 5              # x 变小 → 方块往左
```

`pygame.K_LEFT`、`K_RIGHT`、`K_UP`、`K_DOWN` 分别代表左右上下四个方向键（K 是 key"键"）。用 `if` 判断哪个被按着，就往哪个方向挪。因为这个检查每圈都做，你**按住不放**方块就会一直动，特别顺滑。

## 完整程序：能用方向键移动的方块

把上面所有零件拼起来，就是本章的完整成品。新建 `move_block.py`：

```python-norun
# move_block.py
import pygame

pygame.init()

screen = pygame.display.set_mode((600, 400))   # 600×400 的窗口
pygame.display.set_caption("移动方块")
clock = pygame.time.Clock()                     # 用来控制游戏速度的"节拍器"

# 方块的初始位置和大小
x = 275          # 方块左上角的横坐标（放在窗口中间附近）
y = 175          # 方块左上角的纵坐标
size = 50        # 方块边长
speed = 5        # 每次移动多少像素

running = True
while running:
    # ① 处理输入
    for event in pygame.event.get():
        if event.type == pygame.QUIT:      # 点了关闭按钮
            running = False

    keys = pygame.key.get_pressed()        # 现在有哪些键被按着
    if keys[pygame.K_LEFT]:                # 左键：往左
        x = x - speed
    if keys[pygame.K_RIGHT]:               # 右键：往右
        x = x + speed
    if keys[pygame.K_UP]:                  # 上键：往上（y 变小）
        y = y - speed
    if keys[pygame.K_DOWN]:                # 下键：往下（y 变大）
        y = y + speed

    # ② 更新状态：别让方块跑出窗口
    if x < 0:
        x = 0
    if x > 600 - size:                     # 窗口宽 600，减去方块自身宽度
        x = 600 - size
    if y < 0:
        y = 0
    if y > 400 - size:                     # 窗口高 400
        y = 400 - size

    # ③ 画出画面
    screen.fill((0, 0, 0))                             # 背景涂黑
    pygame.draw.rect(screen, (224, 106, 59), (x, y, size, size))  # 画橙色方块
    pygame.display.flip()                              # 显示出来

    clock.tick(60)                         # 控制每秒最多转 60 圈（60 帧）

pygame.quit()
```

在自己电脑上运行它，一个黑窗口里出现橙色方块，按住方向键，它就朝那个方向滑动，撞到边就停住，跑不出窗口。**这是你做出的第一个真正能玩的游戏！**

又冒出两个新东西，认识一下：

- **`clock = pygame.time.Clock()` 和 `clock.tick(60)`**：主循环转得太快的话，方块会"嗖"地飞出去。`Clock` 是个节拍器，`tick(60)` 让循环每秒最多转 60 圈（也就是 **60 帧**），画面又稳又不费电。几乎所有 pygame 游戏都会加这一句。
- **"别让方块跑出窗口"那几个 `if`**：这就是主循环第②步"更新状态"在干的事——不光挪位置，还要检查规则。这里的规则是"不许出界"，超出边界就把它按回边上。

试着改改看：把 `speed` 调大方块跑得更快；把颜色 `(224, 106, 59)` 换成 `(63, 155, 92)` 变成绿色；把 `size` 改大改小。**改一个数，跑一次，看看有什么不同**——这就是做游戏最好玩的地方。

## 常见错误

pygame 要装、要在电脑上跑，新坑主要在环境和安装上：

**1. 没装 pygame 就运行**

```python-norun
import pygame
```

报错：

```
ModuleNotFoundError: No module named 'pygame'
```

`ModuleNotFoundError`（找不到模块）意思是电脑上还没有 pygame 这盒工具。回到终端跑一次 `pip install pygame` 装上它。如果 `pip` 也提示找不到命令，说明 Python 没装好或没配置好，回[附录 A](appendix-a.md)。

**2. 点了网页的运行按钮没反应**

和上一章一样：**pygame 的游戏窗口浏览器里开不出来。** 必须存成 `.py` 文件，在自己电脑上用 `python move_block.py` 运行。

**3. 窗口弹出来就卡死、转圈圈**

多半是**忘了写处理 `QUIT` 事件的那段**，或者干脆漏掉了 `for event in pygame.event.get():`。没有它，pygame 收不到你的操作，系统会以为程序"没反应"而卡住。记住主循环里**每一圈都要处理事件**。

**4. 画了方块却看不见**

如果方块死活不出现，检查两点：一是 `pygame.draw.rect` 是不是写在了 `screen.fill(...)` **后面**（先涂背景再画方块，不然会被背景盖住）；二是最后有没有 `pygame.display.flip()`——**不 flip，画什么都不会显示。**

## 本章小结

从文字游戏到真正的游戏窗口，你完成了一次大跨越！回顾这一章：

- **pygame** 是做游戏的工具箱，Python **不自带**，要先 `pip install pygame` 安装。
- 每个游戏都有一个**主循环**，不停地重复三件事：**处理输入 → 更新状态 → 画出画面**。
- 骨架四件套：`pygame.init()` 启动、`set_mode()` 开窗口、`while` 主循环、`pygame.quit()` 收尾。
- 每圈都要用 `pygame.event.get()` **处理事件**（尤其是关窗口的 `QUIT`），否则窗口会卡死。
- 用 `pygame.draw.rect()` 画方块；坐标原点在**左上角**，x 向右增大、y 向下增大。
- 用 `pygame.key.get_pressed()` 读方向键，改方块的 `x`、`y` 让它动；`pygame.display.flip()` 才能把画面显示出来；`clock.tick(60)` 控制速度。
- 老规矩：**pygame 要在自己电脑上运行**，网页运行按钮跑不了。

你已经会让电脑说话、算数、玩文字游戏、画图，现在还能做真游戏的窗口了。下一章，我们要用这一章学的 pygame，做一个**完整的、能玩上瘾的经典游戏——贪吃蛇**！它会把你学过的列表、循环、碰撞判断、记分全部用上，是你游戏生涯的第一个"大作"。做完它，你就真正入门游戏开发了。翻页吧！
