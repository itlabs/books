# 写作规范（本书统一风格）

面向读者：中国 7—9 年级学生（12—15 岁），零编程基础。

## 语气与语言
- 用"你"称呼读者，像一位耐心的大哥哥/大姐姐在旁边讲解。
- 句子短，多打比方（用生活里的事物类比抽象概念）。
- 不假设读者懂英语、数学超纲内容。出现的英文单词要顺带解释含义。
- 允许适度幽默，但不油腻；鼓励为主，多说"你已经会……了"。

## 结构（每章固定套路）
1. **本章你会做出什么**：开头用 2—3 句话+一张"成品预览"说明本章结束时能跑出的东西。
2. **正文**：概念 → 最小例子 → 讲解 → 动手改一改。每个新概念都配一段能直接运行的完整代码。
3. **完整代码**：每章至少有一份从头到尾能运行的完整程序，代码块顶部用注释写出文件名。
4. **常见错误**：列 2—4 个新手最容易踩的坑，给出报错样子和修法。
5. **动手挑战**：3 道循序渐进的小任务（★ 简单 / ★★ 中等 / ★★★ 挑战），只描述目标不给答案。
6. **本章小结**：用要点列表回顾学到的东西，并预告下一章。

## 代码规范
- 全部 Python 3。代码要**能直接复制运行**，不留 `...` 省略。
- 变量、函数名用英文小写+下划线（如 `player_name`），但注释一律中文。
- 注释解释"为什么"和"这行在干嘛"，密度适中，不要每行都写。
- 代码块标注语言：` ```python `。终端命令用 ` ```bash `。
- 展示程序输出时，用单独的代码块并在前面写"运行结果："。

## 排版
- 用 Markdown。标题层级：章标题 `#`，小节 `##`，子标题 `###`。
- 重要术语首次出现时**加粗**并给出中英文，例如"**变量（variable）**"。
- 用 > 引用块放"小贴士"和"安全提醒"。
- 每章开头写 `# 第 N 章 章节名`。

## 交互与可视化（重要！本书会转成网页）
Markdown 会被构建脚本转成交互式 HTML，请用好这些特性：

- **可运行代码**：普通 ```python 代码块在网页上会自动带"▶ 运行"按钮，
  学生能在浏览器里直接跑、直接改。所以 python 代码块务必是**完整可独立运行**的，
  不依赖前文定义的变量（每个块独立运行）。
- **不可运行的代码**（需要联网/密钥的 AI 代码、或故意展示报错的代码）：
  语言标 ```python-norun ，这样只高亮、不给运行按钮。
- **"本章你会做出什么"** 段落：紧跟标题，用一个 HTML 盒子包起来，格式：
  `<div class="goal-box">` ...内容（可含 markdown 无效，请写纯 HTML 段落 `<p>`）... `</div>`
  盒子里简述成品 + 可放一个"运行结果预览"的 `<pre>` 块。
- **概念图示**：讲抽象概念（变量是盒子、if 分支、循环、列表等）时，插入内联 SVG 图，
  用 `<div class="concept">...<svg ...>...</svg><div class="caption">图说明</div></div>` 包裹。
  SVG 用简单的矩形、箭头、文字即可，配色用橙(#e06a3b)、蓝(#2f6db3)、绿(#3f9b5c)。
  每章至少 1 张概念图。图要真的帮助理解，不是装饰。
- HTML 标签（div/svg/pre）直接写在 markdown 里即可，构建脚本会原样保留。
- 注意：写在 `<div>` 内部的内容不会被当作 markdown 解析，需要用 HTML 标签（`<p><ul><code>`）。

## AI 章节的特别约定（第12—15章）
- 使用 Anthropic 官方 Python SDK：`pip install anthropic`。
- 创建客户端：`client = anthropic.Anthropic()`，从环境变量 `ANTHROPIC_API_KEY` 读密钥。
- 调用：`client.messages.create(model="claude-haiku-4-5", max_tokens=..., messages=[...])`。
  面向学生、成本敏感，默认用便宜快速的 `claude-haiku-4-5`，并说明可换成更强的模型。
- 读取回复：遍历 `response.content`，判断 `block.type == "text"` 再取 `block.text`。
- 反复强调：**API 密钥是秘密，绝不能写进代码、发给别人、上传到网上**。
- 强调家长/老师协助申请密钥与注意花费。
